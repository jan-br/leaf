# Proxy & Interception Substrate  `[proxy-interception]`

_The one compile-time-generated transparent delegating-newtype-per-service-trait routing through the ordered/replayable/short-circuitable Interceptor chain whose innermost step is a TargetSource; the auto-proxy creator's O(1) ProxyPlan, advisor/pointcut model, and scoped-proxy TargetSource re-resolution._

The Proxy & Interception Substrate is the single wrap primitive on which all transparent, DI-seam-attached interception in leaf rests. There is exactly ONE proxy shape — a compile-time-generated transparent delegating newtype per advisable service trait (the rust-native CGLIB analogue moved to codegen) — and exactly ONE call-routing primitive — an ordered, replayable, short-circuitable `Interceptor` chain (`AdviceChain`) whose innermost step is a `TargetSource` (the per-call target supplier). The four features it consolidates are NOT four mechanisms but four uses of that one wrap:\n\n- **aop-proxying** owns the wrap-and-swap seam + the `Interceptor`/`Call`/`Next` call-routing primitive (the generated transparent newtype + the `ErasedProxy` dynamic fallback are its outputs).\n- **auto-proxy-creator** owns the O(1) frozen `ProxyPlan` computed at the `seal()` boundary (collect → match → union → order, role-filtered by a binary-root-assembled `CreatorPolicy`), installed at the ownership-model `after_init` decoration hook.\n- **aspect-model** owns advice authoring: `#[aspect]`/`#[advice]`/`#[pointcut]` lower to flat const `AdvisorDescriptor`s with typed-combinator pointcuts (and an optional expression front-end), all collected through the shared cross-crate linkme substrate.\n- **scoped-proxies** is NOT a second proxy machine: a scoped+advised bean is ONE proxy whose innermost `TargetSource` (`ScopeTarget`) re-resolves the live instance from the ambient `Cx` per call, feeding the unchanged advice chain. Pure scope-mismatch (no advice) is served by the deferral family's honest handle (`Lookup`/`LazyRef`), owned upstream by injection-mechanics — the substrate provides only the transparent `ScopeTarget` lane.\n\nEvery declarative cross-cutting feature (tx, cache, validation, retry, @Async, exception-translation) is a `Role::Infrastructure` chain entry over this one substrate (dogfooding). The whole subsystem is fixed by ADR-08 (proxy-substrate); Phase 3 here is the concrete, implementable realization that resolves the remaining open sub-questions on the toolkit primitives.

## Shared machinery (how this rests on the toolkit)

The subsystem mints almost nothing new; it is a thin assembly over the fixed toolkit primitives, and the unification is that all four features share them rather than each owning a mechanism:\n\n1. ONE handle currency (ownership-model). The proxy IS an `ErasedBean = Arc<dyn Any+Send+Sync>`. The generated `OrderProxy: Order` upcasts to `Arc<dyn Order>` (trait upcasting, stable 1.86) at zero extra alloc, so DI injects it as the SAME type the target satisfies — autowiring (injection-mechanics `Selector`) is untouched, Spring's invisibility preserved. `TargetSource::get` yields an `ErasedBean`; `Published::Shared` for singleton/scoped targets, the `Published::Owned` lane for advised prototypes. The `Send+Sync+'static` bound rides the handle in the one leaf-core location — the proxy inherits it for free.\n\n2. ONE creation/decoration seam, installed by the TWO-SITE rule (C3 — bean-lifecycle, ownership-model). Proxy installation is NOT a single unconditional seam: (i) the origin-agnostic `after_init` instance-decoration hook (`ErasedBean -> ErasedBean` swap inside `Engine::create`'s publish pipeline — not a new SPI) proxies ONLY `Multiplicity::Once` singletons and advised-prototype `Published::Owned` targets — the chain fires exactly once; (ii) a `PerContextKey` (request/session) scoped-AND-advised bean is proxied at the INJECTION/resolver seam over a `ScopeTarget` (chain built once per holder, memoized), and `InstanceStore` holds the BARE `ErasedBean` (after_init is SKIPPED for PerContextKey+advised) so the live target is re-resolved per call and never double-wrapped. ProxyPlan is frozen at `seal()` and is a declared input of `App<Wired>::validate()` (consumed at after_init R4 AND at the injection/validate seam); injecting an advised bean by its concrete `self_type` is a distinct `ResolveError::AdvisedConcreteInjection` (not ScopeMismatch). The auto-proxy creator is dogfooded `Role::Infrastructure` post-processing, not a kernel surface. `make_interceptor(&Container)` resolves the aspect bean through the same `Provider`/`resolve` machinery, so advice injects collaborators without the advisor itself being a bean.\n\n3. ONE ambient context (async-context-model). Scoped `ScopeTarget` re-resolution and tx/locale/tracing binding read the poll-re-installing `Cx` bundle — never a ThreadLocal, never an `AopContext.currentProxy()` analogue. The tx resource is an `Isolate` `Cx` key. `ScopeTarget` reaches the per-`Cx`-key `InstanceStore` via `StoreSource::AmbientStore(ScopeKind)` exactly as bean-lifecycle defined; the `Weak<ContainerCore>` back-ref avoids an Arc cycle.\n\n4. ONE async-across-dyn standard (async-context-model 5b). `BoxFuture` at every `Interceptor`/`TargetSource`/`make_interceptor` seam (AFIT/RPITIT not dyn-compatible). The generated newtype's per-method typed pack/unpack glue keeps the common path typed (a move, not a per-arg box); only the dynamic-source `ErasedProxy` pays per-call erased-args boxing.\n\n5. ONE comparator (injection-mechanics `leaf_core::order`). The advice chain sorts by `cmp_chain(&ChainKey,&ChainKey)` at CORRECTNESS grade — RoleTier first (Infrastructure outermost), then the unchanged pure `cmp_order(OrderKey)` (lower-value-wins, `OrderSource::Interface` beats `Annotation`), then the `ContractId` stable tiebreak. `cmp_chain` (the ONE correctness-grade composite over `ChainKey { tier, order, id }`, leaf_core::order — SEAMS C6) is what ProxyPlan::freeze's documented `(RoleTier, cmp_order, ContractId)` sort resolves to; `AdvisorRef::chain_key()` builds the ChainKey from the existing `{id,order,role}` fields. No second comparator; plain `cmp_order` stays the comparator for the non-tiered sites (PriorityRank, collection-injection ordering). Collection-injection's preference-grade use of `cmp_order` does not weaken this determinism.\n\n6. ONE freeze boundary + plan (compile-runtime-split, registry-substrate). The `ProxyPlan` is computed once at `seal()` over the frozen dense `BeanId(u32)` registry + the const `AdvisorDescriptor` stream; decoration is an O(1) `BeanId`-keyed lookup. Advisor identity/tiebreak is the stable `ContractId`, never a serialized `TypeId`. Pointcut matching is a pure predicate over the macro-emitted `MethodTable`/`AnnotationMetadata` DATA (no reflection).\n\n7. ONE discovery + anti-DCE defense (codegen-boundary, cross-crate-discovery). Advisor descriptors collect into a linkme `ADVISORS` slice via absolute `::leaf_core` paths; the mandatory expected-vs-found self-check covers the advisor stream AND the creator/execution-facility presence (a DCE'd tx-advisor crate is a loud `AntiDceError::SourceVanished`, never a silent no-op — and for @Async never a silent sync run).\n\n8. ONE error/diagnostic model (error-model). `AdviceError` and `ResolveError` flow into `LeafError`/`Diagnostic`; mandatory-trait violations are Tier-0 compile diagnostics (`#[diagnostic::on_unimplemented]`); the assembled chain order is an inspectable §2.9 diagnostic (`Container::advice_chain_of` + `leaf doctor`).\n\n9. ONE cancel-safety discipline (async-context-model 5d). Around-advice releases resources via sync RAII `Drop` on cancel; genuinely-async finalization is a `TeardownLedger` entry — no async Drop.

## Features

### `aop-proxying`
MECHANISM = the generated transparent newtype (Phase-1 R1), with the runtime erased table (Phase-1 R2) demoted to a dynamic-source-only fallback and tower-per-edge (Phase-1 R3) rejected. For a bean `OrderSvc` exposing advisable trait `dyn Order`, `leaf-macros` (thin) emits `pub struct OrderProxy { chain: Arc<AdviceChain>, source: Arc<dyn TargetSource> }` and `impl Order for OrderProxy` with one delegating method per trait method: `async fn place(&self,a:A)->R { self.chain.invoke(Call::new::<{Order::place}>(&*self.source, ErasedArgs::pack((a,)))).await.unpack() }`. It upcasts to `Arc<dyn Order>` (1.86). CALL-ROUTING (leaf-core, hand-writable): `trait Interceptor: Send+Sync { fn intercept<'a>(&'a self, call: Call<'a>, next: Next<'a>) -> BoxFuture<'a, Result<ErasedRet, AdviceError>>; }`; `Next::proceed(&mut self) -> BoxFuture<..>` is REPLAYABLE (FnMut -> fresh future, for retry) and SKIPPABLE (an interceptor may return `Ok(ret)` without calling proceed, for cache short-circuit); `AdviceChain { ordered: Box<[Arc<dyn Interceptor>]> }` walks then finally calls `TargetSource::get(cx).await` + the real method. TARGETSOURCE: `trait TargetSource: Send+Sync { fn get<'a>(&'a self,cx:&'a ResolveCtx<'a>)->BoxFuture<'a,Result<ErasedBean,ResolveError>>; }` with `FixedTarget(ErasedBean)` (singleton) and `ScopeTarget{scope:ScopeKind,key:BeanKey,weak:Weak<ContainerCore>}` (scoped). DYNAMIC FALLBACK: `ErasedProxy{chain,source,methods:&'static MethodTable}` where `MethodEntry{key:MethodKey, invoke: fn(&ErasedBean,ErasedArgs,&ResolveCtx)->BoxFuture<Result<ErasedRet,AdviceError>>}` drives downcast thunks — used ONLY where no compile-time type exists (origin-agnostic per ownership-model). Resolved open sub-questions: (1) mechanism = generated newtype; (2) service trait MANDATORY for advised beans (no subclass-a-concrete-class), macro hard-errors with a `#[diagnostic::on_unimplemented]` note steering to 'declare a service trait or move the concern to a collaborator'; concrete-only beans stay injectable, un-advisable; (3) transparent `dyn Svc`, NOT a visible handle (deferral family owns the honest handle); (4) ErasedArgs/ErasedRet ABI -> Phase 4 measure (default: generated per-method typed tuple pack/unpack, a move not a per-arg box; small-vec-of-Box fallback only for `ErasedProxy`); (5) async standard = BoxFuture-per-hop (inherited from 5b), monomorphized single-advisor fast-path DEFERRED to Phase 4 measurement; (6) cancel-safety = sync RAII Drop + ledger; (7) early-proxy is runtime-only + trivial (proxy holds a TargetSource not a live target, materializable over a not-yet-built target with no .await).

**Resolved open questions:** Proxy mechanism = compile-time generated transparent newtype (R1); R2 erased table demoted to dynamic-only ErasedProxy fallback; R3 tower-per-edge rejected (breaks wrap-once-globally, mismatches Service shape); Service trait is MANDATORY for any advised bean; macro hard-errors via diagnostic::on_unimplemented; concrete-only beans injectable but un-advisable (named divergence from Spring's CGLIB); Proxy injects as the SAME dyn Svc type (transparent, autowiring untouched); honest visible handles belong to the deferral family, not here; Erased-vs-typed args bridged by generated per-method typed pack/unpack glue on the common path (move, not box); erased downcast thunks only in ErasedProxy; exact ABI is a Phase-4 measure; Early-proxy (getEarlyBeanReference) sub-tier is runtime-only (AOT disallows cycles) and trivial: the proxy wraps a TargetSource, so it materializes over a not-yet-built target with no .await; the frozen ProxyPlan makes early-ref==final automatic; Async-across-dyn standard = BoxFuture per interceptor hop (async-context-model 5b); monomorphized single-advisor fast-path deferred to Phase 4, measured before built; Around-advice cancel-safety = sync RAII Drop on the cancel path (rollback/release/pop); async finalization rides the TeardownLedger; no async Drop; Wrap-decision lives once at after_init (per-bean global, every external caller advised), never per-edge

### `auto-proxy-creator`
ONE per-context creator computing a frozen `ProxyPlan` at the registry `seal()` boundary (Phase-1 R2 precomputed plan adopted; R1 per-decoration runtime match rejected as O(beans×advisors) + needing a mutable early_proxied dedup set; R3 advisors-as-beans rejected for the bootstrap chicken-and-egg). `struct ProxyPlan { by_bean: HashMap<BeanId, Box<[AdvisorRef]>> }`, `struct AdvisorRef { id: ContractId, order: i32, role: Role }`; `ProxyPlan::freeze(advisors:&[AdvisorDescriptor], registry:&Registry, policy:&CreatorPolicy) -> Result<ProxyPlan, AssemblyError>` runs match ONCE over each bean's declared `MethodTable`/`AnnotationMetadata` (a pure predicate, never a reflective walk), filters by the active `CreatorPolicy`, computes the UNION, and sorts by `(RoleTier, cmp_order(OrderKey), ContractId)`; `ProxyPlan::advisors_for(BeanId)->&[AdvisorRef]` is the O(1) decoration lookup. INSTALL: at the ownership-model `after_init` hook (origin-agnostic `ErasedBean->ErasedBean` swap), the creator does the O(1) plan lookup; un-advised beans pass through UNWRAPPED (eligibility gate — bare-engine parity: no creator installed = advice inert). `make_interceptor(&Container)->BoxFuture<Result<Arc<dyn Interceptor>,ResolveError>>` resolves each advisor's interceptor at refresh (advice injects collaborators). `getEarlyBeanReference` consults the SAME frozen plan (early-ref==final by construction; NO dedup set). CreatorPolicy = capability lattice `{admit_infrastructure, admit_application}`, infra-floor UPGRADED in place to admit application aspects (the @EnableAspectJAutoProxy analogue) — assembled at the binary/app-root (`#[leaf::main]`) where the full enabled-feature set is visible, NEVER racing post-processor beans (link order is unspecified). Conditional advisors gated by the descriptor's `condition: Option<&'static dyn Condition>` evaluated at `seal()` (when container state is known — no plan-per-profile explosion). Resolved open sub-questions: matching happens at seal() (build-on-frozen-registry), advisors are a flat descriptor stream NOT beans, wrap-decision is per-bean-global at after_init, the capability lattice is binary-root-assembled and deterministic, advisor activation reconciles link-time collection vs runtime conditions via the descriptor condition at seal(), ordering is correctness-grade and inspectable, method-metadata vocabulary is the macro-emitted stable MethodKey name-hashes + attribute-presence flags.

**Resolved open questions:** Matching happens ONCE at the seal() freeze boundary into a frozen BeanId-keyed ProxyPlan (R2); per-decoration runtime match (R1) and advisors-as-beans (R3) rejected; Advisors are a flat const AdvisorDescriptor stream collected via the linkme cross-crate substrate, NOT first-class beans (avoids the bootstrap chicken-and-egg); make_interceptor resolves the aspect bean at refresh so advice still injects collaborators; Wrap-decision is per-bean-global at the after_init decoration hook (every external caller advised), never per-edge; CreatorPolicy capability lattice (infra-floor upgraded to admit application aspects) is assembled deterministically at the binary/app-root, never as racing processor beans; eliminates Spring's which-creator-registered-first fragility; Link-time collection vs runtime conditions reconciled by the descriptor's condition predicate evaluated at seal() when container state is known (no plan-per-profile explosion); Early-proxy sub-tier consults the SAME frozen plan so early-ref==final-proxy automatically; the earlyProxyReferences dedup set is dissolved; Advisor ordering is CORRECTNESS-grade (RoleTier first, then cmp_order, then ContractId) and surfaced as an inspectable diagnostic via Container::advice_chain_of and leaf doctor; Matchable method-metadata vocabulary = macro-emitted stable MethodKey name-hashes + per-method attribute-presence flags in AnnotationMetadata (pure data, no reflection)

### `aspect-model`
`#[aspect]` is a `#[component]` (a normal bean, so advice injects collaborators) whose `#[advice(around|before|after|after_returning|after_throwing, pointcut=…, order=…)]` methods lower (thin macro, `::leaf_core` paths) to flat const `AdvisorDescriptor { id: ContractId, order: i32, role: Role, pointcut: &'static dyn Pointcut, condition: Option<&'static dyn Condition>, make_interceptor: fn(&Container)->BoxFuture<Result<Arc<dyn Interceptor>,ResolveError>> }` rows. before/after/after_returning/after_throwing COLLAPSE to one around-shaped `Interceptor` at lowering (the chain sees ONE kind — Phase-1 R1 typed aspects adopted; R3 aspect-as-bare-interceptor-bean rejected to avoid the bootstrap entanglement, R2 expression strings kept as an OPTIONAL front-end). POINTCUTS = typed combinators PRIMARY: `trait Pointcut: Send+Sync { fn matches(&self, jp:&JoinPointMeta)->bool; }` over `JoinPointMeta{bean_type:TypeId, method:MethodKey, markers:&AnnotationMetadata, arg_types:&[TypeId], ret_type:TypeId}`; `within::<dyn Order>() & annotated::<Transactional>() & returns::<Result<_,_>>()` (composable `&`/`|`/`!`). OPTIONAL expression front-end `#[pointcut("within(::svc::*) && @annotation(Transactional)")]` lowers to the SAME predicate at macro-expansion (zero runtime parser, Spring-feel) and lives in a normal crate, never the macro's hidden logic. leaf's own tx/cache/validation/@Async/retry/exception-translation author against the EXACT same vocabulary emitting `Role::Infrastructure` descriptors (maximal dogfooding §2.11) — so the user-facing and framework surfaces are ONE. Typed around-advice reaches concrete args via the generated pack/unpack glue (cache-key-from-a-param, `proceed(newArgs)` for validation). Resolved open sub-questions: pointcuts are typed combinators primary + optional expression front-end; an aspect lowers to flat const descriptors (not a bean) with make_interceptor as the bean bridge; const-constructibility constrains the combinator algebra to const-fn-buildable `&'static dyn Pointcut` (the expression front-end and any non-const combinator codegen a const wrapper at macro time); typed arg access is uniform via generated glue (richer on monomorphized path); ordering is the one correctness-grade model; self-invocation is uniform-honest across ALL advisors; framework features share the exact vocabulary.

**Resolved open questions:** Pointcuts authored as typed predicate combinators (R1) PRIMARY, with an OPTIONAL compile-time-parsed expression front-end (R2) that lowers to the same predicate; left-agnostic Aspect.matcher (R3) rejected; An aspect lowers to a FLAT const AdvisorDescriptor (R1/R2), not a first-class bean (R3); make_interceptor is the bridge that resolves the aspect bean so advice still injects collaborators without the bootstrap carve-out; Pointcut predicate must be &'static dyn Pointcut on the linkme path => const-constructible: the combinator algebra is const-fn-buildable; the expression front-end and non-const combinators emit a const wrapper at macro-expansion time; Typed arg/return access through the dyn-erased chain is via the generated per-method pack/unpack glue (typed on the common path); the ErasedProxy path is the boxed fallback; Advice ordering is the SINGLE correctness-grade model shared with auto-proxy-creator (RoleTier, then cmp_order, then ContractId), surfaced as an inspectable diagnostic; Self-invocation boundary is UNIFORM and honest across ALL advisors (this.method() bypasses the proxy); diagnostic note + sanctioned workarounds (refactor to collaborator; inject SelfRef/Ref<dyn Self>); no AopContext.currentProxy() ThreadLocal analogue; leaf's own tx/cache/validation/@Async/retry/exception-translation author against the EXACT user-facing vocabulary emitting Role::Infrastructure descriptors (one authoring surface, dogfooded); before/after/after_returning/after_throwing collapse to one around-shaped Interceptor at lowering; the chain sees ONE advice kind

### `scoped-proxies`
NOT a second proxy machine — UNIFIED into the substrate via the `TargetSource` seam (Phase-1 R3 LiveTargetProvider core adopted; R1 fuse-into-routing rejected because it distorts proceed()'s fixed-inner contract; R2 honest-handle-only rejected as a SOLE realization because it loses transparency for the bare-trait-required and scoped-AND-advised cases). A request-scoped bean `Cart` exposing `dyn Cart` injected into a singleton holder is ONE transparent proxy whose innermost `TargetSource` is `ScopeTarget { scope: ScopeKind, key: BeanKey, weak: Weak<ContainerCore> }`. On EACH call, `ScopeTarget::get(cx)` reads the active scope from the poll-re-installing ambient `Cx` (NEVER a ThreadLocal — the defining clash), reaches the per-Cx-key `InstanceStore` via `StoreSource::AmbientStore(ScopeKind)`, and calls `InstanceStore::get_or_init(BeanId, provider, cx)` (bean-lifecycle), yielding the LIVE `ErasedBean`; the UNCHANGED advice chain then wraps it. proceed()'s contract stays un-distorted (scope re-resolution is the innermost target supplier, NOT a fake advisor). The proxy is itself a safely-published singleton-shaped NON-OWNING re-resolver (holds resolver+key+Weak, never the live target), so prototype hand-off-and-forget and request/session teardown stay bean-lifecycle's `TeardownLedger` concern; the Arc strong-count governs the real free, so in-flight advised work past a scope boundary stays sound. PROXYMODE re-expressed for Rust (no class-vs-interface synthesis distinction): `TARGET_CLASS`+`INTERFACES` collapse to ONE transparent mode (requires a service trait) | `NO` = inject raw + fail-fast on mismatch. The HONEST visible handle (`ScopedRef`/`Lookup`/`LazyRef`) is the deferral family's surface owned upstream by injection-mechanics, for the purely-deferred (un-advised) scope-mismatch case — leaf offers BOTH: transparent ScopeTarget for the scoped-AND-advised / bare-trait-required case, honest handle for the visible-by-choice case. ADVISED PROTOTYPE: the `Published::Owned`-aware TargetSource lane constructs-and-owns a fresh instance per resolution (the prototype owned-move invariant is never silently violated). Resolved open sub-questions: transparent default for advised/scoped + honest handle for pure deferral (both offered, split along the advice seam); the ambient is the one Cx bundle; per-call yield is an ErasedBean (Ref<T> for shared, owned T for prototype) re-resolved per call so the holder cannot stash a stale target through the transparent face; scoped-proxies inherits the substrate's one mechanism (no independent mechanism); proxyMode collapses to transparent/NO; mismatch detected at App<Wired>::validate (an advised bean injected by concrete type, or a raw shorter-scoped Arc, is a startup error); teardown-on-cancellation is bean-lifecycle's ledger concern (the proxy owns nothing to orphan).

**Resolved open questions:** Transparent stand-in (ScopeTarget) is the default for scoped-AND-advised + bare-trait-required cases; the honest deferral handle (Lookup/LazyRef, owned by injection-mechanics) serves the purely-deferred case — both offered, split along the advice seam, resolving the transparent-vs-honest keystone area-wide; The per-call active-scope ambient is the ONE poll-re-installing Cx bundle (async-context-model), never a ThreadLocal/RequestContextHolder analogue; ScopeTarget reads StoreSource::AmbientStore(ScopeKind); Per-call yield is an ErasedBean re-resolved through InstanceStore::get_or_init on EVERY call (Ref<T> for shared, owned T for prototype via the Owned lane) — the transparent face cannot stash a stale target; the holder is scope-agnostic; Scoped-proxies inherits the substrate's single proxy mechanism via the TargetSource seam — it is NOT settled independently and adds NO second proxy machine (R3 decomposition, with the LiveTargetProvider role played by ScopeTarget); ScopedProxyMode re-expressed: TARGET_CLASS+INTERFACES collapse to one transparent mode (service-trait required) | NO = inject raw + fail-fast; no Rust class-vs-interface distinction; The singleton-holds-shorter mismatch is detected at App<Wired>::validate() (a raw shorter-scoped Arc, or an advised bean injected by concrete type, is a startup error) — caught before any request runs with the richest diagnostic; The proxy is a non-owning re-resolver (holds resolver+key+Weak<ContainerCore>, never the live target), so teardown-on-cancellation stays bean-lifecycle's TeardownLedger concern; cancellation drops cleanly with nothing to orphan; advised prototypes use the Published::Owned TargetSource lane

## Cross-feature interactions
- **aop-proxying ↔ scoped-proxies** — UNIFICATION via the TargetSource seam (the duplicated-wrap the Phase-1 coherence note flagged). A scoped+advised bean is ONE object: ScopeTarget (scoped-proxies' innermost target supplier) yields the live target, then aop-proxying's unchanged AdviceChain wraps it. proceed()'s fixed-inner contract is un-distorted; scope re-resolution is NOT a fake advisor. The transparent-vs-honest keystone is resolved area-wide: advice = transparent dyn Svc; pure deferral = honest handle (owned by injection-mechanics).
- **auto-proxy-creator ↔ aspect-model** — The creator CONSUMES the AdvisorDescriptor stream that aspect-model PRODUCES. Both share one ordering model (correctness-grade RoleTier+cmp_order+ContractId) and one decision point (the seal() frozen ProxyPlan). aspect-model decides 'what could contribute' (flat descriptors + condition); the creator decides 'what does wire up' (match at seal() against the frozen registry, filtered by CreatorPolicy). The bootstrap chicken-and-egg is avoided because advisors are descriptors, not beans, with make_interceptor as the late bean bridge.
- **auto-proxy-creator ↔ aop-proxying** — The creator's after_init O(1) plan lookup drives aop-proxying's wrap-and-swap (ErasedBean->ErasedBean). getEarlyBeanReference consults the SAME frozen plan so early-ref==final-proxy by construction (the earlyProxyReferences dedup set is dissolved); the early stand-in is trivially materializable because the proxy holds a TargetSource, not a live target.
- **scoped-proxies ↔ aop-proxying (advised prototype)** — The substrate adds a Published::Owned-aware lane to the TargetSource contract: an advised prototype gets a per-resolution proxy whose TargetSource constructs-and-owns a fresh instance, so the prototype owned-move/no-refcount invariant is never silently violated (a fixed cross-ADR seam).
- **all four ↔ declarative cross-cutting features (tx/cache/validation/retry/@Async/exception-translation)** — Each is ONE Role::Infrastructure Interceptor authored via aspect-model's exact vocabulary, ordered by the creator, wrapping a TargetSource. They depend jointly on Next being REPLAYABLE (retry calls proceed N times) and SKIPPABLE (cache short-circuits), typed-arg access via the generated glue (validation/cache-key), sync-RAII cancel-safety across proceed (tx rollback, permit release), and the deterministic inspectable chain order (cache-outside-vs-inside-tx). @Async stays a chain advisor (uniform self-invocation; body-rewrite rejected as default, hybrid deferred to Phase 4).
