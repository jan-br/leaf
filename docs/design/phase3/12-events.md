# Event Bus & Multicaster  `[events]`

_The in-process observer pub/sub on the context: @EventListener/ApplicationListener registration, the single replaceable multicaster with synchronous-ordered (cmp_order) dispatch + opt-in async + return-value-as-new-event chaining, and the built-in lifecycle/availability events on the same bus._

This subsystem is leaf's in-process observer bus: the publish/subscribe model (@EventListener / ApplicationListener), the single replaceable multicaster that owns dispatch policy (sync-ordered by cmp_order, opt-in async, return-value-as-new-event chaining, error isolation), and the built-in lifecycle/availability facts that ride the very same bus. The Phase-3 re-unification collapses the three Phase-1 forks onto fixed toolkit primitives instead of minting bus-local machinery: (1) listeners are ordinary beans whose @EventListener methods are emitted as const ListenerDescriptor rows into a linkme EVENT_LISTENERS slice — the exact COMPONENTS pipeline — so discovery, force-link, and the anti-DCE expected-vs-found self-check are SHARED, not re-invented (resolving every feature's "listener never fires"/silent-DCE openQuestion at one gate). (2) The multicaster is realized as event-multicaster-R3 (the composable pipeline) over a NEW `DispatchInterceptor` sibling around-advice trait (C5 — it fans OUT over N listeners, structurally distinct from aop's one-`Call` `Interceptor`, so NOT a verbatim reuse) that SHARES with the proxy substrate only `cmp_chain` + the RoleTier grade — async, error-isolation, context-propagation, metrics are `DispatchInterceptor` chain entries ordered by cmp_chain (RoleTier-first; SEAMS C6), with the inline-await loop as the innermost CoreDispatch sink; it is the magic-named applicationEventMulticaster bean resolved by BeanKey::ByName with an install-default branch (the SAME pattern container-shape names for messageSource), and it is a Role::Infrastructure bean Context::refresh() auto-detects. (3) Dispatch sync-default = inline await on the caller's task, so the ambient Cx (locale/trace/tx) is naturally in scope; opt-in async hands the listener future to the upstream Spawner with the CxDecorator carrying only Inherit keys. (4) Lifecycle facts are typed events on this bus; availability is the upstream watch<RunState> reactive cell (the ONE reactive-state primitive async-context-model already mints) projected onto the bus — killing the "two notification substrates" duplication the area coherence notes flag. Every error is a LeafError node; the whole-graph validation pass aggregates malformed-listener/dangling-condition failures into the one AssemblyReport.

## Shared machinery (how this rests on the toolkit)

The subsystem rests entirely on the fixed toolkit; nothing here is a new primitive.

1. ONE creation/identity seam (ownership-model, registry-substrate, container-shape). A listener is NOT a bespoke registration — its hosting bean is an ordinary Provider/Descriptor/BeanId, and each @EventListener method is one const ListenerDescriptor row carrying the host bean's ContractId + an adapter fn-pointer. The multicaster and the projector are Role::Infrastructure Providers auto-detected by Context::refresh() (the inert→live switch, ADR-05). The magic-named applicationEventMulticaster is BeanKey::ByName("applicationEventMulticaster") resolution with an install-default branch — the identical pattern container-shape fixed for messageSource. The EventPublisher and AvailabilityHandle are ALWAYS-PRESENT structural Context fields injectable through the ResolvableDependency terminal layer (injection-mechanics §F) — no Aware setter phase.

2. ONE codegen + anti-DCE pipeline (codegen-boundary, cross-crate-discovery). The thin #[event_listener]/#[transactional_event_listener] macros emit data only: one const ListenerDescriptor into a new sibling linkme slice EVENT_LISTENERS (joining COMPONENTS/CONDITIONS/AUTO_CONFIGS/FAILURE_ANALYZERS) via ::leaf_core paths, plus a hand-writable adapter that downcasts the host Arc and the event &dyn Any. The binary-crate ExpectedManifest + freeze-time self_check(&EXPECTED) turns a DCE-dropped listener-only crate into a loud AntiDceError::SourceVanished naming the crate — the SAME defense, so "my listener never fires" becomes structural-loud, not Spring's silent gotcha.

3. ONE ordering + interception spine (injection-mechanics, proxy-substrate). Listener order WITHIN an event type uses leaf_core::order::cmp_order over OrderKey{value,source} — listeners are not RoleTier-graded, so the pure cmp_order (the SAME comparator as autowiring tie-break) is correct there; no bus-local comparator. The DISPATCH-CONCERN pipeline is a NEW `DispatchInterceptor` chain (C5 — a sibling around-advice trait that fans OUT over N listeners, structurally distinct from aop's one-Call `Interceptor`): AsyncDispatch / ErrorIsolation / ContextPropagation / Metrics are `DispatchInterceptor` entries sorted by `cmp_chain` (RoleTier-first composite over the unchanged cmp_order; SEAMS C6), the CoreDispatch inline-await loop the innermost step. It SHARES with AOP only cmp_chain + the RoleTier grade — the unification event-multicaster-R3 bet on, now made real with one comparator across two interceptor shapes (NOT one shared Interceptor type).

4. ONE async + ambient spine (async-context-model). publish is async-first; sync default = sequential inline awaits on the caller's task so the ambient Cx (LocaleKey/trace Inherit, tx Isolate) is preserved with zero extra machinery. Opt-in async hands each listener BoxFuture to the upstream Spawner; the CxDecorator captures only Inherit keys (tx is Isolate, never rides to a child task). Listener futures box at the dyn boundary (BoxFuture, forced by AFIT-not-dyn-safe), monomorphized typed fast path at known-concrete publish sites is the Phase-3 escape. No async Drop: the multicaster quiesce (drain in-flight async dispatches) and the projector-task shutdown are awaited TeardownLedger entries drained by Context::shutdown() after stop_all().

5. ONE reactive-state primitive (async-context-model 5f). Availability liveness/readiness is the upstream watch<RunState>-shaped cell (watch_run_state()), NOT a second mechanism — resolving the area-coherence "two overlapping notification substrates" risk: state transitions live in the watch cell, and a single projector re-publishes each transition as an AvailabilityChanged event so the bus stays stateless.

6. ONE error/diagnostic spine (error-model). DispatchError, MalformedListener, DanglingCondition are LeafError nodes (ErrorKind::ConstructionFailed for a listener build failure, Condition for a bad condition expr, the open Integration{ContractId} arm for a detached-async-listener panic routed via AsyncUncaughtFailureHandler). A malformed listener / dangling qualifier aggregates into the App<Wired> AssemblyReport at the same validation pass as NoSuchBean; the milestone "ready-listener failed" routes a LeafError into refresh's cancel path. FailureAnalyzers ship for the missing-multicaster and never-fires-listener cases.

## Features

### `events`
The publish/subscribe MODEL + listener authoring/discovery, realized as the events-R3 hybrid bus collapsed onto toolkit primitives.

BUS SHAPE (resolves events openQ 'typed channels vs erased vs hybrid'): two cooperating stores frozen at refresh, behind one EventPublisher inherent surface on Context. (a) FAST: `typed: HashMap<TypeId, TypedChannel>` where TypedChannel is a `Box<[ListenerEntry]>` (frozen, lock-free reads on the hot path — no interior mutability, satisfying no-global-lock). The channel key is `TypeId::of::<E>()`, so EntityCreated<Person> vs <Order> are DISTINCT keys FOR FREE (monomorphization solves Spring's ResolvableTypeProvider problem — the generic-targeting openQ dissolves). (b) FALLBACK: `erased: Box<[ErasedListenerEntry]>` for cross-type/marker-trait/supertrait listeners, each carrying a `supports(TypeId)->bool` const predicate; consulted only when an event type has matching erased entries. A publish merges typed + matching erased entries into one cmp_order-sorted sequence ONCE at refresh (a frozen per-TypeId merged plan), so the hot publish path never sorts.

LISTENER AUTHORING: two models, ONE compiled shape. (1) `trait ApplicationListener<E>: Bean { fn on(&self, e:&E) -> BoxFuture<'_, Result<ListenerOutcome, LeafError>>; }` — the classic interface. (2) the preferred POJO `#[event_listener]` on any managed-bean method; the macro reads the single param type E and emits a ListenerDescriptor + adapter. Both compile to a ListenerEntry { adapter: ErasedAdapterFn, order: OrderKey, condition: Option<CondExprFn>, chains: bool, host: ContractId }. ListenerOutcome::{None, Emit(SmallVec<[ErasedEvent;1]>)} carries return-value chaining; the macro hard-ERRORS if an async/spawn-dispatched listener declares a chaining return (the 'disabled on async' caveat becomes a compile error, resolving that openQ).

DISCOVERY (resolves the T1 silent-DCE openQ): each #[event_listener] emits `#[distributed_slice(EVENT_LISTENERS)] static __L: ListenerDescriptor` via ::leaf_core paths. At App<Define> RegistryBuilder lifts EVENT_LISTENERS alongside COMPONENTS; at refresh the listeners are bound to channels by resolving each host bean's ContractId→BeanId. The anti-DCE ExpectedManifest self-check covers EVENT_LISTENERS, so a DCE'd listener crate is a loud AntiDceError::SourceVanished. A publish to a type with zero listeners is a cheap silent no-op (Spring parity) BUT a host bean that is @Lazy and has @EventListener methods is force-included into the wiring set (diverging from Spring to fix the gotcha — resolves the lazy openQ); the self-check reports any host whose listeners were dropped.

CONDITION FILTERING: the condition slot is the expr-i18n CondExprFn = `fn(&EvalCx)->Result<bool,ExprError>` (SYNC-PURE by contract), evaluated AT DISPATCH on the hot path reading the ambient Cx via EvalCx — for a deferred (transactional) listener it is re-evaluated AT FIRE-TIME (resolves the area-coherence condition-timing inconsistency: publish-time for inline, fire-time for deferred, because #root context differs). No boxing — sits directly on the dispatch path.

HIERARCHY DOUBLE-DELIVERY: publish in a child walks self channels then Context::parent() channels via the ADR-05 parent: Option<Ref<Context>> link; the merged ListenerSeq carries originating ContainerId (a ContractId) so a listener registered in multiple contexts disambiguates.

PAYLOAD events: a plain domain type IS the event (channel key = its TypeId); the optional Event<P>{source,timestamp,origin} envelope (the missing event-envelope owner the coherence notes flag) is offered for listeners that need the source handle.

**Resolved open questions:** Bus shape (typed/erased/hybrid): HYBRID — frozen typed channels keyed by TypeId (fast, lock-free, generics-free) + a small erased fallback for cross-type/marker listeners; merged+cmp_order-sorted once at refresh.; Generic event targeting: SOLVED by monomorphization (distinct TypeIds); no ResolvableTypeProvider analogue needed for static instantiations; the erased fallback covers the rare runtime-resolved key.; Listener discovery substrate: the linkme EVENT_LISTENERS slice = the COMPONENTS pipeline; force-link + ExpectedManifest self-check is the SHARED anti-DCE defense.; Return-value chaining on async listeners: a COMPILE-TIME error from the macro (chaining return + async/spawn dispatch is rejected), not a runtime check.; Lazy-bean listener gotcha: leaf DIVERGES from Spring — a @Lazy host with @EventListener methods is force-included into wiring; the self-check reports any dropped host.; Condition-evaluation timing: at-publish (hot path) for inline listeners, re-evaluated at-fire-time for deferred/transactional listeners; the CondExprFn is the shared expr-i18n sync-pure closure.

### `event-multicaster`
The single replaceable dispatch SEAM, realized as event-multicaster-R3 (the composable pipeline) by REUSING the proxy-substrate Interceptor chain — NOT a bus-local mechanism (resolves the seam-shape openQ in favor of AOP-primitive sharing now that proxy-substrate fixed that primitive).

SEAM SHAPE: `trait Multicaster: Send+Sync { fn dispatch<'a>(&'a self, ev:&'a ErasedEvent, seq: ListenerSeq<'a>) -> BoxFuture<'a, DispatchOutcome>; }` is the public seam, but its DEFAULT impl is `PipelineMulticaster { core: CoreDispatch, chain: Box<[Arc<dyn DispatchInterceptor>]> }` where `DispatchInterceptor` is a NEW sibling around-advice trait (C5) — NOT reused from aop verbatim, because the advice `leaf_core::aop::Interceptor` wraps ONE `Call → ErasedRet` while the multicaster must fan OUT over N listeners: `trait DispatchInterceptor: Send+Sync { fn intercept<'a>(&'a self, ev: &'a ErasedEvent, seq: ListenerSeq<'a>, next: ListenerNext<'a>) -> BoxFuture<'a, DispatchOutcome>; }` with a forward-only `ListenerNext` over the sequence. It SHARES with aop ONLY `leaf_core::order::cmp_chain` (the RoleTier-first composite over the unchanged cmp_order) + the RoleTier grade, NOT the Interceptor type. Dispatch concerns are `DispatchInterceptor` chain entries, ordered by cmp_chain over the pinned consts (SEAMS C6: `ASYNC_DISPATCH_ORDER=100 < ERROR_ISOLATION_ORDER=200 < CONTEXT_PROP_ORDER=300 < METRICS_ORDER=400`): AsyncDispatchInterceptor (hands inner next to the upstream Spawner), ErrorIsolationInterceptor (catch + route to a typed sink), ContextPropagationInterceptor (CxDecorator capture/restore around an async hop), Metrics/Tracing. The innermost step is CoreDispatch — the sequential inline-await loop over the cmp_chain-sorted ListenerSeq, collecting ListenerOutcome::Emit for re-publish. 'Replace the whole multicaster' is the degenerate case of swapping CoreDispatch.

MAGIC NAME + INSTALL-DEFAULT (resolves the container-core 'where does it live' openQ): the multicaster is BeanKey::ByName("applicationEventMulticaster"), Role::Infrastructure, auto-detected by Context::refresh(); if absent, Context installs a DefaultMulticaster Provider in the fixed-order refresh template (the SAME ByName+install-default branch container-shape names for messageSource). Its mandatory presence is enforced by the refresh self-check (a publish to a missing multicaster would be catastrophic — same severity as the missing ExecutionFacility HARD-FAIL).

SYNC-DEFAULT (resolves the 'caller's context without thread-locals' openQ): with NO AsyncDispatchInterceptor present, CoreDispatch awaits each listener sequentially on the caller's TASK, so the ambient Cx is in scope (Inherit keys via the poll-re-installing Scoped<F>; tx is Isolate). publish().await does not resolve until every sync listener returns. Cancelling the publisher future cancels in-flight listeners at await points (cooperative, structured scope).

OPT-IN ASYNC: an AsyncDispatchInterceptor hands each listener BoxFuture to the upstream Spawner; the engine does NOT await (publish returns after scheduling), chaining is forced off (CoreDispatch collects no outcomes), errors route to the AsyncUncaughtFailureHandler (entering LeafError via the open Integration{ContractId} arm — resolves the 'no caller to throw to' clash), ordering is abandoned. spawn_blocking-needing listeners use the upstream BlockingOffload.

ERROR POLICY (resolves the area-coherence default-isolation fork): the DEFAULT is abort-on-first-and-propagate (Spring-faithful, fail-loud) for application events; an injected ErrorIsolationInterceptor flips to isolate-each-route-to-sink. The NON-uniform milestone contract lifecycle-events needs is expressed as a per-dispatch policy: the ready/started milestone dispatch observes the DispatchOutcome and routes any listener LeafError into refresh's cancel path (→ Failed), while ordinary facts isolate. This is one error model with a per-dispatch override, not three incompatible defaults.

CHAINING ERASURE: a chained event of a different type re-enters publish as ErasedEvent (Box<dyn Any+Send> + TypeId); the re-publish pays one downcast at the next channel boundary — the partial-purity wrinkle, localized to the chaining path only.

TRANSACTIONAL-EVENTS SEAM: phase-deferral is a listener KIND in the resolved ListenerSeq (a PhaseAwareAdapter that, at dispatch, registers a tx-synchronization on the Isolate tx CxKey and defers firing), NOT a dispatch interceptor — so the engine stays shape-agnostic and the deferral decision lives in the resolved descriptor before the ListenerSeq (resolves the area-coherence 'where does deferral live' ambiguity).

TEARDOWN: no async Drop — the multicaster's quiesce (stop accepting publishes, drain in-flight async dispatches up to a deadline) is an awaited TeardownLedger entry drained by Context::shutdown() after stop_all(), before the listener-hosting beans are destroyed.

**Resolved open questions:** Seam shape (R1 dyn-bean / R2 policy-data / R3 pipeline): R3 — a `DispatchInterceptor` chain (a NEW sibling around-advice trait that fans OUT over N listeners, C5) over a CoreDispatch sink, SHARING with AOP only `cmp_chain` + the RoleTier grade (NOT the one-`Call` aop `Interceptor` type — the bet pays off as one comparator across two interceptor shapes).; Default error-isolation: abort-on-first-and-propagate for app events; injected ErrorIsolationInterceptor for isolate; milestone dispatch uses a per-dispatch observe-and-fail-startup override — one model, per-dispatch policy.; 'Sync on caller's context' without thread-locals: inline sequential awaits on the caller's task preserve the ambient Cx (Inherit keys via Scoped<F>); async hop carries Cx only via the CxDecorator.; Where the multicaster lives: a Role::Infrastructure bean on the Context, BeanKey::ByName('applicationEventMulticaster') with an install-default branch, auto-detected at refresh, mandatory-presence self-checked.; Per-listener @Async vs global async: the listener arrives pre-wrapped (an async-dispatching listener KIND from the events layer); the engine just awaits whatever future it returns — global async is the AsyncDispatchInterceptor.; transactional-events extension seam: a listener KIND in the ListenerSeq (PhaseAwareAdapter registering a tx-synchronization on the Isolate tx CxKey), not an interceptor — the deferral decision lives in the resolved descriptor.; Cross-crate guarantee the engine exists: force-link + the refresh expected-vs-found self-check (mandatory-presence, HARD-FAIL), reusing the SAME substrate as the ExecutionFacility self-check.

### `lifecycle-events`
The built-in lifecycle FACTS + availability STATE on the same bus, realized as the R1/R3 hybrid: facts are typed events; availability is the upstream watch reactive cell with a single projector (resolves the state-vs-fact fork AND the reactive-primitive-duplication area note).

FACTS (HALF 1): plain typed structs published through the EventPublisher at precise milestones by their owning lifecycle phases — `Refreshed{container:ContractId, generation:u32}`, `Started`, `Stopped`, `Closed{reason:CloseReason}`, `StartupFailed{phase, error:Arc<LeafError>}`. The container is the sole legitimate publisher (the EventPublisher is the structural Context field; a fact type is publishable only by container-internal code). The generation counter makes the multi-fire ContextRefreshed reality explicit and guardable (resolves that openQ). refresh-lifecycle fires Refreshed at end-of-refresh; runtime-lifecycle fires Started/Stopped around phased start/stop; context-close fires Closed; the CANCEL path fires StartupFailed and DELIBERATELY NOT Refreshed/Closed (the cancel-vs-close distinction is structural in the publishing code, not emergent).

AVAILABILITY (HALF 2): NOT a fact — the upstream reactive cell. `Availability { liveness: WatchSender<LivenessState>, readiness: WatchSender<ReadinessState> }` backed by the SAME watch<RunState>-shaped primitive async-context-model mints via watch_run_state() (resolves the reactive-state-primitive openQ by reuse, not a new primitive). Reads are cold-path point reads (`availability.readiness()` -> current value, lock-free borrow — a k8s probe handler, legitimate cold-path per charter 2.4); subscription is reactive (woken on change, never polled). Writes: ANY component holds an injectable AvailabilityHandle (resolvable like EventPublisher) and flips state at runtime (a broken DB pool flips readiness to REFUSING_TRAFFIC); transitions are validated enum state machines, not free strings. Both liveness/readiness are orthogonal axes (readiness flips while Running) — kept as two separate watch cells, NOT collapsed into one composite phase machine (resolves the 'three axes one machine?' openQ: R3's single-machine is rejected as state-explosion / Phase-2-keystone overreach; phase facts + two watch cells with explicitly-defined interactions — Closed forces REFUSING, Failed forces BROKEN).

PROJECTION: ONE long-lived container-scoped projector task subscribes to the watch cells and re-publishes each transition as an `AvailabilityChanged{kind, old, new, source}` event on the bus — the dogfooding intent ('availability rides the same bus') WITHOUT making the bus the source of truth. The projector is shut down explicitly via a TeardownLedger entry (no async Drop). The cell is updated strictly BEFORE the event is published (the contract that defuses R1's state/event ordering-skew con).

EARLY FACTS: pre-context facts (Starting/EnvironmentPrepared analogues) that fire before the multicaster exists go through an explicit EarlyEventBuffer the refresh template drains at the phase that installs the multicaster (the registerListeners-flush analogue). This is the ONE early-buffering primitive shared with startup-instrumentation (resolves the area-coherence 'three early-emission mechanisms' note — they share one buffer drained at one refresh phase).

MILESTONE FAILURE: the started/ready milestone dispatch uses the multicaster's per-dispatch observe-and-fail-startup policy: the bring-up task observes the DispatchOutcome and routes a listener LeafError into refresh's cancel path (→ StartupFailed, suppressing Refreshed/Closed), while ordinary fact-listener failures isolate (the NON-uniform contract, now expressible).

CLOSE ORDERING: on clean close, readiness flips to REFUSING_TRAFFIC FIRST (quiesce), THEN Closed is published, THEN runtime-lifecycle stop_all(), THEN the TeardownLedger drains — an explicit container-driven ordered sequence (no async Drop).

**Resolved open questions:** Availability state model (separate cell vs retained-last events): a SEPARATE reactive cell (R1) reusing the upstream watch<RunState> primitive + one projector to the bus — retained-last-on-bus (R2) rejected (pushes state into a stateless engine, amplifies multi-fire footgun).; Reactive-state primitive: REUSE the async-context-model watch<RunState>-shaped cell (watch_run_state()); no new primitive — this also resolves the area-coherence 'two notification substrates' duplication (state lives in the cell, the bus stays stateless, one projector bridges).; 'Ready-listener failure fails startup': a per-dispatch observe-and-fail-startup policy on the multicaster; the bring-up task observes the DispatchOutcome and routes a LeafError into refresh's cancel path.; Where publisher + AvailabilityHandle live: ALWAYS-PRESENT structural fields on the Context (container-shape guarantee), injectable via the ResolvableDependency terminal layer; a bare Engine emits no facts.; Pre-context early facts: an explicit EarlyEventBuffer drained at the refresh phase that installs the multicaster — the ONE early-buffering primitive shared with startup-instrumentation.; Multi-fire ContextRefreshed: modeled with a generation:u32 on the Refreshed fact; one-time-vs-every-time is a documented consumption choice; no retained replay to amplify the footgun.; Three axes (phase/liveness/readiness) one machine?: NO — phase facts as typed events + two orthogonal watch cells with explicitly-defined interactions (Closed→REFUSING, Failed→BROKEN); the single-composite-machine R3 is rejected as overreach.; Lazy-bean lifecycle listeners: same policy as events — force-included into wiring (diverging from Spring); a never-updating probe becomes a loud self-check diagnostic, not a silent miss.

## Public API sketch

// ═══ leaf-core::events — publish/subscribe model on the always-present Context ═══

// Listener outcome + chaining (return-value-as-new-event)
pub enum ListenerOutcome { None, Emit(smallvec::SmallVec<[ErasedEvent; 1]>) }
pub struct ErasedEvent { pub value: Box<dyn std::any::Any + Send>, pub type_id: std::any::TypeId,
                         pub source: Option<ErasedBean>, pub origin: ContainerId }   // ContainerId = ContractId
pub type ContainerId = ContractId;

// Classic interface listener (a Bean)
pub trait ApplicationListener<E: 'static>: Bean {
    fn on<'a>(&'a self, e: &'a E) -> BoxFuture<'a, Result<ListenerOutcome, LeafError>>;
}

// The const row the #[event_listener] macro emits into the linkme slice (joins COMPONENTS/CONDITIONS/...)
pub struct ListenerDescriptor {
    pub host: ContractId,                                  // the bean hosting the method
    pub event_type: std::any::TypeId,                      // None-keyed channel = erased/supertrait listener
    pub supports: Option<fn(std::any::TypeId) -> bool>,    // Some => erased fallback entry
    pub order: leaf_core::order::OrderKey,                 // sorted by the ONE cmp_order
    pub condition: Option<leaf_core::expr::CondExprFn>,    // SYNC-PURE; reads ambient Cx via EvalCx
    pub chains: bool,                                      // false for async/spawn-dispatched (macro hard-errors otherwise)
    pub adapter: fn(ErasedBean, &dyn std::any::Any) -> BoxFuture<'_, Result<ListenerOutcome, LeafError>>,
}
#[linkme::distributed_slice] pub static EVENT_LISTENERS: [ListenerDescriptor] = [..];
// #[event_listener(condition="event.priority > 5")] fn on(&self, e: &OrderPlaced) -> ()  -> one const row + adapter
// #[event_listener] fn on(&self, e: &OrderShipped) -> NotificationSent  -> chaining (re-published)

// EventPublisher — ALWAYS-PRESENT structural Context field, injectable via ResolvableDependency

impl EventPublisher {

}
impl Context { pub fn events(&self) -> &EventPublisher; }                  // always Some (structural guarantee)

// ═══ leaf-core::events::dispatch — the single replaceable multicaster SEAM ═══

pub trait Multicaster: Send + Sync {                                      // BeanKey::ByName("applicationEventMulticaster")
    fn dispatch<'a>(&'a self, ev: &'a ErasedEvent, seq: ListenerSeq<'a>) -> BoxFuture<'a, DispatchOutcome>;
}
// DEFAULT impl = the proxy-substrate Interceptor chain over CoreDispatch (NOT a bus-local mechanism)

// Dispatch concerns are DispatchInterceptor entries sorted by cmp_chain (RoleTier-first; shares ONLY cmp_chain + RoleTier grade with aop):

## Cross-feature interactions
- **registry-core (upstream)** — Listeners are ordinary beans: each host has a Descriptor/BeanId/Provider; the multicaster, projector, and default multicaster are Role::Infrastructure Providers auto-detected by Context::refresh(). The magic-named applicationEventMulticaster is BeanKey::ByName resolution with an install-default branch — the SAME pattern registry-core/container-shape fix for messageSource. ListenerDescriptor rows carry the host ContractId and bind to channels at refresh by ContractId→BeanId.
- **execution-context (upstream)** — Opt-in async dispatch hands listener BoxFutures to the upstream Spawner; the CxDecorator carries only Inherit Cx keys (tx is Isolate, never auto-propagated to a spawned listener). Sync-default inline awaits preserve the ambient Cx via Scoped<F>. Availability reuses the watch<RunState> reactive primitive. Detached async-listener panics route to the AsyncUncaughtFailureHandler. Multicaster quiesce + projector shutdown are awaited TeardownLedger entries (no async Drop). A blocking listener uses BlockingOffload.
- **expr-i18n-resources (upstream)** — The @EventListener condition slot IS the expr-i18n CondExprFn = fn(&EvalCx)->Result<bool,ExprError>, SYNC-PURE, evaluated on the dispatch hot path reading the ambient Cx (incl. the current LocaleKey). No boxing. A malformed condition expression is a LeafError aggregated into the AssemblyReport at the same validation pass.
- **proxy-substrate / AOP (sibling)** — The multicaster pipeline defines a NEW sibling around-advice trait `DispatchInterceptor` (C5: it fans OUT over N listeners via `ListenerNext`, structurally DISTINCT from the aop `Interceptor` that wraps ONE `Call → ErasedRet`) and SHARES with AOP only `leaf_core::order::cmp_chain` (the RoleTier-first composite over the unchanged cmp_order) + the RoleTier grade — NOT the Interceptor type. async/error-isolation/context-propagation/metrics are `DispatchInterceptor` chain entries ordered by cmp_chain over the pinned ASYNC_DISPATCH/ERROR_ISOLATION/CONTEXT_PROP/METRICS consts, the CoreDispatch inline loop the innermost step. This is the comparator-and-ordering-grade-sharing bet, realized (not type-level Interceptor reuse). An advised listener-hosting bean is dispatched through its service-trait view exactly as any advised bean.
- **transactional-events (sibling, built on this seam)** — Phase-deferral is a listener KIND (PhaseAwareAdapter) in the resolved ListenerSeq that registers a tx-synchronization on the Isolate tx CxKey and defers firing to the chosen tx phase — proving the bus is extensible without engine edits. The condition is re-evaluated at fire-time (different #root context). The no-active-tx skip is made LOUD/diagnosable (a LeafError-backed warning), not Spring's silent no-op.
- **refresh-lifecycle / runtime-lifecycle / context-close (sibling triggers)** — These phases are the sole publishers of lifecycle facts at their milestones; refresh fires Refreshed (+ drains the EarlyEventBuffer at the multicaster-install phase), runtime fires Started/Stopped, close fires Closed after flipping readiness=REFUSING first. The cancel path fires StartupFailed and suppresses Refreshed/Closed. The ready/started milestone uses the observe-and-fail-startup per-dispatch policy.
- **error-model (upstream)** — DispatchError / malformed-listener / dangling-condition / detached-panic are LeafError nodes (ConstructionFailed, Condition, the open Integration{ContractId} arm); the App<Wired> validation pass aggregates them into the one AssemblyReport. FailureAnalyzers ship for missing-multicaster and never-fires-listener; the silent-DCE listener-drop surfaces via the shared anti-DCE expected-vs-found self-check.

## Remaining risks (→ Phase 4)
- Merged dispatch-order stability: collecting typed-channel entries AND matching erased-fallback entries into one cmp_order-sorted ListenerSeq is computed once at refresh, but a subtle mis-merge (e.g. cross-store tie-break) is a silent wrong-order hazard — the cmp_order ContractId tiebreak must be applied uniformly across both stores; needs a property test in Phase 4.
- Per-dispatch error-policy matrix: AbortAndPropagate (default) vs IsolateEach vs ObserveAndFailStartup (milestone) must present a coherent, documented matrix; a misconfiguration (async + AbortAndPropagate) yields surprising semantics (async errors hit the sink, there is no caller) — needs an explicit refresh-time validation that async dispatch implies a sink and chaining=off.
- Interceptor ordering correctness: error-isolation INSIDE vs OUTSIDE async-dispatch changes whether async listener failures are catchable; the contract must pin which interceptor is innermost/outermost (CoreDispatch innermost is fixed; AsyncDispatch must sit outside ErrorIsolation for sync-listener isolation to hold) — a sharp tool needing a fixed RoleTier assignment in Phase 4.
- Projector coherence: the cell-updated-strictly-before-event contract must hold under concurrent writers flipping availability from multiple tasks; a missed or doubled projection means a fact silently never fires or double-fires — needs the projector to be the single serialization point (one task draining the watch change stream).
- Generic-event erased fallback: the rare runtime-resolved generic key (EntityCreated<T> where T is only known dynamically) still has no clean key beyond the const supports() predicate; whether any real use needs a richer-than-TypeId token is deferred until demand is measured.
- Chaining-path erasure cost: a chained event of a different type pays one downcast per channel boundary on re-publish; acceptable but unmeasured on deep chains — measure-before-optimize, with a monomorphized typed re-publish escape if the publisher's type is statically known.
- Early-buffer/startup-instrumentation unification: asserting ONE EarlyEventBuffer drained at one refresh phase is correct only if startup-instrumentation's pre-recorder steps genuinely share that buffer; this must be co-confirmed when startup-instrumentation is re-unified (not yet designed).
