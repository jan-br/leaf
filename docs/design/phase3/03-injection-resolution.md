# Injection & Candidate Resolution Mechanics  `[injection-resolution]`

_The single fixed-order traced layer-fold Selector spine resolving every injection point: type/generic/qualifier/primary-fallback/name layers, collection/map shapes, injection styles, deferral handles (Lookup/LazyRef/Inject), and deferral-only cycle breaking._

This subsystem is the single resolution spine that answers every injection point in leaf: given a const `InjectionPoint` row, it produces a `CandidateSet`, runs the fixed-order traced layer fold to arbitrate 0/1/many, and hands the winner (or the whole set, for collections) to `Engine::create` for the async construct/publish step. It owns NO storage and NO construction — `registry-substrate` produces the keyed candidate set, `ownership-model`'s `Provider::provide -> BoxFuture<Published>` builds the winner, `Engine`/`Context` drive it. The unification is total: nine Phase-1 features (autowiring-resolution, candidate-resolver, primary-fallback, collection-injection, injection-styles, deferral-primitives, circular-references, lookup-method [extra-1], self-injection [extra-2]) collapse onto exactly four pieces of shared machinery — (1) the `Selector::resolve_one` fold over a `const LAYERS` array with a SINGLE terminal fail-fast rule; (2) the candidate-resolver's keyed-SET filtering (type+generics+marker-qualifier by index, residual string dimensions by a three-valued Match fold) feeding that fold but never selecting; (3) the renamed deferral family `Lookup<T>/LazyRef<T>/Inject<T>` over ONE `resolve(Strictness,Cardinality)` driver, which is also the ONLY cycle break, the lookup-method mechanism, the self-injection handle (`SelfRef`), and the optional/lazy/multiple surface; (4) the one `leaf_core::order::cmp_order` comparator. Construction is single-phase (factory params ARE the injection points), so constructor cycles are logically fatal at the `App<Wired>::validate()` pass and the only break is an explicit deferral edge. Selection is sync/pure over the frozen snapshot (lock-free); only winner construction is async. The whole thing fails fast all-or-nothing into one `AssemblyReport`, with lazy/scoped points failing at first acquisition with the identical `ResolveError`.

## Shared machinery (how this rests on the toolkit)

Every feature rests on the FIXED Phase-2 toolkit primitives, sharing rather than minting:

1. ONE selection primitive — `Selector::resolve_one(ip, set) -> (Resolved, Option<Trace>)`, a generic fold over `const LAYERS: &[Layer]` (`[GenericNarrow, QualifierNarrow, PrimaryPromote, NameMatch, QualifierNameMatch, PriorityRank, DefaultCandidate, ResolvableDependency]`). Each layer is a pure `fn(&InjectionPoint, &[&Cand]) -> Verdict{Unique|Narrowed|Abstain}`; the driver short-circuits on Unique/narrow-to-one and applies the len-0/1/>1 fail-fast rule in EXACTLY ONE place. autowiring-resolution IS the fold; primary-fallback IS its `PrimaryPromote` layer (two-pass internally) with fallback-demote folded into the same layer; collection-injection BYPASSES the fold (collect-all + order).

2. ONE candidate SET, keyed for FILTERING only — `registry-substrate`'s `Registry::candidates(ty) -> &[BeanId]` plus the name/alias overlay, wrapped as `CandidateSet`/`Cand` read-views over the frozen `MergedDefinition`. candidate-resolver keys the set by `(produced TypeId, [generic-arg TypeIds], [qualifier-marker TypeIds])` via index lookup (R3 keying for SPEED), then runs a three-valued `Match{Yes,No,Abstain}` fold for residual stringly dimensions. It produces the set; it NEVER selects. The dense `BeanId(u32)` space makes the SmallVec working set and trace allocation-light on the >1 path only.

3. ONE deferral driver — `resolve(Strictness, Cardinality)` over the renamed consumer family `Lookup<T>/LazyRef<T>/Inject<T>` (container-shape). `Single` -> `resolve_one`; `Multiple` -> collect-all + `cmp_order` (selection bypassed); `Strictness{Strict,AbsenceTolerant,FullyTolerant}` maps `Resolved::{None,Ambiguous}` onto `Result`/`Option`. This ONE driver is the home for: optional arity (`Option<T>` -> AbsenceTolerant), lazy (`LazyRef` = `OnceCell`-cached for singleton targets), jakarta `Provider<T>` parity (`Inject<T>`), lookup-method injection (`Lookup<T>`, no separate mechanism), the cycle break (a Deferral-kind point removes the construction edge), and self-injection (`SelfRef`). The handle holds `ownership-model`'s `Resolve = Arc<dyn Fn(..) -> BoxFuture<Published>>` closure over a `Weak<ContainerCore>` (no Arc cycle).

4. ONE comparator — `leaf_core::order::cmp_order(&OrderKey, &OrderKey)` over `OrderKey{value, source: OrderSource{Interface,Annotation,Implicit}}`. Consumed by `PriorityRank`, collection/map ordering, `Lookup::ordered_stream`, runtime-lifecycle phases, and the advisor chain. No site defines its own.

5. ONE creation seam — `Provider::provide -> BoxFuture<Result<Published,ResolveError>>` driven by `Engine::create`. The macro emits a single-phase factory whose typed params are the injection points; setter/field/`@Lookup`/`Self`/`@Value` are KINDS on the per-bean `InjectionPlan` const, not separate surfaces. No two-phase populate hook.

6. ONE validation pass — `App<Wired>::validate()` runs `resolve_one` over every eager single-valued `InjectionPoint`, cycle-classifies the construction graph (constructor-param edges only; Deferral/Lazy edges removed), and aggregates all `NoSuchBean`/`NoUniqueBean`/`Cycle`/`ScopeMismatch`/`AdvisedConcreteInjection` into one `AssemblyReport`, unified with the anti-DCE expected-vs-found self-check. validate() is PURE-PROJECTION over the frozen snapshot EXCEPT for ONE bounded carve-out (`[C2-config-locus]`): it (a) PRE-MATERIALIZES the config-properties beans — the only bean class whose `Provider::provide` is pure-projection (`&Env`+`BindHandler`, no `ResolveCtx`) and therefore safe to build before wiring is live — running their binder-backed factory + JSR validation and storing the bound `Arc` into each slot's OnceCell, and (b) runs a THROWAWAY `@Value` dry-run over every eager bean's `PointKind::Value` rows (`resolve_into::<T>`, value DISCARDED). Both feed the SAME Tier-2 `AssemblyReport`, so a malformed `@Value` coercion (`order.max-retries=abc → u16`) and a failed config-properties JSR check (`DB_POOL_SIZE=0` vs `range(min=1)`) surface in the ONE aggregated report at validate(), NOT at refresh R5 inside `Provider::provide`. The carve-out is the SOLE place validate() touches construction; no other bean is materialized at validate.

7. ONE error path — `ResolveError` (NoSuchBean/NoUniqueBean/Cycle) -> `LeafError`/`Diagnostic`, enriched at the validation pass with the layer `Trace` + per-candidate crate provenance from the frozen `ConditionReport` + anti-DCE manifest.

## Features

### `autowiring-resolution`
`Selector::resolve_one(ip: &InjectionPoint, set: CandidateSet) -> (Resolved, Option<Trace>)` is the spine. The fold threads `SmallVec<[&Cand;4]>` through `const LAYERS` (literal Spring order), short-circuiting on the first `Verdict::Unique` or narrow-to-one; after all layers the driver applies the SOLE fail-fast rule: `len 0 -> Resolved::None`, `len 1 -> One`, `len >1 -> Ambiguous`. `Resolved::{None,Ambiguous}` is non-coercible, so 'refuse to guess' is a type. The early step-3 name/qualifier shortcut is DROPPED: leaf's candidate find is a cheap keyed index probe, not Spring's expensive findAutowireCandidates, so the fast-path-equals-slow-path invariant is not worth its silent-divergence-bug class; name_match runs only in-ladder. The `Trace` (`SmallVec<[(&'static str,usize,usize);8]>`) is recorded ONLY on the >1 path, so the warm len<=1 path stays allocation-free and branch-only. Selection is sync/pure over the frozen snapshot (lock-free); winner construction is the async Engine::create step on the caller's task.

**Resolved open questions:** Shape: R2 traced-fold chosen over R1 hand-ladder and R3 typed-key-primary; fixed const LAYERS array, decomposed for testability/diagnostics, NOT a pluggable strategy seam.; Fail-fast timing: at App<Wired>::validate() for eager single-valued points (richest context post-link); lazy/scoped/prototype at first acquisition with the identical ResolveError; typo'd marker TYPE is the only compile-tier arm.; Step-3 early name/qualifier shortcut: DROPPED (keyed probe makes it redundant); one in-ladder name_match.; Trace cost: recorded only on the >1 path; per-candidate crate provenance for cross-module NoUniqueBean legibility, off the warm path.; Greediest-satisfiable multi-arm constructor: REQUIRE one deterministic factory per bean; force multiple-ways-to-build onto explicit @Bean/FactoryBean (avoids silent-fall-through-to-smaller-arm).

### `candidate-resolver`
FILTERING only, never selection. Produces `CandidateSet` by keying `Registry::candidates(produced_type)` then narrowing by `(generic-arg TypeIds, qualifier-marker TypeIds)` via index lookup over the dense BeanId space, then folding residual stringly dimensions with `enum Match{Yes,No,Abstain}` (veto-wins / abstain-falls-through). Qualifiers are the hybrid the tension demands: a `QualifierReq{marker: Option<TypeId>, name: Option<&'static str>}` carries a typed zero-sized marker (#[qualifier] struct -> marker TypeId, must live in a crate both bean and injection point depend on) AND/OR an open string. Marker + generics narrow by the keyed SET (R3 speed, free generics-as-qualifier via distinct concrete TypeIds); bean-name-as-implicit-qualifier and @Fallback stay in the fold. NOT a pluggable dyn CandidateFilter chain (the kernel SPI is rejected upstream); filtering is concrete leaf-core code. resolvableDependencies fold into the by-type count at the terminal layer. autowire_candidate (hard by-type removal, name-reachable) and default_candidate (weak: excluded from plain autowiring, injectable when explicitly qualified, the lever that lets get_if_unique resolve) are const flags in Descriptor.meta.

**Resolved open questions:** Shape: keyed-SET filtering (R2/R3 blend) for type+generics+marker, three-valued Match fold for residual string dimensions; NOT a dyn filter chain.; Open-vs-typed qualifiers: hybrid QualifierReq (marker TypeId OR open string); typed marker for compile-safe single-marker + free generics, string path for bean-name + config-valued + name-fallback.; Custom-qualifier attribute SETS (match-ALL): ride the string/Abstain path as a captured key/value bag in AnnotationMetadata, matched at the fold; NOT combinatorial marker generics.; Dense bean-index space: the registry's frozen BeanId(u32) (WASM/dynamic epoch dropped); qualifier-filter masks materialized lazily on first resolution (measured).; @Lazy/SpEL dynamic per-candidate predicate tier: OUT of filtering (owned by the deferral family); no dynamic-predicate escape hatch in the resolver.

### `primary-fallback`
The PrimaryPromote layer of the fold (Spring's 5a / determinePrimaryCandidate), executed live during selection (R1 fused two-pass, NOT R2 pre-pass; keeps effective_primary from being a fragile per-type-key durable flag and avoids dragging cross-crate count into the registry rewrite). PASS 1: unique @Primary; two at the same min hierarchy_depth -> a PrimaryConflict folded into Resolved::Ambiguous; a local primary (min depth) beats a parent primary via Cand.hierarchy_depth: u16 (0=local). PASS 2 (only if zero primaries): partition into regulars (!is_fallback) and fallbacks; exactly one regular -> Unique (zero-annotation cross-module auto-win); >1 regulars -> Narrowed(regulars) (drop fallbacks, hand to name_match); zero regulars (all fallback) -> Abstain (fall through with the full set). is_primary/is_fallback are const flags in Descriptor.meta. @Fallback is the auto-config-override story made declarative and the single most cross-crate-coupled layer: evaluated over the FULLY-collected post-link set at the validation pass; the anti-DCE expected-count self-check is a hard PRECONDITION (a dropped regular-bean crate silently lets a library @Fallback win), surfaced as a NoSuchBean naming the missing source crate, distinct from a Cycle. STRUCTURAL non-interaction with collections: arity (Single vs Collection on the InjectionPoint) decides whether the layer runs at all.

**Resolved open questions:** Live (R1 fused) vs precomputed (R2) vs decomposed-shared-layers (R3): LIVE fused two-pass inside the single PrimaryPromote layer (R3's decomposition collapses into it since the fold is the chosen autowiring shape, but kept as one layer to match Spring's primary-first order and avoid the demote-first equivalence proof).; Where the complete set is available: at seal()/App<Wired> (all static rows present, dynamic/WASM dropped), so @Fallback is provably after full assembly.; Per-type-key scoping: natural; resolve_one is always invoked per InjectionPoint.produced (a specific dyn-Svc or concrete TypeId view), so fallback arithmetic is per-type-view by construction; no global effective_primary bool.; Anti-DCE precondition: expected-vs-found self-check runs at freeze()/validate, granular per-source-crate; NoSuchBean enriches with 'N candidates registered from crates [..]'.; hierarchy_depth: a u16 on Cand labeled by Context when merging parent-delegated finds (explicit opt-in walk); local-shadows-parent + local-primary-beats-parent via min-depth scan.; Escape hatch: at the DEFINITION level only (programmatic primary/fallback flags), never at SELECTION level.

### `collection-injection`
MULTI-VALUED injection BYPASSES selection: resolve(Strictness, Cardinality::Multiple) collects ALL qualifying candidates (candidate-resolver's set, generics-narrowed elementwise) and never runs the PrimaryPromote/NameMatch/etc. layers. The InjectionPoint captures element produced TypeId, generics, and a CollectionShape{Vec, HashSet, Array(n), Map}. Ordering uses the ONE cmp_order over each Cand's OrderKey (interface-source beats annotation-source before numerics, lower value wins, registration-order/SequenceKey as the stable final tie-break) for Vec/Set/Array; Array length-mismatch is a rich error. Map<String,T> keys on bean NAMES from the by_name overlay; Spring's resolveMultipleBeanMap NO-SORT asymmetry is RECONCILED by using insertion-ordered IndexMap (= registration order) so the Map is deterministically ordered without diverging from intent. Empty qualifying set -> empty collection, never NoSuchBean. NullBean does NOT participate in collection counts. The eager List<T> is built at wiring (resolved once, stored as a typed field); the lazy 0..N counterpart is Lookup::stream()/ordered_stream() (deferral family); both use the SAME cmp_order so eager-collection and lazy-stream ordering never diverge.

**Resolved open questions:** Shape: R1 aggregate-then-sort over the borrowed set, NOT R2 const-pre-sorted-index (ordering is a consumer concern computed at injection; the candidate set is dynamic post-link).; Map ordering asymmetry: RECONCILED via IndexMap insertion order = registration order; deterministic, no Spring-style no-sort wart.; @Order vs @Priority scope: OrderKey.source distinguishes Interface (Ordered-impl, more-specific) from Annotation (#[order]/#[priority]); #[priority] is struct-level-only modeled as Annotation that factory-method beans cannot carry; kept orthogonal to startup/dependency order.; Selection bypass: structural; Cardinality::Multiple routes around resolve_one entirely; primary/fallback/default-candidate apply only to single-valued points.; NullBean in collections: excluded from collection counts (couples to the arity/NullBean mapping).

### `injection-styles`
SINGLE-PHASE construction: every bean is built by ONE total async factory Provider::provide whose typed params ARE the injection points (R1; R2 two-phase populate and R3 type-state-builder-as-injection-surface both REJECTED; two-phase reopens a mutable half-built bean fighting ownership-model's atomic seam, the type-state builder can't ride the runtime-by-TypeId container path). The thin macro emits a per-bean InjectionPlan{ points: &'static [InjectionPoint] } const + a build that calls the user's hand-written Self::new(..) and returns Published::Shared(Arc::new(..)) (singleton/scoped) or Published::Owned(Box::new(..)) (prototype). Each InjectionPoint{produced, generics, qualifiers, name, arity: Arity{Single,Optional}, kind: PointKind{Bean,Value,Deferral,SelfRef}}. Setter = Option<T> param (arity=Optional) or #[inject(default)]; field = #[inject] field synthesized into the same build; @Lookup/Self/@Value are KINDS; NO post-construction inject(&mut self) hook. Method-level all-or-nothing optional setter: simplified to per-param Option<T> (grouped Option<(A,B)> available where genuinely needed). Bare generic #[component] hard-errors with a register_component!(Concrete) hint. Immutability is free; the value is frozen the instant it exists, so safe publication is the Arc clone itself. The type-state builder returns ONLY as an optional Phase-3 hand/test convenience layered over the flat factory, never the container-path surface.

**Resolved open questions:** Sole injection surface: factory-fn parameter set; NO post-construction typed-param method hook (rejects Spring's arbitrary @Autowired method, subsumed by ctor/field params).; Construction model: single-phase R1; two-phase populate (R2) and type-state-as-injection-surface (R3) rejected per ownership-model's atomic Provider->Published seam.; Multiple-constructor greediest-satisfiable: NOT offered; one deterministic factory per bean; multiple build-ways via explicit @Bean/FactoryBean.; Method-level all-or-nothing setter: per-param Option<T> by default; grouped Option<(A,B)> as an opt-in shape.; get_owned vs get -> Ref: macro-inferred from the injection-point declared type (prototype param -> owned move via Published::Owned; shared -> Ref<T>); explicit Engine::get_owned/get at the programmatic escape hatch.; Type-state builder: deferred to a Phase-3 hand/test convenience, not a second container injection surface.

### `deferral-primitives`
ONE engine, thin typed facades (R2). resolve(Strictness{Strict,AbsenceTolerant,FullyTolerant}, Cardinality{Single,Multiple}) is the single driver over the ownership-model Resolve closure (Arc<dyn Fn(&InjectionPoint, Arity) -> BoxFuture<Published>> capturing Weak<ContainerCore>). The public family (container-shape's renamed names, freeing 'Provider' for the construction seam): Lookup<T> (= ObjectProvider: get/get_if_available/get_if_unique/stream/ordered_stream/stream_filtered), LazyRef<T> (= @Lazy: OnceCell-cached for singleton targets, re-resolve for shorter scopes, NEVER cache for prototype targets, the fresh-per-call contract), Inject<T> (= jakarta Provider parity, strict-basic get). The three-tier strictness ladder lives in ONE place (the driver's match on Strictness) so the most-missed semantic (get_if_available tolerates absence but STILL errors on ambiguity, only get_if_unique swallows both) cannot drift. Single -> Selector::resolve_one; Multiple -> collect-all + cmp_order (selection bypassed; ordered_stream applies the comparator, stream is registration-order; stream_filtered filters candidate descriptors before any construction). Each get().await runs the target's construct on the CALLER's task (the explicit async execution-context shift; blocking factories spawn_blocking via ExecutionFacility); first-await over a background-bootstrap bean is the reactive readiness join point (waker-driven). The handle is NOT a Supplier/Deref-to-T; failure is honest at the call site. @Lazy is fully subsumed: LazyRef<T> IS the injection-point case; lazy-initialization keeps only the node-flag/global-default/exclusions.

**Resolved open questions:** @Lazy distinct vs subsumed: SUBSUMED; LazyRef<T> is 'Lookup + cache-on-first-get for singletons'.; Visible handle vs transparent proxy: visible-in-the-type handle (the docs' own preference); advice stays transparent (proxy-substrate) but the deferral family is the honest visible handle; no CGLIB-style transparent @Lazy.; Family shape: R2 one-engine + thin facades (Strictness/Cardinality as data) over R1 bespoke-types and R3 compile-split; public types Lookup/LazyRef/Inject are the surface, one driver underneath.; Container back-reference: Weak<ContainerCore> captured by the Resolve closure (no Arc cycle), the 'static-storable handle; the lifetime question is closed by ownership-model.; jakarta Provider: shipped as Inject<T> (faithful in shape; cross-ecosystem JSR-330 identity acknowledged-lost in Rust but the migration-familiar name is kept).; Strictness inference: macro-inferred from declared injection-point type (Option<T> -> AbsenceTolerant unifying with injection-styles Optional arity; Inject<T> -> Strict); precise total mapping.; stream membership: irreducibly a runtime query over the post-link collected set; ordered_stream uses the shared cmp_order.; Diagnostic cost: candidates-considered/provenance materialized only on the absent/ambiguous/error path.

### `circular-references`
Deferral-only, constructor-cycles-fatal (R3). Construction is single-phase, so there is NEVER a half-built instance to early-expose; setter/field cycle tolerance is structurally impossible and NOT offered (a real, defensible divergence from Spring's off-by-default runtime tolerance; consistent with allow-circular-references=false and AOT). The construction graph is built from constructor-param edges ONLY; PointKind::Deferral and lazy edges are REMOVED from the graph (a deferral-typed point does not trigger recursive creation during build). The graph is cycle-classified at the App<Wired>::validate() pass (R2-style plan-time analysis, not runtime re-entrancy; earliest/richest diagnostic, names the whole loop before any construction): an all-mandatory loop is a fatal ResolveError::Cycle{ path, fix_hint: 'convert injection point X from T to Lookup<T>' }. No early-exposure hook, NO middle singletonFactories cache tier, NO raw-vs-wrapped consistency check; the deferral handle resolves the FULLY-built, already-proxied-if-advised target post-build via the Weak back-ref, so a collaborator can never hold a stale/un-advised reference. ONE cycle behavior across runtime and AOT. The allow_circular policy on EnginePolicy is effectively moot (no auto-tolerance to gate). Cancelling a build leaks nothing (single awaitable, owned-move prototype, no half-populated shell).

**Resolved open questions:** Tolerance posture: NO setter/field cycle tolerance; deferral-only break (R3), forced by single-phase construction.; Cycle detection locus: plan-time graph analysis at App<Wired>::validate() (R2) over the frozen construction graph; constructor cycles fatal before any instance exists.; Early-exposure machinery: NONE; no middle cache level, no getEarlyBeanReference, no allowRawInjectionDespiteWrapping check; the deferral handle resolving the final object makes them unnecessary.; Constructor vs setter distinction: collapsed; all real construction-time cycles are fatal; the only break is removing an edge via a Deferral-kind point.; Cross-crate cycles: detected at the app-root whole-graph validation pass; broken by a single local Lookup<T> edge in the consuming crate.; async re-entrant deadlock: structurally absent for deferral-broken edges (the edge is removed from the construction call graph; the handle resolves post-refresh).

### `extra-1`
Lookup-method injection (@Lookup) FOLDS ENTIRELY into the deferral family (R1); no separate mechanism, no LookupCtx, no abstract-component admission gate. A Spring @Lookup Notifier createNotifier() becomes a Lookup<Notifier> field (a PointKind::Deferral point on the InjectionPlan); fresh-per-call is the handle's resolve semantics (a prototype target builds a fresh Published::Owned each get().await; the driver must NOT cache for prototype targets). The method-shaped DX is preserved with thin macro sugar: #[lookup] async fn create_notifier(&self) -> Notifier { self.notifier.get_owned().await }, but the canonical form is just calling the handle. Named lookup (@Lookup("x")) maps to a BeanKey::ByName resolution on the same driver. The component-scanner's 'abstract @Component admitted iff it declares @Lookup' gate is RETIRED, not re-homed: a #[component] struct with a Lookup<T> field is fully concrete and always has a construction recipe, so the abstract-admission exception is deleted (a bare generic still hard-errors). Lookup, scoped-proxy re-resolution, and ObjectProvider thus collapse onto ONE re-resolving-per-call primitive.

**Resolved open questions:** Distinct feature vs alias: NOT distinct; Lookup<T> IS the mechanism (R1); #[lookup] is pure sugar over a deferral field.; Scoped-proxy / @Lookup / ObjectProvider unification: YES, all three are the one resolve(Strictness,Cardinality) re-resolving handle viewed differently.; Freshness contract: the driver does NOT cache for prototype/non-singleton targets; LazyRef caching is singleton-only.; Component-scanner gate: RETIRED; 'has a construction recipe' replaces the @Lookup abstract-admission exception; no re-homing.; No LookupCtx: a second deferral-ish primitive is rejected (charter unification); the captured back-ref is the same Weak<ContainerCore> Resolve closure.; Container-completes-incomplete-bean capability: not needed (WASM/dynamic dropped); R3's deletion is safe.

### `extra-2`
Self-injection is the explicit SelfRef deferral handle (R-A; the implicit lowest-precedence self-candidate R-B is REJECTED; it couples resolution to the creation-driver's in-creation set, risks a strong Arc-of-Self leak, and makes the demotion invisible). A PointKind::SelfRef point is flagged by one descriptor is_self bit; the macro emits NO normal candidate edge, so the engine returns the SAME handle it publishes (the advised handle if the bean is advised; never an un-advised raw self). SelfRef<T> is a lazy erased closure (the same Resolve/Weak<ContainerCore> shape), NOT a strong Arc-of-Self, so there is no Arc-self-cycle and self never beats a real collaborator (it is not in by-type matching at all). Self is resolved post-build on first call (re-entry needs the proxy, which exists by then), consistent with single-phase construction. A constructor SelfRef (self as a mandatory ctor param) fails fast at the validation pass like any constructor cycle. This reuses the deferral family + the Weak back-ref; no self-registry, no implicit candidate, no in-creation runtime guard.

**Resolved open questions:** Implicit vs explicit: explicit SelfRef handle (R-A); no implicit self-candidate (R-B rejected for the Arc-self leak + invisible-demotion + creation-driver coupling).; Self-ness representation: one is_self descriptor flag + PointKind::SelfRef, emitting no by-type edge; travels with the bean.; Advised self: returns the advised/published handle (proxy-substrate), never a raw un-advised target; consistent with the COHERENCE seam rule that advised beans inject only through the service-trait view.; Arc cycle: avoided; SelfRef is a lazy Weak-captured closure, not a strong Arc-of-Self.; Constructor self: fatal at the validation pass (logically a cycle); resolves only post-build on the deferral path.; AdvisedSelfAware (R-C): not needed; the single SelfRef handle returning the published (advised) object subsumes the AopContext.currentProxy intent.

## Public API sketch

// leaf_core::resolve — THE selection spine (order is a fixed kernel contract, NOT pluggable)
pub enum Arity { Single, Optional }
pub enum PointKind { Bean, Value, Deferral, SelfRef }   // Deferral/SelfRef remove the construction-time edge
pub struct QualifierReq { pub marker: Option<std::any::TypeId>, pub name: Option<&'static str> }

#[derive(Clone, Copy)]
pub struct InjectionPoint {
    pub produced:   std::any::TypeId,
    pub generics:   &'static [std::any::TypeId],
    pub qualifiers: &'static [QualifierReq],
    pub name:       &'static str,            // declared param/field ident = implicit string qualifier
    pub arity:      Arity,
    pub kind:       PointKind,
    pub collection: Option<CollectionShape>, // Some => Cardinality::Multiple, bypass selection
}
pub enum CollectionShape { Vec, HashSet, Array(usize), Map }
pub struct InjectionPlan { pub points: &'static [InjectionPoint] }   // const, macro-emitted per bean

pub struct CandidateSet<'a> { /* dense BeanId-backed, &[Cand]; SmallVec only on the >1 path */ }
pub struct Cand<'a> { /* read view of frozen MergedDefinition: is_primary/is_fallback/default_candidate,
                         OrderKey, name+aliases, hierarchy_depth: u16, ErasedBean obtainer */ }

pub enum Verdict<'a> { Unique(&'a Cand<'a>), Narrowed(smallvec::SmallVec<[&'a Cand<'a>; 4]>), Abstain }
pub enum Resolved<'a> { One(&'a Cand<'a>), None, Ambiguous(smallvec::SmallVec<[&'a Cand<'a>; 4]>) }
pub struct Layer { pub name: &'static str, pub eval: for<'a> fn(&InjectionPoint, &[&'a Cand<'a>]) -> Verdict<'a> }

pub const LAYERS: &[Layer] = &[
    Layer { name: "generic_narrow",   eval: layers::generic_narrow },
    Layer { name: "qualifier_narrow", eval: layers::qualifier_narrow },
    Layer { name: "primary_promote",  eval: layers::primary_promote },  // fused two-pass; local beats parent
    Layer { name: "name_match",       eval: layers::name_match },
    Layer { name: "qualifier_name",   eval: layers::qualifier_name },
    Layer { name: "priority_rank",    eval: layers::priority_rank },     // calls cmp_order
    Layer { name: "default_candidate",eval: layers::default_candidate },
    Layer { name: "resolvable_dep",   eval: layers::resolvable_dep },    // HashMap<TypeId, ErasedBean>, lowest
];
pub struct Trace(pub smallvec::SmallVec<[(&'static str, usize, usize); 8]>); // recorded only on the >1 path

pub struct Selector;
impl Selector {
    pub fn resolve_one<'a>(ip: &InjectionPoint, set: CandidateSet<'a>) -> (Resolved<'a>, Option<Trace>);
}

// leaf_core::order — THE single comparator
#[derive(Clone, Copy)] pub struct OrderKey { pub value: i32, pub source: OrderSource }
#[derive(Clone, Copy, PartialEq)] pub enum OrderSource { Interface, Annotation, Implicit }
pub const DEFAULT_ORDER: i32 = 0;
pub fn cmp_order(a: &OrderKey, b: &OrderKey) -> std::cmp::Ordering; // lower value wins; Interface<Annotation<Implicit

// ResolveError (rich, fail-fast; built at App<Wired>::validate)
pub enum ResolveError {
    NoSuchBean   { point: InjectionPoint, of_type: std::any::TypeId, registered_from: &'static [&'static str] },
    NoUniqueBean { point: InjectionPoint, candidates: smallvec::SmallVec<[BeanKey; 4]>, trace: Trace },
    Cycle        { path: smallvec::SmallVec<[BeanKey; 8]>, fix_hint: &'static str },
}

// consumer deferral family (container-shape) over the ONE driver
pub enum Strictness { Strict, AbsenceTolerant, FullyTolerant }
pub enum Cardinality { Single, Multiple }
// resolve(Strictness, Cardinality): Single -> Selector::resolve_one; Multiple -> collect-all + cmp_order.
impl<T: Bean + ?Sized> Lookup<T> {
    pub async fn get(&self) -> Result<Ref<T>, ResolveError>;                       // Strict
    pub async fn get_if_available(&self) -> Result<Option<Ref<T>>, ResolveError>;  // absence None, ambiguity Err
    pub async fn get_if_unique(&self) -> Option<Ref<T>>;                           // both tolerated
    pub fn stream(&self) -> impl futures::Stream<Item = Result<Ref<T>, ResolveError>>;          // reg order
    pub fn ordered_stream(&self) -> impl futures::Stream<Item = Result<Ref<T>, ResolveError>>;  // cmp_order
    pub fn stream_filtered(&self, pred: impl Fn(&Descriptor) -> bool) -> impl futures::Stream<Item = Result<Ref<T>, ResolveError>>;
    pub async fn get_owned(&self) -> Result<T, ResolveError> where T: Sized;        // prototype targets, fresh
}

- **container-shape / Engine+Context (upstream)** — The renamed consumer family Lookup/LazyRef/Inject and the one resolve(Strictness,Cardinality) driver are container-shape's; this spine binds to exactly those names. ## Remaining risks (→ Phase 4)
- The per-resolution Trace gating threshold and the exact per-candidate crate-provenance payload are measure-before-build: provenance must make cross-module NoUniqueBean/NoSuchBean legible without warm-path cost — confirm the >1-path-only gating actually keeps the prototype/scoped re-resolution path allocation-free under realistic candidate counts.
- Custom-qualifier attribute-SET (match-ALL key/value, possibly config-valued) representation on the string/Abstain path is fixed in direction (a captured key/value bag in AnnotationMetadata, matched in the fold) but the precise encoding and how it composes with collection/Map injection over a supertype is deferred-precision — verify it doesn't quietly reintroduce a per-candidate dynamic-predicate need.
- resolvableDependencies exact set + whether ignoreDependencyInterface's 'do not autowire this Aware interface' half survives now that Aware facts arrive as factory-fn parameters (it collapses to nothing at the engine level), and whether a synthetic resolvable def with default_candidate=false participates correctly in the candidate count — confirm the terminal ResolvableDependency layer's count-folding keeps ambiguity arithmetic correct and that an infra-handle type ALSO registered as a real bean shadows predictably (loud-or-documented, not silent).
- @DependsOn precise semantics: which BeanKey a forced-ordering edge references, conditioned-out-target = error vs skip, and confirming the edge truly rides the same construction-graph topo + cmp_order machinery rather than a separate forced-init ledger.
- The COHERENCE-flagged advised-concrete-injection seam (a bean that HAS a service trait, is advised, yet is injected somewhere by its concrete type) must be enforced in the Selector: a Concrete TypeMatch against an advised bean is a startup-validation error, not a silent raw handoff — verify the Selector + proxy eligibility gate jointly reject it.
- hierarchy_depth labeling contract: the parent-delegated find must correctly label each Cand's depth (0=local) for local-beats-parent and local-shadows-parent; a mislabel silently flips which primary wins — needs a precise contract with Context's hierarchy walk.
- The deliberate divergence from Spring's setter/field cycle tolerance (deferral-only break) is a known parity gap; the actionable 'convert X: T to Lookup<T>' diagnostic must be reliably generated for deep/multi-edge cycles so the DX cost is genuinely mitigated.
- NullBean typed-boundary mapping at the arity model: get<T> maps NULL_BEAN to Ok(None) for an Option point vs a hard error for a mandatory Ref<T> point, and NULL_BEAN is excluded from collection counts — confirm these three states (NullBean / NoSuchBean / condition-not-met) stay crisply distinct in diagnostics.
