# Registry & Container Core  `[registry-core]`

_The frozen dense-BeanId registry, the ErasedBean/Ref<T>/Published handle currency, BeanDefinition metamodel, name overlay, and the Engine/Context shape that everything HAS-A._

This subsystem is the load-bearing foundation every other leaf subsystem HAS-A. It consolidates seven Phase-1 features onto the Phase-2 toolkit with one unifying move: the registry is built ONCE, coherent-by-construction, at a single `seal()`/`freeze()` boundary into an immutable `FrozenContainer`, and from then on it is a lock-free read snapshot. Every Phase-1 "two keying schemes must stay coherent" / "stale-flag merged cache" / "global singleton lock" bug class dissolves because there is no transactional mid-life mutation: descriptors append during cold `App<Define|Resolve>`, then a typestate-consuming `freeze()` materializes the dense `BeanId(u32)` slot space, both indices, the slot-indexed `singletons: Box<[OnceCell<ErasedBean>]>`, and runs the `ContractId` collision guard.

The handle currency is fixed: `ErasedBean = Arc<dyn Any+Send+Sync>` sugared as `Ref<T>`, with the two-arm `Published{Shared,Owned}` carrying the only ownership divergence (prototype = owned move). Construction is origin-agnostic through one `Provider::provide → BoxFuture<Published>` seam driven by exactly one concrete `Engine::create`. The container shape is two named types: a bare inert `Engine` (DefaultListableBeanFactory analogue) HAS-A'd by one `Context` façade (ApplicationContext analogue) that auto-detects `Role::Infrastructure` providers at `refresh()`.

The seven features map cleanly: container-core IS the Engine/Context shape; bean-registry IS the frozen `Registry` + `Engine` internals; bean-definition IS the const `Descriptor` row (the metamodel, with parent-merge resolved at freeze, not as a runtime stale cache); bean-naming IS the macro-derived name + the `IndexMap<BeanName,BeanId>` overlay + alias map + loud collision guard; container-hierarchy IS a per-`Context` `Option<Ref<Context>>` parent link with a `BeanKey`-walk delegation policy; factory-bean IS one `FactoryBeanProvider` impl (no second slot); extra-4 (NullBean) IS the canonical `NULL_BEAN` ErasedBean occupying a real slot, mapped to typed absence at `get<T>`.

## Shared machinery (how this rests on the toolkit)

Every feature rests on the SAME small set of toolkit primitives, and the unification is that they share machinery rather than each minting their own:

1. ONE dense join key `BeanId(u32)`. The `rows: Box<[Descriptor]>`, `by_type`, `by_name`, `aliases`, `by_contract`, `providers: Box<[Arc<dyn Provider>]>`, and `singletons: Box<[OnceCell<ErasedBean>]>` ALL index/point at `BeanId`. bean-registry's stores, bean-naming's overlay, factory-bean's two type-index entries (factory type + product type), container-hierarchy's local stores, and NullBean's occupied slot are all expressed as `BeanId` rows. There is no second keying scheme to keep coherent — the freeze builds all indices together in one pass.

2. ONE `seal()`/`freeze()` boundary owned by `RegistryBuilder::freeze() -> FrozenContainer`. This is the markBeanAsCreated analogue, the merged-definition computation point, the name-collision/alias-cycle check point, the `ContractId` collision guard, and the anti-DCE expected-vs-found self-check point — all collapsed into one cold synchronous transition. The typestate (`App<Resolve>` holds the mutable builder; `App<Wired>` holds the frozen container) makes a post-freeze edit unrepresentable.

3. ONE handle currency: `ErasedBean`/`Ref<T>`/`Published{Shared,Owned}`. The singleton store value, the FactoryBean product, the NullBean sentinel, a parent-supplied cross-level bean, and a test double are all the identical `ErasedBean`. `Send+Sync+'static` rides the handle in one leaf-core location.

4. ONE creation primitive `Provider::provide → BoxFuture<Result<Published,ResolveError>>` and ONE concrete driver `Engine::create`. Native beans, `FactoryBeanProvider`, the NullBean provider, and infrastructure beans are all just `Provider`s the Engine cannot tell apart (origin-blind by construction — the `Origin` field is diagnostic-only and never read on a resolution path).

5. ONE stable cross-build identity `ContractId(u64)` (const hash, collision-guarded at freeze) for everything durable: auto-config exclusion, name-collision provenance, hierarchy cross-registry identity. TypeId is the in-process fast key only, never serialized.

6. ONE `Descriptor` const row shape (the bean-definition metamodel) the thin macro emits per bean via `::leaf_core` paths: `contract`, `self_type`, `provides[]` (TypeRow upcasts), `declared_name`/`aliases` (bean-naming), `scope` (ScopeDef triple), `role` (Role), `meta` (AnnotationMetadata), `parent` (Option<ContractId> for template merge). All seven features read from or contribute to this one row.

7. ONE error/diagnostic path: `ResolveError`/`AssemblyError` → `LeafError`/`Diagnostic`, with the whole-graph `App<Wired>::validate()` aggregating NoSuchBean/NoUniqueBean/scope-mismatch/AdvisedConcreteInjection/cycle AND the `[C2-config-locus]` config-properties bind+JSR-validation + the throwaway @Value coercion dry-run into one `AssemblyReport` at Tier-2 (validate-time, before refresh) — so malformed config values never escape to the refresh R5 cancel-cascade.

## Features

### `container-core`
ADOPT R1 (two named types) exactly as fixed by ADR-05. `pub struct Engine { registry: FrozenContainer, policy: EnginePolicy }` is the bare inert DI engine — no `dyn Engine` trait, no pluggable-strategy kernel. It owns the frozen `Registry`, the slot-indexed `singletons` (the per-bean creation guard IS the per-slot `OnceCell`, no global lock), and `EnginePolicy { allow_override: bool, allow_circular: bool, strict_locking: bool }`. Public surface: `async fn get<T: Bean + ?Sized>(&self) -> Result<Ref<T>, ResolveError>` (singleton/scoped), `async fn get_owned<T: 'static>(&self) -> Result<T, ResolveError>` (prototype), `async fn get_erased(&self, key: BeanKey) -> Result<ErasedBean, ResolveError>` (dynamic/&-deref lane), `fn contains(&self, key: &BeanKey) -> bool`, plus the escape-hatch builder. The ONE concrete creation driver is `async fn create(&self, id: BeanId, cx: &ResolveCtx<'_>) -> Result<Published, ResolveError>` = `guard(slot) { publish(providers[id].provide(cx).await?) }` (single-phase construct, no two-phase populate/&mut self hook — per the settled construction constraint). `pub struct Context { engine: Engine, events: EventPublisher, messages: MessageSource, resources: ResourceLoader, env: Env, parent: Option<Ref<Context>> }` HAS-A's exactly one Engine; context services are ALWAYS-PRESENT fields (structural guarantee), never `Option<capability>`. `async fn refresh(&mut self) -> Result<(), RefreshError>` runs the FIXED-ORDER concrete template defined authoritatively by 13-container-lifecycle.md / SEAMS seam #1 (this prose DEFERS to that template, it does not re-order it). The ordering is validate-BEFORE-refresh: `seal_environment → route_conditions → run_autoconfig → seal() → App<Wired>::validate()` (the aggregated AssemblyReport pass — which under `[C2-config-locus]` pre-materializes the pure-projection config-properties beans and dry-runs every eager @Value coercion) all run BEFORE `refresh()`. `refresh()` itself is the R0..R8 template: R0 anti-DCE ROW-COUNT reconcile (NOT a pre-validate step — validate already ran) → R1 BFPP no-op assert → R2 auto-detect `role==Role::Infrastructure` (cmp_chain, RoleTier-first) → R3 init context services → R4 freeze ProxyPlan as the after_init table → R5 eager wave-instantiate the non-lazy singletons (the eager bitset EXCLUDES the config-properties beans validate() already pre-bound — eager-EXCLUDED-because-PREBOUND — so R5 publishes their already-bound Arc and never re-runs their factory) → R6 SmartInitializing barrier → R7 start_all() → R8 publish. A bind/coercion/JSR failure is therefore a Tier-2 AssemblyReport at validate() (RunState still pre-refresh), NEVER an R5 cancel-cascade (RunState→Failed). The Engine→Context delegation surface is a single inherent-method set on Engine that Context forwards by hand (resolving the open sub-question: hand-forwarded inherent methods, NOT a re-declared public trait, to avoid surface-drift while keeping the shared surface one place). `leaf::run()` is the orchestration layer ATOP Context, never a third container type.

**Resolved open questions:** Registry substrate / get<T> return type: SETTLED by toolkit — singleton store is `Box<[OnceCell<ErasedBean>]>`, get returns `Ref<T>` for shared / `T` for prototype.; BeanFactory/ApplicationContext split survives as two named types (R1) — NOT collapsed into a capability container (R2) and NOT a kernel+extension SPI (R3).; Async-across-dyn: BoxFuture at the Provider dyn seam is the origin-agnostic default; monomorphized typed fast-path is a deferred Phase-3-escape, not the kernel default.; ONE concrete Engine (no `dyn Engine` trait, no pluggable strategy kernel) — the swappable-strategy SPIs are rejected upstream.; Auto-detection completeness / silent-DCE: handled by the mandatory expected-vs-found `ExpectedManifest` self-check run inside refresh(), per-Context.; Delegation surface Engine↔Context: hand-forwarded inherent methods (low-stakes, chosen to minimize drift).

### `bean-definition`
ADOPT a const flat `Descriptor` row (R1's record fidelity) with R2's collapse-at-freeze for parent→merged (NOT a runtime stale-flag cache, NOT an open attribute map). The macro emits ONE const `Descriptor { contract: ContractId, self_type: TypeId, provides: &'static [TypeRow], declared_name: Option<&'static str>, aliases: &'static [&'static str], scope: ScopeDef, role: Role, meta: &'static AnnotationMetadata, parent: Option<ContractId>, origin: Origin }` — qualifiers/markers/depends_on/init/destroy/primary/fallback live in the flat const `AnnotationMetadata` (closed core + the already-stable annotation-model tables, NOT a `HashMap<TypeId,Box<dyn>>` — that R3 heap/dyn cost is rejected). The construction recipe is the origin-agnostic `ProviderSeed` (a const fn-pointer that BUILDS the Provider), never editable arg metadata — resolving the reflective-property-bag clash: a BFPP-analogue can rewrite METADATA (scope/lazy/role/qualifiers) during `App<Resolve>` but the typed factory closure is opaque/fixed at the declaration site; placeholder-resolved values become `ResolveCtx`-read inputs the Provider reads, not definition edits. Parent→merged: `parent: Option<ContractId>` references a template definition; at `freeze()` the asymmetric merge (child takes scope/lazy/depends_on/role from itself, inherits qualifiers/init/destroy/meta from parent) is computed ONCE producing the effective frozen `Descriptor` — there is no separate `MergedBeanDefinition` runtime type and no stale/recompute machinery, because the freeze IS the merge moment. An `abstract`/template-only parent (no `ProviderSeed`) is never instantiated (gets no `BeanId` slot in the providers array, or a slot that errors on construction). Definitions hold no live resources (pure data + an opaque fn-pointer), so no async-Drop concern. The merge is total-by-construction (closed schema → one merge fn covered by a test, NOT R1's hand-maintained per-field overlay that drifts).

**Resolved open questions:** Mutable-in-place vs immutable-layered vs open-attribute-map: SETTLED — flat const closed-schema `Descriptor` (R1 record) with R2 collapse-at-freeze for parent-merge; R3's open `HashMap<TypeId,Box<dyn>>` rejected for heap/dyn cost. Cross-cutting features attach typed metadata via the const `AnnotationMetadata` tables, not a boxed map.; What a BFPP-analogue can rewrite: editable METADATA (scope/lazy/role/qualifiers) during App<Resolve>; the typed `ProviderSeed` factory is opaque/fixed; resolved values are ResolveCtx inputs, not definition edits.; Ownership of stored/merged def: the frozen `Descriptor` lives in `rows: Box<[Descriptor]>` indexed by BeanId — owned by the FrozenContainer, no per-def Arc.; Codegen boundary: macro emits a flat const `Descriptor` + const `ProviderSeed` (R1/R2 const-friendly path) via absolute ::leaf_core paths.; Merge/validation timing: parent→merged collapse + bad-parent/abstract-instantiation errors at freeze; wiring/ambiguity at the App<Wired> validation pass (richest-diagnostic timing).; Freeze/seal boundary: typestate-consuming `RegistryBuilder::freeze()` (the markBeanAsCreated analogue) — a post-freeze edit is unrepresentable (the mutable builder is consumed).; qualifiers/depends_on/autowire_candidate boundary: first-class core fields in `AnnotationMetadata` (read on the hot-ish candidate path), not an open extension surface.; Factory handle shape across dyn: boxed-future `ProviderSeed`/`Provider::provide` (alloc per construction, cold for singletons) is the origin-agnostic default.; definition-parent (template merge) vs context-parent (hierarchy lookup): kept explicitly separate — `Descriptor.parent: Option<ContractId>` is template merge; `Context.parent: Option<Ref<Context>>` is hierarchy lookup (see container-hierarchy).

### `bean-registry`
ADOPT R2 (the two-epoch frozen-snapshot registry) as fixed by ADR-02/ADR-04, with R3's pluggable-strategy kernel explicitly REJECTED (merge/store/guard/resolver are each one concrete implementation, never a `Box<dyn>` SPI). `pub struct Registry { rows: Box<[Descriptor]>, by_type: HashMap<TypeId, SmallVec<[BeanId;1]>>, by_name: IndexMap<BeanName, BeanId>, aliases: HashMap<BeanName, BeanId>, by_contract: HashMap<ContractId, BeanId>, providers: Box<[Arc<dyn Provider>]>, singletons: Box<[OnceCell<ErasedBean>]> }` — all indexed/joined on `BeanId.0`. `RegistryBuilder` (cold, append-only): `register(d: Descriptor, p: Arc<dyn Provider>) -> Result<BeanId, RegisterError>` (dup-name/contract = loud error gated by `allow_override`), `freeze(self) -> Result<Registry, AssemblyError>` (materializes dense space + both indices + runs ContractId collision guard + anti-DCE self-check), `reopen_epoch(reg) -> Self` (the sanctioned re-freeze seam; carries forward initialized `OnceCell` cells for retained static BeanIds so a singleton keeps ONE instance across the swap). Read surface: `candidates(ty: TypeId) -> &[BeanId]` (concrete + dyn-Svc TypeRow rows; autowiring picks the winner — the registry STOPS at candidate sets), `resolve_id(key: &BeanKey) -> Result<BeanId, ResolveError>`, `descriptor(id)`, `provider(id)`. NO global lock: the per-slot `OnceCell<ErasedBean>` IS the per-bean creation guard (lock-free ready read = bounds-checked array index `singletons[id.0].get()`, at-most-once init by data shape). Single-phase construction means NO three-level cache (ready/early/factories) — the speculative middle ObjectFactory level is dropped; constructor cycles are logically fatal (reported at the validation pass), and the only cycle break is a deferral edge (`Lookup<T>`/`LazyRef<T>`) removed from the construction graph. The strict-vs-lenient reconciliation hazard dissolves: the OnceCell slot IS the arbiter (concurrent first-resolution racers await the winner's published value), so background/wave-parallel eager init needs no lenient-build-outside-the-guard reconciliation. The 'bare engine is dumb / context installs the rich resolver' split is preserved structurally: the bare Engine is inert (no Role::Infrastructure auto-detect), the rich candidate-resolution layer (qualifiers/generics) is the same one concrete implementation but only EXERCISED once the context's infrastructure + injection points exist.

**Resolved open questions:** Singleton-cache handle type: SETTLED — `OnceCell<ErasedBean>` slot-indexed by BeanId (R2), origin-agnostic, no per-bean Arc churn beyond the ErasedBean itself.; Three singleton-cache levels or two: NEITHER — single-phase construction + deferral-only cycles means ZERO early-reference levels; the speculative middle ObjectFactory level is dropped.; Per-bean creation guard substrate: per-slot `OnceCell` (R2). The strict/lenient lenient-build-and-reconcile hazard is dissolved (the slot is the arbiter); no global lock.; Mutable-concurrent-map vs immutable-snapshot: immutable-snapshot-at-freeze (R2). Mid-bootstrap dynamism (registrars/init-methods registering new defs during preInstantiate) is NOT supported mid-steady-state; the only sanctioned re-entry is `reopen_epoch()` → re-seal.; Async-across-dyn boxing: BoxFuture at the Provider seam, default; monomorphized fast-path deferred.; Name-vs-type coherence: TypeId-primary, name-overlay, built coherent ONCE at freeze (R2 dissolves the transactional desync bug class).; Injectable-as-supertype: macro-emitted `TypeRow` upcast rows (one per declared injectable supertrait; policy = all-declared for cross-crate safety, measured for row-count). Concrete handles match exact TypeId only.; ONE concrete engine vs pluggable kernel: ONE concrete `Engine` + ONE concrete `Engine::create` (R3 strategy SPIs rejected — they re-open settled decisions, widen the ABI, kill the monomorphized path).; Build-time vs runtime merge/validation: parent-merge + ContractId guard at freeze; whole-graph wiring validation at App<Wired>; `cargo leaf prepare` build-time plan is an opt-in accelerator, not v1-required.

### `bean-naming`
ADOPT R1 (macro-derived canonical name + interned overlay + guarded collision policy) — NOT R2's lazy-derive-on-demand (which defers collision detection past startup, violating the fail-fast-at-seal invariant). The `#[component]`/`#[bean]` macro derives the default name at expansion via a pure leaf-core `fn derive_default_name(simple: &str) -> Cow<str>` (decapitalize + the acronym edge case: first two chars uppercase → preserved), emitting a const literal into `Descriptor.declared_name`/`aliases`; an explicit `#[component(name=...)]` overrides (first listed = canonical, rest = aliases). `BeanName = Arc<str>` interned at `freeze()` (cheap-clone identity, integer-cheap comparison via pointer identity, not String churn). The name overlay is `by_name: IndexMap<BeanName, BeanId>` (insertion-ordered = deterministic listing for collection injection) + `aliases: HashMap<BeanName, BeanId>` resolving to the target slot in one hop. Collision is a LOUD `RegisterError`/`AssemblyError` at register()/freeze() by default (carrying both contributors' crate provenance via ContractId + declaration-site spans), gated by the per-registry `allow_override` toggle; an `is_compatible` predicate (same self_type + same ProviderSeed identity) tolerates idempotent re-detection. Alias resolution is ORDER-INDEPENDENT (resolved in the freeze pass after all rows collected, never relying on registration order) — forward/cross-crate aliases work. Alias cycles (a→b→a) are a loud freeze-time error. The opt-in fully-qualified name generator (ContractId-derived FQN) is the cross-crate same-simple-name collision remedy. The interner-across-hierarchy concern is resolved: each registry interns its own names; cross-registry name identity is by string-equality at the `BeanKey::ByName` boundary (a parent BeanId is never a child BeanId), so no shared/contended interner is needed. Five name-needing semantics ride this overlay: aliases, name-collision-as-error, name-keyed `Map<String,T>` injection, `&FactoryBean` deref, bean-name-as-implicit-qualifier.

**Resolved open questions:** Eager-derive + guarded-collision (R1) vs lazy-derive overlay (R2): SETTLED on R1 — eager derivation + collision-at-seal preserves the fail-fast-at-startup determinism the charter and the seal boundary demand. R2's lazy detection (collision only at ambiguous by-name access) rejected.; Acronym edge-case spec: implemented as a pure, unit-testable leaf-core fn over the simple ident (last path segment); needs its own tested spec given Rust's lexicon differs from Java's.; Interner across hierarchy: each registry interns locally; cross-registry name match is by string at the BeanKey::ByName boundary (no shared interner, no contention).; Forward/cross-crate aliases + ordering: alias graph resolved order-independently in the freeze pass; alias cycles are a loud freeze error.; allow_override granularity: a per-registry boolean for the genuinely-loud case; auto-config 'intended override' is handled by the @Fallback CandidateRole (softer back-off), NOT by flipping allow_override.; Cross-package clash remedy: opt-in ContractId-derived fully-qualified name generator, never auto-prefixing.; BeanName representation: `Arc<str>` interned at freeze (cheap-clone, integer-cheap comparison), satisfying the no-String-churn discipline.

### `container-hierarchy`
ADOPT R1 (parent link as a queryable field) — NOT R2 (borrowed ResolutionScope, which breaks deferral handles that outlive the stack frame and has no queryable parent for Aware-style features) and NOT R3 (swappable HierarchyPolicy SPI, which makes the load-bearing lowest-wins/listing-local guarantee user-reconfigurable and widens the ABI). The parent link is the toolkit-fixed `Context.parent: Option<Ref<Context>>` field (Option only because a root has no parent — NOT a capability). Hierarchy is a relationship BETWEEN per-registry FrozenContainers, layered on the existing resolution surface, never an engine internal. Resolution routing encodes the asymmetry by hand (Rust offers no automatic analogue): `get<T>`/`get_erased(BeanKey)` check the LOCAL registry first; on a local MISS, delegate `parent.get(..).await` (lowest-factory-wins). `contains_local` = local only; `contains` = local OR parent. Single-valued by-type resolution: run the LOCAL candidate pipeline + fail-fast tie-break; unique local winner → done (local shadows parent); local set EMPTY → delegate to parent; local set AMBIGUOUS → fail LOCALLY (ambiguity at a level never silently falls through — per autowiring no-guess-on-ambiguity, encoded in the hierarchy_depth tie-break the injection-mechanics Selector already owns). LISTING (`names()`/`beans_of_type()`) is LOCAL-ONLY by construction; spanning is a separate explicit `*_including_ancestors()` that walks parent and collapses a locally-shadowed name to the LOCAL entry (shadow-de-dup). Cross-level handles are origin-agnostic ErasedBean (a parent's ONE cached singleton is shared into the child via the same Ref<T>; no re-creation). Per-container isolation is structural: each Context owns its own FrozenContainer + infrastructure set + `singletons`; a parent's post-processors physically cannot see child beans. `set_parent` runs a runtime ancestor-cycle check. Teardown order is composer-driven: children shut down before parents (the `Ref<Context>` parent link's strong-count keeps the parent alive until children drain). Cross-level NoSuchBean records the level in the ResolveError chain ('not found locally in child `web`; delegated to parent `root`; parent reported ambiguous X,Y'). MessageSource resolution delegates upward on miss via the same parent walk.

**Resolved open questions:** Parent-link lifetime/ownership: SETTLED — `Option<Ref<Context>>` (Arc-backed, R1), refcounted so a parent outlives its children; cross-level shared singleton is the same origin-agnostic ErasedBean. R2's borrowed-chain rejected (breaks deferral/scoped-proxy handles).; Queryable vs ambient parent: QUERYABLE stored link (R1) — Aware-style features and programmatic walks can ask `context.parent()`. R2's ambient-in-resolution parent rejected.; Same upward-walk primitive for context hierarchy AND scope chains: kept SEPARATE for v1 (scope stores are the ambient `InstanceStore` reached via `Cx`, a different axis); the BeanKey-walk delegation does not also serve scope layering, avoiding R2/R3's premature unification.; Fixed parent field (R1) vs swappable ResolverSource/HierarchyPolicy chain (R3): FIXED field — the lowest-wins + listing-local guarantee is privileged and NOT user-reconfigurable (R3's policy footgun rejected; the SPI widens the stable ABI for flexibility nothing requests).; Lowest-wins by-type edge: local-empty→delegate, local-ambiguous→fail-local, local-unique→shadow — encoded via the Selector's hierarchy_depth tie-break.; Listing-local vs injection-hierarchy-aware asymmetry: manufactured by splitting `names()` (local) from `*_including_ancestors()` (explicit opt-in, shadow-de-dup).; Per-container post-processor/namespace isolation: structural (each Context owns its FrozenContainer); a parent BPP cannot process child beans.

### `factory-bean`
ADOPT R2 (FactoryBean is ONE Provider impl, no second registry slot) as fixed by ADR-05 — NOT R1 (dual-slot, which doubles definition count and forces a factory↔product link bean-naming must model) and NOT R3 (indirection-as-metadata, which can't express runtime-dynamic product types). A user factory implements `trait FactoryBean { type Product; async fn create(&self, cx: &ResolveCtx) -> Result<Self::Product, ResolveError>; fn product_type(&self) -> TypeId; fn is_singleton(&self) -> bool { true } }` (+ optional `SmartFactoryBean` for `is_eager_init`/`supports_type`/`create_as` multi-type). The macro emits ONE `Descriptor` for the PRODUCT whose `provides[]` carries the product TypeId (so candidate-resolution finds the product pre-construction WITHOUT realizing — the getObjectType-without-getObject contract is a `provides[]` row). The Provider is `FactoryBeanProvider<F: FactoryBean> { factory_id: BeanId }`: its `provide` does `let f: Ref<F> = cx.engine().get_by_id(self.factory_id).await?; Ok(f.create(cx).await?.into_published())`. CRITICAL: the factory itself is a FULL bean (its own BeanId, full Aware/init lifecycle, resolved THROUGH the one `Engine::create` driver) — resolving R2's biggest weakness (factory-not-getting-full-treatment) because the factory is a real reachable bean by `BeanKey::ByName`, not an internal detail. The product singleton memo lives uniformly in `registry.singletons[product_id]` (the per-slot OnceCell guards the commit; `create().await` runs OUTSIDE any held guard, dissolving Spring's getObject-under-singletonLock deadlock). A prototype/scoped product has no memo (Published::Owned, re-create per resolve; author owns create thread-safety there). The `&factory` deref is `BeanKey::ByName` + a Deref flag on the name overlay (factory vs product), or typed `get_factory::<F>()` — dereferencing a non-factory is a loud lookup error, never a runtime sigil parse. SmartFactoryBean multi-type emits multiple product TypeIds as `provides[]` rows; `create_as(t)` dispatches. A null product → the canonical NULL_BEAN (see extra-4). Teardown: the factory registers its own destruction in the TeardownLedger; product teardown rides the factory's ledger entry (disciplined — divergent from Spring's orphan-and-leak, mandatory given no async Drop).

**Resolved open questions:** Two-slots (R1) vs Provider-of-Provider (R2) vs indirection-metadata (R3): SETTLED on R2 (one Provider, no second slot) — ADR-05 fixed it. The factory stays a FULL bean reachable by BeanKey::ByName (R2's weakness fixed by routing factory construction through Engine::create, not re-implementing lifecycle inside the Provider).; Singleton-handle type / product cache substrate: product memo lives in `registry.singletons[product_id]` uniformly with all singletons (no separate factory_bean_products cache; no compound key).; Factory↔product encoded as two slots or one Provider wrapping another: one `FactoryBeanProvider` wrapping the factory BeanId — no factory→product link field to model, simpler cross-crate.; Async-across-dyn boxing: `create` is async; `provide` is BoxFuture at the dyn seam; cold for singleton products, a per-resolution cost for prototype products (accepted).; Build-time vs runtime type info: product TypeId emitted as a const `provides[]` row (type-match-without-realize is reading the row). Runtime-dynamic multi-type via SmartFactoryBean::supports_type consulted at candidate matching.; getObject-under-lock deadlock: dissolved — `create().await` runs outside any held creation guard; only the commit into the OnceCell slot is guarded.; Product teardown: rides the factory's TeardownLedger entry (disciplined, no orphan leak), since there is no async Drop. The orphan-vs-delegate Phase-2 knob is resolved to delegate.; Re-entrant-during-create: the per-slot OnceCell double-check handles a re-entrant get during create; a null product WHILE in-creation is a cycle error (see extra-4), not a NULL_BEAN.

### `extra-4`
ADOPT the toolkit-fixed NullBean realization: the canonical `NULL_BEAN: ErasedBean` (a `Lazy<Arc<NullMarker>>` singleton in leaf-core) occupies a REAL BeanId slot, distinguishing three states crisply: (A) present-but-null = the slot holds NULL_BEAN; (B) real value = the slot holds a normal ErasedBean; (C) never-defined = no slot / `by_type`/`by_name` map-miss = NoSuchBean. This is the 'typed absence-marker stored in the erased handle' realization (it preserves Spring's exists-and-type-matchable-but-null property — the slot answers `candidates()` by-type so collection scans see it), NOT the side-table realization (which loses type-match membership) and richer than pure Option-collapse (which can't keep present-but-null as a distinct type-match member). A `#[bean] async fn make() -> Option<T>` (or `-> Result<Option<T>, _>`) declares producible-absence in the type; the macro emits a Provider closure mapping user-`None` → `Published::Shared(NULL_BEAN.clone())`. At the typed boundary, `get<T>` maps NULL_BEAN to typed absence per the injection point's arity (resolving the toolkit's flagged sub-question): a mandatory `Ref<T>` point against NULL_BEAN is a rich `ResolveError` ('bean exists but was produced absent; this point requires a value'); an `Option<Ref<T>>`/`Lookup<T>::get_if_available`/`#[nullable]` point folds NULL_BEAN → `Ok(None)` (converging with map-miss for the consumer, but the diagnostic + count differ). Deferral ladder agreement: `Lookup::get` (strict) errors on NULL_BEAN; `get_if_available`/`get_if_unique` fold it to None. Collection injection rule (resolving the toolkit sub-question): a NULL_BEAN member is COUNTED for type-match/uniqueness arbitration (it can make a single point ambiguous, or as the lone candidate satisfy an optional point) but is FILTERED OUT of the materialized `Vec<Ref<T>>`/`Map<BeanName,Ref<T>>` values (with the name retained for diagnostics). In-creation guard: a producer returning `None` WHILE its slot's OnceCell init is in flight (re-entrant) is a `BeanCurrentlyInCreation`/cycle error, never recorded as NULL_BEAN (null = 'not done', not 'deliberately absent'); a CANCELLED producer future records nothing (left un-set, identical to in-creation). Origin-agnostic: a FactoryBeanProvider or test-double yielding nothing funnels into the identical NULL_BEAN.

**Resolved open questions:** Distinct sentinel vs Option-collapse vs side-table: SETTLED on the typed absence-marker (canonical NULL_BEAN in a real slot) — it preserves type-match-membership (which Option-collapse and the side-table lose) and keeps present-but-null/never-defined/condition-not-met as three crisp states.; NULL_BEAN typed-boundary mapping per arity (the toolkit's flagged sub-question): mandatory Ref<T> → rich error; Option<Ref<T>>/get_if_available/#[nullable] → Ok(None). Folds onto the injection-styles arity model.; NULL_BEAN in collection injection counts (the toolkit's flagged sub-question): COUNTED for arbitration/uniqueness, FILTERED from materialized values, name retained for diagnostics.; Registry-substrate marker vs typestate vs Option: SETTLED — a real BeanId slot holding the canonical NULL_BEAN ErasedBean (NOT a side-table = NoSuchBean reserved for map-miss; NOT a per-handle typestate).; Null-while-in-creation: a cycle/BeanCurrentlyInCreation error, never a NULL_BEAN. A cancelled producer records nothing.; Deferral strictness-ladder agreement: NULL_BEAN counts as 'present' for strict-get (errors), 'absent' for the tolerant tiers.; Origin-agnostic absence: WASM/test-double/FactoryBean nothing all funnel into the one NULL_BEAN; the macro hard-codes the `::leaf_core::NULL_BEAN` path.

## Cross-feature interactions
- **bean-definition** — bean-registry STORES the const `Descriptor` rows in `rows: Box<[Descriptor]>` and OWNS the freeze where parent→merged collapse happens; the metamodel defines the row shape, the registry materializes the dense BeanId space over it. The freeze IS the markBeanAsCreated boundary for both.
- **container-core** — Engine HAS-A the frozen `Registry` (bean-registry); Context HAS-A the Engine. The container-core Engine::create is the ONE creation driver that bean-registry's get/eager-instantiate calls through; container-core's refresh() is where the registry is sealed and the App<Wired> validation pass runs.
- **bean-naming** — The name overlay (`by_name` IndexMap + `aliases`) folds onto the SAME BeanId as the type index — coherent by construction at freeze. bean-registry enforces bean-naming's loud collision policy at register()/freeze(); the five name-needing semantics all ride this one overlay.
- **factory-bean** — FactoryBeanProvider is just one Arc<dyn Provider> in the `providers` array (no special registry slot); the product TypeId is a `provides[]` TypeRow so candidate-resolution finds it pre-construction. The factory is itself a full bean reachable by BeanKey::ByName (the &-deref). Product singleton memo is `registry.singletons[product_id]` — uniform with all singletons.
- **extra-4 (NullBean)** — NULL_BEAN occupies a real BeanId slot in the same `singletons`/`providers` arrays; a FactoryBeanProvider yielding None publishes NULL_BEAN. get<T> maps it to typed absence at the boundary. NoSuchBean (map-miss) stays crisply distinct from present-but-null (NULL_BEAN slot) — three states the diagnostics model keeps separate.
- **container-hierarchy** — Each Context owns its own FrozenContainer (per-registry BeanId space — a parent BeanId is never a child BeanId); cross-registry delegation is by BeanKey walk, never by raw slot or shared registry. A parent's ONE cached singleton is shared into the child as the same origin-agnostic ErasedBean. bean-naming's within-container loud collision is distinguished from cross-level silent shadowing (local shadows parent).
- **injection-mechanics (upstream)** — bean-registry STOPS at `candidates(ty) -> &[BeanId]` + the name/alias overlay; the Selector/cmp_order/determine_winner ladder (primary→name→qualifier→@Priority→hierarchy_depth) is injection-mechanics' concern. The App<Wired> validation pass runs Selector::resolve_one over the frozen registry. NULL_BEAN arity-folding and hierarchy lowest-wins are encoded in that Selector.
- **conditional-strategy + autoconfig (upstream)** — Descriptors register during App<Resolve> incrementally so OnMissingBean/DefinitionProbe see the growing set; an auto-config bean registers with `Role`/CandidateRole::Fallback so a user bean transparently supersedes. A condition-false bean mints NO slot (distinct from NULL_BEAN's present-but-null and from a DCE-vanished crate's NoSuchBean).
- **async-context-model (upstream)** — Provider::provide returns BoxFuture; Engine::create awaits it; the per-slot OnceCell init spans .await with no executor-blocking lock. Context-scope InstanceStores key on the SAME BeanId via the ambient Cx. Teardown is the explicit TeardownLedger drain (factory + product teardown), no async Drop.
- **cross-crate-discovery (upstream)** — freeze() runs the expected-vs-found ExpectedManifest self-check — a DCE-dropped crate's absent BeanId surfaces as a loud AntiDceError::SourceVanished, distinct from a genuine NoSuchBean. ContractId (collision-guarded at freeze) is the stable cross-crate identity; reopen_epoch() is the dynamic re-seal seam.

## Crate hints
- leaf-core: the ultra-stable kernel ABI every macro hard-codes `::leaf_core` paths to. Hosts ErasedBean/Ref/Bean/Published, Provider, BeanId/BeanName/ContractId/BeanKey, Descriptor/TypeRow/Role/Origin/AnnotationMetadata, ScopeDef, Registry/RegistryBuilder, Engine/EnginePolicy/Context, FactoryBean/FactoryBeanProvider, NULL_BEAN/NullMarker, ResolveError/AssemblyError. This subsystem IS most of leaf-core's surface.
- Dependencies: `indexmap` (by_name insertion-ordered overlay), `smallvec` (candidate SmallVec<[BeanId;1]>), `once_cell` (per-slot OnceCell singleton store + Lazy NULL_BEAN), `futures` (BoxFuture). `linkme` for the distributed-slice COMPONENTS/AUTO_CONFIGS channels (cross-crate-discovery owns the slices; this subsystem consumes them via RegistryBuilder::from_slices).
- leaf-macros (proc-macro crate, thin): emits exactly one const `Descriptor` + const `ProviderSeed` per #[component]/#[bean]/#[configuration] via absolute ::leaf_core paths; calls the leaf-core `derive_default_name` fn (bean-naming) at expansion or emits the const literal. Hard-errors on generic beans (register_component!(Concrete) hint). Carries no merge/index/freeze logic.
- leaf-boot / leaf::run: the orchestration layer ATOP Context (App<Define→Resolve→Wired→Running> phase machine, seal(), run_autoconfig). Builds Engine→Context, drives refresh().await. Not a container type.
- Naming policy fn (`derive_default_name`) lives as a plain pure fn in leaf-core so it is unit-testable in a normal crate (the acronym edge case needs its own tested spec).

## Remaining risks (→ Phase 4)
- TypeRow supertrait-emission policy (all-declared-injectable-supertraits vs only-used): chosen all-declared for cross-crate safety, but row-count bloat is unmeasured — needs a benchmark on a realistic large graph before Phase 4 locks it. Interacts with collection/Map injection over a supertype.
- ContractId derivation inputs + collision policy: exact 'crate-stable name' feed (Cargo package + module path + ident + declared-name?) and whether the freeze-time guard hard-errors or salts is a semver/exclusion surface co-decided with auto-config exclusion identity — not fully pinned. A re-export/rename changing the ContractId is a latent semver footgun.
- reopen_epoch singleton carry-forward correctness: seeding epoch N+1's singletons from N's initialized OnceCell cells for retained static BeanIds must preserve the Once multiplicity contract across the arc_swap; in-flight reconciliation across the swap and dynamic-row precedence vs a static @Primary are deferred (dynamic lane is v1-droppable, native-only).
- The acronym edge-case name-derivation spec on Rust idents (snake_case modules, _-bearing idents, type aliases, re-exports) reads differently from Java class names — needs a well-specified tested spec; 'the simple name' is less unambiguous than Java's class name.
- Dropping the three-level singleton cache (middle ObjectFactory level) is sound ONLY because construction is single-phase with deferral-only cycles and transparent advice is shared-shape — if the proxy substrate's advised-prototype Owned lane needs an early-reference hook, the no-early-level assumption must be revisited (cross-checked with proxy-substrate's TargetSource Owned lane).
- Performance escapes deferred and unmeasured: the opt-in singleton-only arena/BeanRef<T> fast lane, skip-Cx-install-when-unchanged, and whether the always-on App<Wired> validation should be opt-out. All measure-before-build.
- Engine↔Context hand-forwarded delegation surface risks drift if the shared inherent-method set grows; low-stakes but needs a discipline (single source-of-truth method set) to not re-declare a public trait.
