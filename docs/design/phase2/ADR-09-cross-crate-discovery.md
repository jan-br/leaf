# ADR 09 — Cross-Crate Discovery: Maximal-Magic + Anti-DCE + Static/Dynamic-WASM Duality  `[cross-crate-discovery]`

> **Note (2026-06-14):** the optional *dynamic-WASM source / WASM-"classpath"* mentioned below is **DROPPED** — leaf autodiscovery is native link-time registration only. Ignore WASM/dynamic-module references here; they are struck from the active design.


## Decision
ADOPT a SINGLE discovery substrate — `linkme` distributed slices as the default static "classpath" on server targets, behind a uniform `&'static [Descriptor]` shape that is swappable to build-codegen (wasm/embedded) and explicit registration (universal escape hatch) — with FOUR concrete mechanisms this keystone owns and the upstream ADRs deferred to it:

(1) MAXIMAL-MAGIC ZERO-DECLARATION DISCOVERY. A crate's `#[component]`/`#[bean]`/etc. emit `#[distributed_slice(::leaf_core::COMPONENTS)]` const `Descriptor` rows (codegen-boundary ADR Layer 1). Adding a crate as a dependency AND referencing it makes its components appear — no per-component declaration, no scan-scope. The "scope" is exactly the set of crates linked + force-linked into the binary; there is NO basePackages analogue and NO opt-in list of components. This is the FIXED owner decision; this ADR makes it robust, not optional.

(2) ANTI-DCE TRIPLE-DEFENSE (the load-bearing core; silent-vanish is the #1 hazard). Three INDEPENDENT defenses, each necessary, layered at the binary crate (the only point seeing the whole graph):
  • DEFENSE 0 — FORCE-LINK (defeats "unreferenced rlib not linked"). `#[leaf::main]` + a generated binary-crate `build.rs` emit `use <crate> as _;` for every participating crate. The participating set = (a) every `dep:`-hidden capability feature that is enabled, UNIONED with (b) an explicit `leaf::scan!{ leaf_tokio, leaf_redis }` list. Enabling a starter feature AUTO-ADDS its crate to the force-link set (the user states the dependency once; force-link is automatic).
  • DEFENSE B — SECTION-RETENTION (defeats `--gc-sections`/LLD `start-stop-gc`). Shipped guardrails, not advice: MSRV >= 1.89 (ELF `#[used(linker)]`/SHF_GNU_RETAIN default) OR a generated `.cargo/config.toml` injecting `-Clink-arg=-Wl,-z,nostart-stop-gc`; `linkme >= 0.3.24` pinned in `[workspace.dependencies]`; `resolver = "3"` set explicitly at workspace root. CI MUST test the final-binary link (release + LTO + `--gc-sections`), never lib tests.
  • DEFENSE MANIFEST — EXPECTED-VS-FOUND SELF-CHECK (turns silent absence into a loud error). The binary-crate `build.rs` reads `cargo metadata` and emits a const `ExpectedManifest { sources: &[SourceTag] }` of the crates that *should* contribute (force-link set + a per-crate `provides_descriptors` marker). At assembly, `RegistryBuilder::self_check(&EXPECTED)` cross-references the link-collected `Descriptor.source_tag`s against EXPECTED; a crate present in EXPECTED but contributing zero rows yields a LOUD `AntiDceError::SourceVanished { source, reason_hint }` naming the missing crate and the likely cause (DCE / missing force-link / linker). This is mandatory because the registry is RECOGNITION not VALIDATION (registry-substrate ADR) — an absent BeanId is indistinguishable from NoSuchBean without it.

(3) EXCLUSIONS MECHANISM (Spring's `exclude=`, the predictability mitigation — NOT declaration). A bean/auto-config is suppressed by a layered exclusion set keyed on the stable `ContractId` (never TypeId — it must survive cross-build/cross-crate, registry-substrate ADR): `#[leaf::main(exclude = [SomeAutoConfig, ...])]` (compile-time, by path → ContractId), PLUS a runtime `leaf.autoconfigure.exclude` property (config-tier, by ContractId-string for ops). Exclusions are applied during the cold assembly pass: an excluded `ContractId` mints NO BeanId (distinct from condition-not-met and from DCE-absent — three crisply separate states the diagnostics model keeps apart). A typo'd exclusion (a ContractId in the exclude set that matches NO discovered row) is a LOUD `ExclusionError::NoSuchTarget` — exclusions get the same expected-vs-found rigor as discovery. Exclusion sits ABOVE auto-config back-off in the precedence ladder: exclude > user-bean-back-off > auto-config-default.

(4) STATIC/DYNAMIC-WASM CLASSPATH DUALITY + DETERMINISM. The `linkme`/build-codegen slice is the STATIC classpath (default, common case). An OPTIONAL dynamic "second classpath" (directory-scanned WASM host-proxies, registering identical origin-agnostic `Descriptor`+`Provider`, registry-substrate ADR) enters via `RegistryBuilder::reopen_epoch()` → re-freeze as a controlled SECOND EPOCH, never mid-steady-state mutation, and keys on `ContractId` (never TypeId — a foreign-toolchain module agrees on ContractId only). DETERMINISM-AS-DATA: because link/section/constructor order is UNSPECIFIED and may be randomized, ordering is NEVER read from the slice. The frozen registry computes ONE canonical total order at freeze — a `SequenceKey { contract: ContractId, epoch: u16 }` lexicographic sort (epoch-then-ContractId), used for every tie-break (collection-injection insertion order, listing order, equal-`@Order` stability). Static rows are epoch 0; each dynamic re-freeze is a higher epoch, so static always precedes dynamic deterministically and the same binary+modules always assembles identically.

## Rationale
- MAXIMAL MAGIC is the FIXED owner decision (#1), and the cross-crate research proves the only stable substrate that delivers 'add a crate dependency, its components appear' with zero runtime cost and no life-before-main is linkme distributed slices (rust-cross-crate §166: 'Default to linkme, not inventory'). inventory's life-before-main is hostile to async-first (charter §2.3); build.rs codegen cannot see a crate the binary's build doesn't already enumerate (§64 directionality wall). So linkme-default + force-link is forced by the owner decision crossed with the capability envelope — not a free choice.
- The silent-vanish hazard is LOAD-BEARING precisely because there is no declared scan-scope to fall back on (owner decision #1 removed it). The research is unambiguous that DCE is 'the #1 failure mode, and it is SILENT' (§345) across THREE independent layers (§346: Layer 0 linkage, Layer A rustc reachability, Layer B linker gc-sections). A maximal-magic framework with no scan-scope CANNOT detect a missing crate any other way, so the triple-defense (force-link + section-retention guardrails + expected-vs-found manifest) is the minimum that converts a silent empty registry into a loud, source-naming error — satisfying charter §2.9 (fail early with the richest context: 'what was expected, what was actually found, and where').
- Mitigating the implicit-global predictability concern via AUTO-CONFIG BACK-OFF + EXCLUSIONS rather than declaration is the FIXED owner decision (#1) AND the exact posture the #[global_registration] author endorses (§36, §362: 'prefer explicit, opt-in cross-crate participation' on predictability grounds). Keying exclusions on ContractId (not TypeId) is forced by the registry-substrate ADR's hard constraint that TypeId is not stable across builds; making a typo'd exclusion a loud error mirrors the discovery self-check so the mitigation mechanism is itself not silently broken — the same earliest-richest-diagnostic discipline applied to the safety valve.
- The static+dynamic duality is the WASM exploration's load-bearing split (§79-82): the native link-time registry IS the classpath for everything compiled in (common case, zero discovery cost); WASM directory-scanning is the genuinely-new DYNAMIC classpath native linking structurally cannot be. Building on the registry-substrate ADR's reopen_epoch second-epoch seam (origin-agnostic Descriptor+Provider, 'WASM-ness stops at the proxy', wasm §57) means the duality is invisible to resolution and the static path pays nothing for the optional dynamic one. The dynamic lane is genuinely optional (the WASM doc is DROPPED for v1) so this ADR specifies the SEAM, not the WASM runtime.
- Ordering-as-DATA with a canonical SequenceKey is forced by two facts the research nails: link/section/constructor order is UNSPECIFIED for both linkme and inventory and 'may be deliberately randomized' (§154, §355: 'Never rely on registration order'), and determinism is a register §7c requirement. Deriving the tie-order from the stable ContractId (already minted, already collision-guarded by the registry-substrate ADR) rather than a fresh key reuses existing machinery (charter §2.11 unification) and makes the same binary+modules assemble bit-identically — the determinism the AOT/native-image/wasm story needs without inventing a second identity.
- It composes with — never contradicts — the two upstream ADRs by completing exactly what they DEFERRED to 'the cross-crate keystone': the codegen-boundary ADR lists force-link, the expected-crate manifest, and scan! DX as binary-crate responsibilities but leaves the exclusion mechanism, the manifest format, and the determinism total-order as this keystone's job; the registry-substrate ADR provides Descriptor/ContractId/reopen_epoch/Origin and explicitly hands the anti-DCE self-check and ordering to register §7. This ADR supplies precisely those, reusing their types verbatim (charter §2.11).
- Thin macros (charter §2.10) hold: the registration macro emits one hand-writable const Descriptor (codegen-boundary ADR); scan!/main emit only `use crate as _;` and a const ExpectedManifest — all hand-writable against the public RegistryBuilder::register/self_check API. Architecture D (explicit App::with(plugin)) is the always-present escape hatch, so the entire substrate is hand-writable and the runtime API is identical across linkme/build-codegen/explicit — the substrate-swappable honesty the research demands (§335) and the charter's escape-hatch-at-every-level mandate.

## Options considered & rejected
- **build.rs codegen (Architecture C, sqlx model) as the DEFAULT server substrate instead of linkme** — rejected: Maximally portable and DCE-immune, but it CANNOT deliver maximal-magic zero-declaration discovery (owner decision #1) on the common server path: a build.rs sees only its own crate (rust-cross-crate §64 directionality wall), so foreign #[component]s are invisible to the binary's build.rs unless every contributor emits a build-time manifest by an unenforced convention — a forgetting contributor silently vanishes, trading the link-time DCE hazard for an equally-silent missing-manifest hazard while ADDING a mandatory build step for every server app. The codegen-boundary ADR already settles C as the OPT-IN wasm/embedded path; promoting it to default would contradict that ADR and defeat owner decision #1. Kept as the swappable wasm/embedded substrate behind the same &[Descriptor] API.
- **inventory (life-before-main) as the default substrate for better DCE-robustness and nominal wasm support** — rejected: More DCE-robust, but life-before-main runs constructors before main on an unknown thread, cannot run async, must not panic, with std possibly not ready and unspecified order — directly hostile to charter §2.3 async-first and §2.9 (a pre-main panic has no diagnostic context). leaf assembles descriptors+factories AFTER main on its executor, so the pre-main cost buys nothing linkme's zero-cost slice doesn't already give, and linkme's const-only constraint is a FEATURE that forces the thin-data-only macro (charter §2.10). The codegen-boundary ADR already reserves inventory for the rare non-const registered value; this ADR keeps that boundary.
- **Requiring an explicit per-component or per-crate component-enumeration list (Spring's @ComponentScan basePackages, literally ported), to make discovery predictable** — rejected: Directly contradicts FIXED owner decision #1 (MAXIMAL MAGIC = zero declaration; the predictability concern is mitigated by back-off + exclusions, NOT by requiring declaration). The owner explicitly chose to NOT reference participating crates by a scan-scope. Re-introducing a declaration list would re-open a fixed decision. The honest analogue we DO keep is the force-link/scan! crate list — but that lists CRATES the app already depends on (one line each, the Spring-Boot-starter granularity), never individual components, and it is auto-populated from enabled features.
- **TypeId-keyed exclusions and dynamic-WASM identity (skip ContractId, use the in-process TypeId everywhere)** — rejected: Fatal for both: TypeId is NOT stable across compiler builds (registry-substrate ADR, nightly-doc §453 'never serialize a TypeId or rely on cross-build ordering'), so a config-file exclusion string or a foreign-toolchain WASM module could never agree on a TypeId. Exclusions are a semver/ops surface (an exclude= in config must survive recompiles) and WASM contracts cross a toolchain boundary — both REQUIRE the stable author-assigned ContractId the registry-substrate ADR already mints and collision-guards. Reusing it (not inventing a parallel key) is charter §2.11 unification.
- **Reading ordering/precedence directly from the linkme slice order (or a #[order(n)] attribute baking precedence into link position)** — rejected: Link/section order is UNSPECIFIED and the Global Registration pre-RFC even proposes deliberate randomization (-Z layout-seed) (rust-cross-crate §154, §355). Any scheme reading order from the slice produces non-deterministic, linker-and-platform-dependent assembly — the silent worst case (passes in debug, varies in release). Ordering MUST be data on the descriptor, resolved at freeze via a deterministic sort. We derive the total order from the already-stable ContractId + epoch, so the same inputs always assemble identically regardless of linker.
- **A single mid-life mutable concurrent registry (DashMap) that both static and dynamic-WASM sources mutate in place, avoiding the epoch/re-freeze machinery** — rejected: Rejected upstream by the registry-substrate ADR (mutable-during-steady-state coherence is a latent bug it declines to own; charter §2.5 Arc-churn caution). For THIS ADR specifically, it would make 'static rows always precede dynamic rows' a transactional ordering invariant maintained on every insert rather than an emergent property of epoch numbering — re-introducing exactly the desync hazard the freeze model dissolves. The reopen_epoch → re-freeze second-epoch seam (registry-substrate ADR) gives the dynamic-source capability while keeping mid-life desync unrepresentable; this ADR builds on it rather than around it.

## Consequences
- FORCES the binary crate (via #[leaf::main] + a generated build.rs) to own the three responsibilities the kernel structurally cannot (rust-cross-crate §244): force-link references for the participating set, the const ExpectedManifest for the self-check, and (optionally) the cargo-leaf-prepare build-time plan. 'Which crates participate' becomes an explicit, scoped, predictable decision = enabled dep:-hidden feature set UNIONED with the scan! list — mitigating the implicit-global concern per owner decision #1 WITHOUT per-component declaration. A leaf binary with NO #[leaf::main]/build.rs cannot get maximal-magic discovery; the explicit App::with(plugin) escape hatch (Architecture D) is the fallback.
- FORCES shipped toolchain guardrails as a hard requirement, not advice: MSRV >= 1.89 OR a generated .cargo/config.toml with -Wl,-z,nostart-stop-gc; linkme >= 0.3.24 (RUSTSEC-2024-0407) and resolver = '3' pinned in [workspace.dependencies]; and CI that links the FINAL BINARY under release+LTO+--gc-sections across the target/linker matrix (GNU ld vs LLD), because DCE failures are silent and linker-choice is a correctness variable (§145, §360). This is a maintenance and documentation obligation on the project, and a constraint on downstream users' build environments.
- FORCES THREE crisply-distinct absence states the diagnostics model (register §6) must keep separate and the registry-substrate ADR already anticipates: (a) condition-not-met / excluded = NO BeanId ever minted (silent-by-design, recorded in the condition/exclusion report); (b) present-but-null = NULL_BEAN occupying a real slot → typed absence; (c) DCE-vanished = a crate in EXPECTED contributing zero rows → LOUD AntiDceError. Conflating any two of these reintroduces the silent-vanish hazard or makes a deliberate exclusion look like a bug. The expected-vs-found self-check is the ONLY thing that separates (a)/excluded from (c).
- FORCES exclusion identity onto ContractId and thus co-binds this ADR's exclusion surface with the registry-substrate/codegen-boundary ADRs' open ContractId-derivation question: the exact stable string feeding contract_hash (package + module-path + ident + declared-name) is now ALSO a semver/ops surface because a config-file leaf.autoconfigure.exclude key references it. Renaming/re-exporting a crate changes the ContractId and breaks an exclusion silently — so the collision guard must hard-error and the derivation inputs must be documented as a public contract. (Co-decided in Phase 3 with the two upstream ADRs.)
- FORCES the dynamic/WASM lane to be a controlled SECOND EPOCH (reopen_epoch → re-freeze), never mid-steady-state mutation, with static rows at epoch 0 and dynamic rows at higher epochs — so static ALWAYS deterministically precedes dynamic in the SequenceKey total order. This forces the container-refresh lifecycle to expose a definition-phase → seal → (optional dynamic epoch) → steady-state progression, and pushes the in-flight-resolution-reconciliation + dynamic-vs-static precedence-beyond-ordering semantics into the WASM-layer Phase 3 (shared open question with the registry-substrate ADR).
- FORCES auto-config back-off, exclusions, and discovery into one ordered assembly precedence the cold pass applies deterministically: exclude (ContractId) > user-bean back-off (@ConditionalOnMissingBean analogue, register §8) > auto-config default. Auto-config (owner decisions #2/#3) is the R1 runtime-deferred-batch engine (codegen-boundary ADR) so OnMissingBean sees the final user-def set; this ADR's exclusion check runs BEFORE that batch so an excluded auto-config never participates in back-off at all. A starter = dep:-hidden umbrella feature that gates compilation AND auto-adds to the force-link set (both required: cfg gates compilation, not linkage, §352).
- ACCEPTS that discovery is NOT a true classpath scan and NOT uniformly passive across platforms (rust-cross-crate §231): it is link-time collection WITHIN each linked crate + explicit opt-in of WHICH crates participate. On wasm/embedded/staticlib the substrate degrades to build-codegen (Arch C) or explicit registration (Arch D) behind the identical &[Descriptor] API. leaf MUST document honestly that 'add a transitive dep and it silently injects beans' is NOT how it works — which is also the predictability posture owner decision #1 endorses (participation = enabled feature + force-link, never an arbitrary transitive dep).
- ACCEPTS a named one-shot cold cost and zero hot-path cost: link-time collection is zero runtime cost; the only cost is the single synchronous around-main assembly pass (lift rows, self-check, evaluate conditions, run exclusion + back-off batch, compute the SequenceKey sort, freeze). The optional cargo-leaf-prepare build-time plan can move union/dedup/exclusion-typo-guard/sort to build time for determinism-sensitive deployments (codegen-boundary ADR Layer 3c). No part of discovery, exclusion, or ordering touches a steady-state hot path.
- ACCEPTS that the BOM-equivalent (owner decision #3) is non-native: Cargo has no BOM, so version coordination = [workspace.dependencies] internally (one copy of leaf-core/linkme/macros, preventing the 'two type registries' hazard, §266) + a version-pinned umbrella facade crate `leaf` downstream that re-exports core+macros+enabled integrations and force-links them. This forces the strict inward-pointing crate topology (integration crates depend on leaf-core, never the umbrella, to avoid cycles) the charter §6 and rust-cross-crate §264 already mandate.
- FORCES a per-crate provides_descriptors marker (a const anchor symbol each contributing crate emits, force-linked alongside use crate as _;) so the self-check can attribute 'this crate was linked but contributed zero Descriptors' versus 'this crate was never linked' — distinguishing a force-link failure (Defense 0) from a section-GC failure (Defense B), yielding the rich reason_hint in AntiDceError. Without this anchor the self-check could only say 'expected source absent', not WHY.

## API / type sketch

// ═══ leaf-core: the cross-crate discovery surface, atop the registry-substrate + codegen-boundary ADRs ═══
// (Descriptor, ContractId, BeanId, RegistryBuilder, reopen_epoch, COMPONENTS slice = upstream ADRs; reused verbatim)

// ── DETERMINISM: ordering-as-DATA. Never read from link/section order. ──
#[derive(Clone, Copy, PartialEq, Eq)]
pub struct SequenceKey { pub epoch: u16, pub contract: ContractId } // lexicographic (epoch, contract); static = epoch 0
// Registry sorts every tie (collection-injection insertion order, listing, equal-@Order) by SequenceKey at freeze.

// ── ANTI-DCE: the expected-vs-found self-check (the load-bearing defense) ──
#[derive(Clone, Copy)]
pub struct SourceTag(pub &'static str);                 // stable crate identity (Cargo package name); on every Descriptor
pub struct ExpectedManifest { pub sources: &'static [SourceTag] } // emitted by binary-crate build.rs from cargo metadata
// each participating crate force-links a const anchor so "linked-but-zero-rows" != "never-linked":
#[macro_export] macro_rules! provides_descriptors { () => { /* emits a #[used] anchor symbol into ::leaf_core::SOURCES */ } }
#[linkme::distributed_slice] pub static SOURCES: [SourceTag] = [..]; // one per crate that called provides_descriptors!()

pub enum AntiDceError {
    SourceVanished { source: SourceTag, reason_hint: VanishReason }, // expected, zero rows → LOUD, names the crate
}
pub enum VanishReason { NotForceLinked, SectionGcDropped, Unknown }   // anchor-present-but-no-rows ⇒ SectionGcDropped, etc.

// ── EXCLUSIONS: Spring's exclude=, keyed on the STABLE ContractId (never TypeId) ──
pub struct ExclusionSet { contracts: std::collections::HashSet<ContractId> } // built from compile-time + runtime sources
pub enum ExclusionError { NoSuchTarget(ContractId) }                 // a typo'd exclude matching NO row → LOUD
impl ExclusionSet {
    pub fn from_compile(excluded: &'static [ContractId]) -> Self;     // #[leaf::main(exclude=[Foo,..])] → paths → ContractId
    pub fn merge_runtime(&mut self, prop: &str /* leaf.autoconfigure.exclude */); // ops-tier, by ContractId-string
    pub fn validate(&self, discovered: &[Descriptor]) -> Result<(), ExclusionError>; // typo guard mirrors the DCE check
}

// ── ASSEMBLY: the single cold pass around main; extends the upstream RegistryBuilder ──
impl RegistryBuilder {
    pub fn self_check(&self, expected: &ExpectedManifest) -> Result<(), AntiDceError>; // Defense MANIFEST
    pub fn apply_exclusions(&mut self, ex: &ExclusionSet) -> Result<(), ExclusionError>; // exclude > back-off > default
    // (evaluate_conditions / run_autoconfig / freeze are the codegen-boundary ADR's; freeze sorts by SequenceKey)
    pub fn reopen_epoch(reg: Registry) -> Self;                       // registry-substrate ADR: dynamic/WASM 2nd epoch
}

// ── STATIC vs DYNAMIC classpath duality (origin-agnostic; "WASM-ness stops at the proxy") ──
// STATIC  (default, epoch 0): #[distributed_slice(::leaf_core::COMPONENTS)] rows, force-linked, self-checked.
// DYNAMIC (optional, epoch≥1): a directory-scanned WASM host-proxy registers an identical Descriptor{origin:DynamicWasm}
//   + Arc<dyn Provider> via reopen_epoch()→re-freeze; keys on ContractId; static always sorts before dynamic.
//   The dynamic lane is OPTIONAL (WASM doc dropped for v1) — this ADR specifies the SEAM, not the runtime.

// ── BINARY-CRATE DX (thin macros; everything hand-writable) ──
// #[leaf::main(exclude = [leaf_redis::RedisAutoConfig])]    async fn main() -> AppResult { App::run().await }
//   expands (codegen-boundary ADR) to, plus this ADR's contributions:
//     use leaf_tokio as _;  use leaf_redis as _;            // Defense 0 force-link (feature set ∪ scan! list)
//     const __LEAF_EXPECTED: ExpectedManifest = ExpectedManifest { sources: &[SourceTag("leaf_tokio"), ..] };
//     const __LEAF_EXCLUDE:  &[ContractId]    = &[ contract_hash("leaf_redis::RedisAutoConfig") ];
// leaf::scan!{ leaf_extra };                                 // add a crate to the participating set explicitly
// App::new().with(leaf_tokio::plugin()).register(d, p).build().await  // Architecture D universal escape hatch

// ── STARTER + BOM (owner decision #3) ──
// integration Cargo.toml:  [features] redis = ["dep:leaf-redis"]      # dep:-hidden capability; enabling auto-force-links
// umbrella `leaf`:         #[cfg(feature="redis")] pub use leaf_redis; // re-export + force-link

## Forces on other concepts
- **Type-Erasure & Registry Substrate (upstream registry-substrate ADR)**: Honored and built upon unchanged: discovery emits that ADR's Descriptor into COMPONENTS; exclusions and WASM identity key on its ContractId (never TypeId); the dynamic classpath uses its reopen_epoch second-epoch seam and its origin-agnostic Provider; the three absence states (condition/excluded = no slot, NULL_BEAN = typed absence, DCE-vanished = loud) are exactly its NullBean/Origin model. This ADR adds the SequenceKey total order (derived from its ContractId) and the SourceTag/anchor self-check; it owns DISCOVERY+ANTI-DCE+EXCLUSION+ORDERING, that ADR owns KEYING+STORAGE.
- **Metadata / Codegen Boundary (upstream codegen-boundary ADR)**: Honored and completed: that ADR fixes linkme-default + the binary-crate build.rs owning force-link + expected-crate manifest + the conditional tier map + R1 auto-config; THIS ADR supplies the exact pieces it deferred to the cross-crate keystone — the ExclusionSet mechanism + typo guard, the ExpectedManifest/SourceTag/anchor format + AntiDceError shape, the SequenceKey determinism total-order, and the static/dynamic epoch precedence. The scan!/main macros stay thin (emit use-as-_, const ExpectedManifest, const exclude list — all hand-writable).
- **Conditional Strategy + Profiles + Auto-Config Back-off (register §8)**: Exclusion sits ABOVE back-off in one deterministic precedence: exclude(ContractId) > user-bean-back-off(OnMissingBean) > auto-config-default. An excluded ContractId mints no BeanId and never enters the R1 deferred back-off batch. A starter feature both gates compilation (cfg) and auto-adds its crate to the force-link participating set (linkage) — both required (§352). cfg-tier (OnClass/OnCapability) conditions MUST pair with force-link or the gated-in row silently vanishes (couples conditions directly to Defense 0).
- **Unified Error / Diagnostics Model (register §6)**: Three timing-separated, source-naming errors this ADR mints feed the unified model: AntiDceError::SourceVanished (assembly-time, names crate + VanishReason hint), ExclusionError::NoSuchTarget (assembly-time, typo guard), and the silent-by-design condition/exclusion report entry (NOT an error). The self-check is the realization of the §6 'anti-DCE missing-bean vs loud collision' distinction; leaf doctor lists discovered SourceTags + per-source row counts so a zero-contribution crate is visible before it fails.
- **WASM Optional Layer / Two-Source Duality (wasm exploration, dropped for v1)**: The static linkme/build-codegen slice (epoch 0) is the default classpath; the optional directory-scanned WASM source is the dynamic second classpath via reopen_epoch→re-freeze (epoch≥1), registering identical origin-agnostic Descriptor+Provider, keyed on ContractId, deterministically sorted AFTER all static rows. This ADR specifies only the SEAM and the static-precedes-dynamic ordering invariant; the WASM runtime, trap/teardown semantics, in-flight reconciliation, and dynamic-vs-static precedence-beyond-ordering are Phase-3 WASM-local (shared with the registry-substrate ADR).
- **Container Refresh Lifecycle & Compile-vs-Runtime Split**: Forces a definite progression: link-time collect → build-time (force-link + ExpectedManifest + optional plan) → around-main lift → self_check → evaluate_conditions → apply_exclusions → R1 back-off batch → freeze (SequenceKey sort, ContractId collision guard) → async eager init. Post-freeze edits are a typestate/diagnostic error; the dynamic/WASM second epoch (reopen_epoch→re-freeze) is the ONLY sanctioned later mutation. Refresh-lifecycle must expose this definition→seal→(optional dynamic epoch)→steady-state shape.
- **Crate Topology & BOM/Starter Structure (charter §6, owner decision #3)**: Forces the strict inward DAG: leaf-macros (proc-macro, host) → leaf-core (registration ABI + COMPONENTS/SOURCES slices + RegistryBuilder) → integration crates (depend on leaf-core ONLY, never the umbrella, to avoid cycles) → leaf umbrella facade (re-exports + force-links enabled integrations). BOM analogue = [workspace.dependencies] (single copy of core/macros/linkme/inventory, preventing duplicate registries) + version-pinned `leaf` umbrella downstream. A starter = dep:-hidden umbrella feature over an integration crate.
- **Bootstrap / Run-Participant Registries (07-bootstrap: runners, listeners, initializers, analyzers, deducers)**: ALL run-participant streams (RunListeners, ContextInitializers, Runners, FailureAnalyzers, AppTypeDeducers) share THIS one substrate — each is another typed distributed_slice (codegen-boundary ADR), force-linked, self-checked, ContractId-keyed, SequenceKey-ordered. The acute asymmetric hazards bootstrap flags (a DCE'd early RunListener is silently absent with no resolution failure; a DCE'd migration Runner = data corruption; a DCE'd analyzer = a quietly-worse message) are exactly why the expected-vs-found self-check + leaf-doctor listing are mandatory and non-negotiable here. No bespoke per-stream registry.
- **Autowiring / Candidate Resolution & Collection Injection**: Consumes the frozen registry's SequenceKey-ordered candidate sets: collection-injection insertion order and equal-@Order tie-break read the deterministic (epoch, ContractId) order, NOT link order. The @Order/@Priority comparator (register §11) breaks final ties by SequenceKey so assembly is reproducible. Exclusions remove candidates before resolution (an excluded ContractId is not in any candidate set); collection-injection expected-count self-checks (register §7b) reuse the per-SourceTag row counts this ADR's manifest already computes.
