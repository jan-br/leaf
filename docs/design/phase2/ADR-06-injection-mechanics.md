# ADR 06 — Injection / Resolution Mechanics & Injection-Point Shapes  `[injection-mechanics]`

> **Note (2026-06-14):** the optional *dynamic-WASM source / WASM-"classpath"* mentioned below is **DROPPED** — leaf autodiscovery is native link-time registration only. Ignore WASM/dynamic-module references here; they are struck from the active design.


## Decision
ADOPT a SINGLE resolution spine in leaf-core with four locked sub-decisions, all built on the upstream ownership-model (`ErasedBean`/`Ref<T>`/`Published`/`Provider`), container-shape (`Engine`, the renamed consumer family `Lookup`/`LazyRef`/`Inject` + the one `resolve(Strictness,Cardinality)` driver), and async-context-model (BoxFuture-at-dyn, ambient `Cx`).

(A) THE RESOLUTION ALGORITHM = a FIXED-ORDER, FAIL-FAST, TRACED LAYER FOLD (Phase-1 R2 shape), keyed where it can be (R3 fold-in for the candidate SET, never for selection ORDER). Split exactly as Spring does: candidate-resolver FILTERS (produces the qualifying set), this spine SELECTS the single winner and arbitrates 0/1/many. The selection ORDER is a fixed kernel contract — NOT pluggable. Concretely in leaf-core:
- `Selector::resolve_one(ip: &InjectionPoint, set: CandidateSet<'_>) -> Resolved<'_>` is a single generic fold over a `const LAYERS: &[Layer]` array. The order is the literal Spring contract: `[TypeMatch (the set's precondition) → GenericNarrow → QualifierNarrow → PrimaryPromote → NameMatch → QualifierNameMatch → PriorityRank → DefaultCandidate → ResolvableDependency]`. Each layer is a pure fn `fn(&InjectionPoint, &[Cand<'_>]) -> Verdict` with `enum Verdict<'a> { Unique(&'a Cand<'a>), Narrowed(SmallVec<[&'a Cand<'a>; 4]>), Abstain }`; the driver short-circuits on the first `Unique`/narrow-to-one, threads the narrowed set otherwise, and applies the SINGLE fail-fast rule in ONE place: after all layers, `len 0 → None`, `len 1 → One`, `len >1 → Ambiguous`. `enum Resolved<'a> { One(&'a Cand<'a>), None, Ambiguous(SmallVec<[&'a Cand<'a>; 4]>) }` makes refusing-to-guess a non-coercible type, not a convention.
- FAIL-FAST is a startup-validation pass over the frozen registry (run inside `Context::refresh()`, before eager instantiation), NOT compile-time and NOT first-resolution: it resolves every eager single-valued injection point and aggregates all `None`/`Ambiguous` verdicts into one `AssemblyReport` (charter §2.9 richest-context-wins; ambiguity needs the whole candidate set + crate provenance, which only exists post-link). Lazy/scoped/prototype points fail-fast at first acquisition with the identical `ResolveError`.
- QUALIFIERS are the hybrid the Phase-1 tension demands: a `Qualifier` is a typed zero-sized marker (`#[qualifier] struct Primary;`) carried as a marker-`TypeId` AND an open string (bean-name / config-valued `@Qualifier("x")`). The candidate-resolver keys the SET by `(produced TypeId, [generic-arg TypeIds], [qualifier-marker TypeIds])` so type+generics+marker-qualifier narrow by index lookup (R3 speed, free generics-as-qualifier via distinct concrete `TypeId`s), then the string layers (NameMatch / QualifierNameMatch) run inside the fold for the irreducibly-stringly bean-name-as-implicit-qualifier behavior. Marker-qualifier mismatch is a typed key miss surfaced at the refresh validation pass; a typo'd marker TYPE is a plain compile error (it does not name a type) — the only compile-time arm.
- The CONSUMER family (`Lookup<T>`/`LazyRef<T>`/`Inject<T>` from container-shape) wraps this same `Selector` through the one `resolve(Strictness, Cardinality)` driver: `Single` cardinality calls `resolve_one`, `Multiple` collects-all-then-orders (bypassing selection), and `Strictness` maps `None`/`Ambiguous` onto `Result`/`Option` (`get` strict, `get_if_available` tolerates absence but errors on ambiguity, `get_if_unique` tolerates both).

(B) THE @Order/@Priority COMPARATOR = ONE owner in leaf-core, not triplicated. `leaf_core::order` owns `pub struct OrderKey { value: i32, source: OrderSource }` and `enum OrderSource { Interface, Annotation, Implicit }`, captured per descriptor by the macro, plus the canonical total comparator `fn cmp_order(a: &OrderKey, b: &OrderKey) -> Ordering` (lower value wins; an `Ordered`-trait-implemented signal — `OrderSource::Interface` — beats a declared `#[order(n)]`/`#[priority(n)]` annotation at equal value; `#[priority]` is struct-level-only, modeled as `OrderSource::Annotation` that factory-method beans cannot carry). It is consumed by: autowiring tie-break `PriorityRank` layer, collection/array/map injection ordering, `Lookup::ordered_stream`, runtime-lifecycle phase ordering, and the auto-proxy advisor chain. There is exactly ONE comparator; every site calls `cmp_order`. (Resolves Cluster 11 "ONE canonical comparator" and Cluster 10 "the ONE comparator".)

(C) CONSTRUCTION MODEL = SINGLE-PHASE, jointly with the circular-ref posture already fixed by ownership-model. A bean is built by ONE total async factory fn whose typed parameters ARE the injection points (Phase-1 injection-styles R1). There is NO two-phase construct-then-populate, NO `&mut self` populate hook, NO early half-built exposure. The `Provider::provide -> BoxFuture<Published>` seam constructs-and-publishes atomically; `Published::Shared` for singleton/scoped, `Published::Owned` for prototype. CONSEQUENCE on cycles (the joint constraint): because there is never a half-built instance, constructor cycles are LOGICALLY FATAL (reported at the refresh plan/validation pass with the full path) and setter/field-cycle tolerance is NOT offered — the ONLY cycle break is an explicit deferral edge (`Lookup<T>`/`LazyRef<T>`), exactly the ownership-model R3 posture (deferral edge removed from the construction graph, resolves the fully-built target post-build, Weak<ContainerCore> back-ref, no early erased handle to retroactively wrap). This is consistent with Spring Boot's own `allow-circular-references=false` default and with AOT contexts; leaf has ONE cycle behavior everywhere.

(D) INJECTION-POINT SHAPES = the factory-fn PARAMETER SET is the ONLY injection surface. There is NO post-construction typed-param method hook (no `inject(&mut self, ..)`). Constructor injection is the mechanism; setter/field are macro SUGAR that desugar to constructor params (mandatory = required param `arity=Single`; optional/setter = `Option<T>` param or `#[inject(default)]` `arity=Optional`; field = `#[inject] field` synthesized into the same `build`). Aware/infrastructure facts arrive AS injection-point parameters too (resolvableDependencies as the lowest-precedence `ResolvableDependency` layer over a registry-owned `HashMap<TypeId, ErasedBean>`), not via a post-construction setter. Each point is a const `InjectionPoint { produced: TypeId, generics: &[TypeId], qualifiers: &[QualifierReq], name: &str, arity: Arity, kind: Bean|Value|Deferral }` row on the per-bean `InjectionPlan`, emitted by the thin macro via absolute `::leaf_core` paths. `Lookup`/`LazyRef`/`Inject`/`Option<T>` are injection-point KINDS on this same plan (a Deferral kind removes the construction-time edge — the cycle break). Self-injection is the explicit `SelfRef` deferral handle (Phase-1 R-A), never an implicit self-candidate. Lookup-method injection folds entirely into `Lookup<T>` (Phase-1 R1/R3): no abstract-component admission gate, no second resolver handle.

## Rationale
- DX#1 + unification (charter §2.1/§2.11): the traced layer-fold (B-of-Phase-1-R2) gives the SAME fail-fast 'refuse to guess' as a hand ladder but puts the no-guess rule in exactly ONE place (the driver's terminal len>1→Ambiguous), decomposes the high-correctness-burden selection logic into independently-testable pure-fn layers, and produces the candidates-considered narrative as a byproduct of the fold — which is precisely the rich-context account charter §2.9 demands. A hand-written ladder (R1) risks the no-guess guarantee being half-forgotten in one branch and reconstructs diagnostics after the fact.
- Keying the SET (not the selection ORDER) by (TypeId, generic-arg TypeIds, qualifier-marker TypeIds) banks R3's wins where they are sound — free generics-as-implicit-qualifier (distinct concrete TypeIds are distinct keys), monomorphized index lookup on the warm prototype/scoped re-resolution path (charter §2.2/§2.4), and compile-time death for a typo'd marker TYPE — WITHOUT R3's fatal flaw: @Fallback is count-and-cross-module-dependent and the bean-name implicit qualifier is irreducibly stringly, so those CANNOT be static keys. Putting only filtering in keys and keeping selection a fixed runtime fold over the post-link frozen set preserves full Spring parity for the cross-module override story while still being fast on the common path.
- Fixing fail-fast at the refresh startup-validation pass (not compile, not first-resolution) is forced by owner-decision #1 (maximal-magic discovery, no declared scan-scope): ambiguity and NoSuchBean are only knowable once the whole link-collected candidate set exists, and the richest diagnostic (full candidate set + per-crate provenance + per-layer elimination reasons) requires the post-link frozen registry. This is the same gate container-shape already mandates for the anti-DCE expected-vs-found self-check (cross-crate §152), so the validation pass is shared, not new.
- ONE OrderComparator with an OrderSource axis is the literal answer to the register's triplication flag (Cluster 11) and the auto-proxy 'ONE comparator' flag (Cluster 10). Capturing Interface-beats-Annotation as data (OrderSource) is the only way to express Spring's AnnotationAwareOrderComparator precedence in Rust, which has no runtime 'does it implement Ordered' check — and centralizing it guarantees single-valued tie-break, collection ordering, ordered_stream, lifecycle phases, and the advisor chain can never diverge (the exact divergence the Phase-1 notes warned about).
- Single-phase construction is forced by the upstream ownership-model, not re-decided here: that ADR already chose Provider::provide→Published (construct-and-publish atomically), the deferral-edge cycle break with a Weak back-ref, and 'constructor cycles remain fatal at the plan/startup pass; there is NEVER an early erased handle to retroactively swap for a wrapped one.' A two-phase populate hook would re-introduce a mutable half-built bean, demand interior mutability/UnsafeCell and a populate-window lock (charter §2.5 violation), and reopen the raw-vs-wrapped allowRawInjectionDespiteWrapping hazard that has no sound Rust answer. Single-phase makes immutability and safe-publication free (the Arc clone IS the happens-before edge) and cancellation leak-free (one awaitable, no half-populated shell).
- The joint circular-ref↔injection-styles constraint the register flagged is RESOLVED by observing they are one decision: choosing single-phase construction (D) determines that setter/field cycle tolerance is structurally impossible (C). Rather than pay R2's mutable-two-phase cost for a behavior Spring Boot disables by default and AOT forbids, leaf elevates the deferral handle (which it needs anyway for laziness/optionality — charter §2.11) to the ONE cycle break, yielding identical runtime and AOT behavior and an actionable diagnostic ('convert injection point X from T to Lookup<T>').
- Factory-fn-params-as-the-only-surface (D) is the maximal-unification reading (charter §2.11): autowiring-resolution sees a uniform InjectionPlan of typed points regardless of whether the user wrote a constructor param, a #[inject] field, an Option, a Lookup, or a @Value — one resolution spine, one plan shape, one erased async-factory descriptor that is origin-agnostic (native/FactoryBean/WASM/test-double all present the identical Provider per ownership-model). A post-construction typed-param hook would be a SECOND injection surface with its own ordering, &mut-dyn-Any populate seam, and cancellation/teardown states — exactly the bespoke-per-feature machinery the charter resists.
- Thin macros (charter §2.10): every emitted artifact — the InjectionPlan const rows, the build factory that calls the user's hand-written new(..), the OrderKey, the qualifier-marker TypeId list — is data or a one-line forwarder a human could write; cargo expand shows the literal constructor call. The Selector fold, the layers, and resolve(Strictness,Cardinality) live in leaf-core as normal testable code, never in a macro.

## Options considered & rejected
- **Resolution as a hand-written fixed-ladder total function (Phase-1 autowiring R1)** — rejected: Highest Spring fidelity but bundles the entire high-risk selection surface into one long short-circuiting function whose early-shortcut-equals-slow-path invariant is a silent-wrong-injection bug class (the exact thing Spring took point releases to stabilize), and it reconstructs the candidates-considered diagnostic after the fact rather than as a byproduct. The traced fold gives identical fixed-order semantics, the same non-pluggable contract (a const LAYERS array), better diagnostics, and decomposed testability — strictly better against charter §2.9 at equal warm-path cost (the trace is gated to the >1 path).
- **Pure typed-key disambiguation as the PRIMARY selection model (Phase-1 autowiring/candidate-resolver R3)** — rejected: Banks compile-time qualifier safety and warm-path speed but cannot represent the bean-name-as-implicit-qualifier (a string used at BOTH the early shortcut and the late tie-break), cannot represent @Fallback (count-and-cross-module-dependent — a library default in crate A superseded by an app bean in crate B, compiled separately), and cannot represent config-valued @Qualifier — all of which spill to a runtime ladder ANYWAY, making R3 alone MORE total machinery (a key probe AND a residual ladder) with a dual failure-timing split. We keep R3's keying for the SET (filtering) where it is sound and a single runtime fold for SELECTION.
- **Two-phase construct-then-populate with a post-construction typed-param inject(&mut self) hook (Phase-1 injection-styles R2)** — rejected: The only model that could break setter/field cycles, but it re-introduces a mutable partially-constructed bean (fights Rust ownership + charter §2.5), needs interior mutability/UnsafeCell and a populate-window lock for safe publication, reopens the raw-vs-wrapped AOP-early-proxy hazard with no sound Rust answer, and complicates cancellation (a constructed-but-unpopulated shell must be evicted). It directly contradicts the upstream ownership-model's atomic Provider→Published seam and its 'no early erased handle to swap' posture. Spring itself disables cycle tolerance by default and AOT forbids it — paying this cost for a discouraged behavior is the wrong ROI.
- **Derived type-state builder as the construction/injection surface (Phase-1 injection-styles R3)** — rejected: Best compile-time mandatory-slot encoding on the hand/test path, but the container resolves deps dynamically at runtime by TypeId and cannot ride a compile-time type-state progression — so the headline benefit applies only to the hand path while the container path uses a flat factory anyway, and it emits the heaviest macro output (combinatorial marker generics, intimidating errors) straining charter §2.10. The single-phase flat factory (R1) gives the same immutability and origin-agnostic descriptor with a thin macro; the type-state builder can return as a Phase-3 hand-facing convenience without being the injection surface.
- **Three separate @Order/@Priority comparators (the triplicated status quo across autowiring, collection-injection, deferral)** — rejected: The register explicitly flags this as a coherence risk: three copies WILL drift, so single-valued tie-break ordering, collection ordering, and ordered_stream ordering could disagree on the same beans. One leaf_core::order::cmp_order with an OrderSource axis is the only way to guarantee they never diverge and the only place Interface-beats-Annotation can be expressed once.
- **Implicit lowest-precedence self-candidate for self-injection (Phase-1 self-injection R-B)** — rejected: Maximal Spring parity but couples resolution to the creation-driver's in-creation set (a runtime guard during construction), risks a strong Arc-of-Self leak, and makes the demotion invisible. The explicit SelfRef deferral handle (R-A) is origin-agnostic (one descriptor flag, no self-registry), reuses the deferral family already chosen, avoids the Arc-self-cycle, and keeps self never beating a real collaborator — consistent with single-phase construction (no early-reference machinery to hook).
- **A bespoke lookup-method (@Lookup) mechanism / second resolver handle (Phase-1 lookup R2's LookupCtx)** — rejected: Introduces a SECOND deferral-ish primitive distinct from the consumer family, fragmenting the toolkit against charter §2.11, and re-homes an abstract-component admission gate that has no Rust analogue (no abstract-class subclassing). Folding @Lookup entirely into Lookup<T> (R1/R3) reuses one primitive, deletes the dangling scanner gate (a concrete struct with a Lookup field always has a construction recipe), and serves the fresh-per-call intent honestly via per-call resolution against the prototype scope.

## Consequences
- FORCES candidate-resolver (Cluster 2 substrate) to materialize the qualifying set as a keyed lookup over (produced TypeId, generic-arg TypeId tuple, qualifier-marker TypeId set) PLUS the three-valued Match{Yes,No,Abstain} fold for the residual stringly dimensions; it produces the set, never selects. The bean-index space backing CandidateSet must be dense and stable across static + dynamic (WASM reopen_epoch) registration so the SmallVec working set and per-layer trace stay allocation-light on the >1 path only.
- FORCES the macro/codegen boundary (Cluster 3) to emit, per bean, a const InjectionPlan: ordered InjectionPoint rows {produced TypeId, generic-arg TypeIds, qualifier-marker TypeIds + string qualifier reqs, declared param/field name, Arity(Single|Optional), Kind(Bean|Value|Deferral|Self)} plus one OrderKey{value, source} per orderable bean, all via absolute ::leaf_core paths. A bare generic #[component] must hard-error (generics-cannot-register); concrete instantiations need register_component!(Concrete).
- FORCES the refresh-lifecycle (Cluster 12) to own a single startup VALIDATION pass that runs Selector::resolve_one over every eager single-valued point against the frozen registry, aggregates all None/Ambiguous into one AssemblyReport with per-layer elimination reasons + per-candidate crate provenance, and fails refresh all-or-nothing BEFORE eager instantiation. This pass is unified with the anti-DCE expected-vs-found self-check (a missing candidate from a DCE'd crate surfaces as NoSuchBean naming the source crate, distinct from a cycle error).
- FORCES cycle handling to be deferral-only and constructor-cycles-fatal: the construction-graph (constructor-param edges only; Deferral-kind and Lazy edges removed) is cycle-classified at the plan pass; an all-mandatory loop is a fatal CycleError naming the full path + the exact edge to convert to Lookup<T>; there is NO early-exposure hook, NO middle (singletonFactories) cache tier required for cycles, NO raw-vs-wrapped consistency check. This consumes the ownership-model's R3 posture and means the proxy/registry keystone's 'middle cache level' does not need to exist for circular references.
- FORCES the deferral family (container-shape's Lookup/LazyRef/Inject) to be the single home for: optional arity (Option<T> → Strictness), lazy injection (LazyRef = @Lazy, OnceCell-cached for singleton targets), jakarta Provider<T> parity (Inject<T>), lookup-method injection (Lookup<T>, no separate mechanism), scoped-proxy-style re-resolution, and the cycle break — all routed through the one resolve(Strictness, Cardinality) driver so getIfAvailable's tolerate-absence-but-error-on-ambiguity semantic lives in one place. A Deferral-kind point must NOT cache for prototype targets (the fresh-per-call freshness contract).
- FORCES resolvableDependencies (infrastructure handles: Engine/Context back-ref, EventPublisher, Env) to be a registry-owned HashMap<TypeId, ErasedBean> consulted ONLY at the terminal ResolvableDependency layer (lowest precedence) and folded into the by-type candidate count so ambiguity arithmetic stays correct; they are injected as ordinary factory-fn parameters (no Aware setter phase), and the container back-ref captured by any stored Lookup closure is Weak<ContainerCore> (no Arc cycle), per ownership-model.
- FORCES @DependsOn to be a pure forced-ordering edge on the construction graph (a non-injecting before/after constraint feeding the same OrderKey/topo machinery), and self-injection to be the explicit SelfRef deferral handle (one descriptor is_self flag, no implicit self-candidate, never beats a real collaborator, constructor-self fails fast) — both reuse this spine's edge + ordering primitives rather than new machinery.
- FORCES the qualifier vocabulary (Cluster 8/annotation-model) to support a typed marker (#[qualifier] zero-sized struct → marker TypeId) that must live in a crate both the declaring bean and the injection point depend on (dependencies-point-inward topology constraint), WITH an open-string fallback for bean-name and config-valued qualifiers. Custom-qualifier attribute SETS (match-ALL key/values) ride the string/Abstain path, not the typed key.
- ACCEPTS a named warm-path cost: per single-valued resolution touches the keyed index (O(matches)) plus, only on the >1 path, the layer fold + a trace SmallVec; the common len<=1 path is allocation-free and branch-only. Selection is sync/pure; winner CONSTRUCTION is the async BoxFuture step owned by Engine::create (per ownership-model/async-context-model). Constructor-injected collaborators are resolved ONCE at the holder's build and stored as typed Ref<T> fields, so steady-state method calls touch no resolution at all (cost paid at wiring time).
- ACCEPTS a real divergence from Spring's runtime tolerance of setter/field cycles (the brief's stated 'tolerates setter/field cycles'): in leaf such a cycle requires an explicit Lookup<T>/LazyRef<T> edge. This is defensible by Spring Boot's own off-by-default posture and AOT behavior, and is the price of single-phase construction + ownership soundness; the actionable diagnostic ('convert X: T to X: Lookup<T>') mitigates the DX cost.
- ACCEPTS that some fail-fast timings differ by arm (a typo'd qualifier marker TYPE is a compile error; a genuinely-ambiguous or missing bean is a refresh-validation-pass error; a lazy/scoped point fails at first acquisition) — a deliberate charter §2.9 'richest-context-wins' split, kept coherent by routing all runtime arms through the one Selector + ResolveError so the diagnostic shape is identical regardless of when it fires.

pub struct Layer { pub name: &'static str, pub eval: for<'a> fn(&InjectionPoint, &[&'a Cand<'a>]) -> Verdict<'a> }

// THE fixed order — the container contract. Filtering already keyed type+generics+marker-qualifier; these are the SELECTION layers.

    Layer { name: "generic_narrow",   eval: layers::generic_narrow },   // single-valued safety net (keyed already)
    Layer { name: "qualifier_narrow", eval: layers::qualifier_narrow },
    Layer { name: "primary_promote",  eval: layers::primary_promote },  // two-pass: unique @Primary (2 local = Ambiguous; local beats parent via hierarchy_depth) else unique non-@Fallback

    Layer { name: "qualifier_name",   eval: layers::qualifier_name },    // @Qualifier("x") == bean name/alias
    Layer { name: "priority_rank",    eval: layers::priority_rank },     // uses the ONE comparator (B)
    Layer { name: "default_candidate",eval: layers::default_candidate },
    Layer { name: "resolvable_dep",   eval: layers::resolvable_dep },    // lowest precedence: HashMap<TypeId, ErasedBean>
];

pub struct Trace(pub smallvec::SmallVec<[(&'static str, usize, usize); 8]>); // (layer, before, after) — recorded only on the >1 path

pub struct Selector;
impl Selector {
    // SYNC, pure, allocation-light on len<=1; the one place fail-fast lives.
    pub fn resolve_one<'a>(ip: &InjectionPoint, set: CandidateSet<'a>) -> (Resolved<'a>, Option<Trace>) { /* fold LAYERS, short-circuit on Unique/narrow-to-one, terminal len rule */ unimplemented!() }
}

## Forces on other concepts
- **Type-Erasure & Registry / Candidate-Resolver Substrate (Cluster 2)**: Must produce the qualifying CandidateSet keyed by (produced TypeId, generic-arg TypeId tuple, qualifier-marker TypeId set) with the three-valued Match{Yes,No,Abstain} fold for residual string dimensions; it FILTERS, never selects. Needs a dense, stable bean-index space (for the SmallVec working set + trace) that survives static + dynamic (WASM reopen_epoch) registration via COW swap on the cold path. Marker qualifiers index by TypeId; bean-name/config qualifiers index by interned name. resolvableDependencies live in a registry-owned HashMap<TypeId, ErasedBean> the terminal layer consults.
- **Injection Styles / Metadata-Codegen Boundary (Cluster 3, Cluster 11)**: Construction is SINGLE-PHASE: the macro emits ONE async factory (Provider::provide) calling the user's hand-written constructor, plus a const InjectionPlan of InjectionPoint rows and one OrderKey per orderable bean, via absolute ::leaf_core paths. Setter/field/@Lookup/Self/@Value are KINDS on that plan, not separate surfaces — there is NO post-construction inject(&mut self) hook. Bare generic #[component] hard-errors; concrete instantiations need register_component!.
- **Deferral Primitives / Lazy / Lookup-method / Self-injection (Cluster 11)**: Bind to the renamed consumer family Lookup<T>/LazyRef<T>/Inject<T> over the one resolve(Strictness,Cardinality) driver. A Deferral-kind point removes the construction-time edge (THE cycle break) and must NOT cache for prototype targets (fresh-per-call). Lookup<T> subsumes @Lookup (no separate mechanism, no abstract-component admission gate). Self-injection is the explicit SelfRef handle (one is_self flag, no implicit self-candidate). @Lazy is LazyRef (OnceCell-cached for singleton targets). The strictness/cardinality ladder lives in one place so get_if_available's semantics can't drift.
- **Circular References & Bean Instantiation (Cluster 9, ownership-model)**: No early-exposure hook, no middle cache tier for cycles, no raw-vs-wrapped consistency check. The construction graph (constructor-param edges; Deferral/Lazy edges removed) is cycle-classified at the refresh plan pass: all-mandatory loop = fatal CycleError naming the path + the exact Lookup<T> conversion; satisfiable cycles exist only via deferral edges resolving the fully-built target post-build (Weak<ContainerCore> back-ref). One cycle behavior across runtime and AOT.
- **Collection / Map Injection, Runtime Lifecycle, Auto-Proxy Creator (Clusters 10, 11, 12)**: All ordering goes through the SINGLE leaf_core::order::cmp_order over OrderKey{value, source}: single-valued tie-break (priority_rank layer), collection/array/map ordering, Lookup::ordered_stream, runtime-lifecycle phase ordering, and the auto-proxy advisor chain. No site may define its own comparator. Collection injection takes ALL qualifying candidates and BYPASSES selection (primary/fallback/default-candidate apply only to single-valued points).
- **Container Refresh Lifecycle & Cross-Crate Anti-DCE (Cluster 12, owner-decision #1)**: refresh() owns a single startup VALIDATION pass that runs Selector::resolve_one over every eager single-valued point against the frozen registry, classifies the construction-graph cycles, and aggregates all NoSuchBean/NoUniqueBean/Cycle into one AssemblyReport — unified with the expected-vs-found anti-DCE self-check so a DCE'd crate surfaces as NoSuchBean naming the missing source crate, distinct from a Cycle. All-or-nothing fail before eager instantiation.
- **Annotation / Qualifier Vocabulary (Cluster 8)**: Must provide a typed zero-sized marker qualifier (#[qualifier] → marker TypeId) that lives in a crate both the bean and the injection point depend on (dependencies-point-inward), PLUS an open-string fallback for bean-name and config-valued @Qualifier. Custom-qualifier attribute SETS (match-ALL) ride the string/Abstain path. A typo'd marker TYPE is a compile error; a marker key miss is a refresh-pass NoSuchBean.
- **Async Execution & Context (async-context-model)**: Selection is SYNC/pure over immutable-after-refresh data (lock-free read). Winner construction is the async BoxFuture Provider::provide owned by Engine::create; resolve(..).await happens on the caller's task and reads the ambient Cx via the poll-re-installing combinator (not ThreadLocal). A Deferral handle's first-call resolution is reactive/waker-driven (charter §2.4); cancelling a prototype build leaks nothing (owned-move, no teardown).
