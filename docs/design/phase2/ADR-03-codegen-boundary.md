# ADR 03 — Metadata / Codegen Boundary  `[codegen-boundary]`

> **Note (2026-06-14):** the optional *dynamic-WASM source / WASM-"classpath"* mentioned below is **DROPPED** — leaf autodiscovery is native link-time registration only. Ignore WASM/dynamic-module references here; they are struck from the active design.


## Decision
ADOPT a FOUR-LAYER pipeline, one sharp rule per layer — "emit data never logic; collect by link; assemble at runtime; codegen only at the binary crate."

LAYER 1 — PROC-MACRO EMITS DATA ONLY (thin, §2.10). leaf-macros (proc-macro=true, host target, exports only macros) expands #[component]/#[bean]/#[configuration]/#[auto_configuration]/#[conditional(..)] into exactly ONE hand-writable, fully-const Descriptor row (the registry-substrate ADR's Descriptor) plus a const fn-constructed ProviderSeed, registered into a linkme distributed slice via absolute ::leaf_core paths. The macro computes self_type (const{TypeId::of::<T>()}, 1.91), ContractId (a const fn FNV/Fx hash over the author-stable identity string), provides[] upcast rows (one TypeRow per declared injectable supertrait, trait-upcasting 1.86), the flat AnnotationMetadata (qualifiers/markers/condition CondExpr tree as const data), and ScopeDef. ALL merge/index/freeze/condition-eval/order/back-off logic lives in normal testable leaf-core/leaf-boot crates — the macro never accumulates state across invocations, never hides logic, cargo expand shows one legible const. What the macro emits equals what a user could write by hand to the public RegistryBuilder::register(d,p) — the always-present escape hatch.

LAYER 2 — LINK-TIME REGISTRATION via linkme (default, server targets). The const Descriptor/ProviderSeed rows collect into &'static [Descriptor] from ::leaf_core::COMPONENTS (and sibling typed slices CONDITIONS, AUTO_CONFIGS, LISTENERS, CONVERTERS — ONE substrate, many slices) with ZERO life-before-main and zero runtime collection cost. inventory is the reserved fallback for the rare non-const registered value (typetag-style trait-object self-registration), NOT the default. The substrate is swappable per target behind a uniform &'static [Descriptor]: on wasm/embedded/staticlib the binary uses LAYER-3 build-codegen to emit the same flat array (Architecture C); the universal escape hatch is explicit RegistryBuilder::register/App::with(plugin) (Architecture D). Toolchain guardrails are mandated: MSRV >= 1.89 (ELF #[used(linker)] default) OR a generated .cargo/config.toml -Clink-arg=-Wl,-z,nostart-stop-gc; linkme >= 0.3.24 pinned in [workspace.dependencies].

LAYER 3 — build.rs / cargo leaf prepare CODEGEN, ANCHORED ONLY AT THE BINARY CRATE (the sole point seeing the whole crate graph). #[leaf::main] + a binary-crate build.rs (or leaf-codegen as a cargo leaf prepare CLI) do THREE jobs only the final binary can: (a) FORCE-LINK — emit use leaf_tokio as _; for every opted-in integration crate so its rlib is actually linked (defeats DCE Layer 0); the opted-in set is the union of dep:-hidden capability features and an explicit leaf::scan!{ leaf_tokio, leaf_redis } list; (b) EXPECTED-CRATE MANIFEST — read cargo metadata and emit a const EXPECTED the runtime self-check cross-references, so a silently-DCE'd crate fails LOUDLY (owner-decision #1's load-bearing anti-DCE check); (c) OPTIONAL build-time auto-config PRE-PLAN — cargo leaf prepare MAY run the build-decidable half of auto-config assembly (union+dedup+exclusion-typo-guard+build-decidable condition prune+topo-sort) and emit a checked-in, --check-able autoconfig_plan artifact (Architecture C) for determinism/native-image/wasm — OPT-IN, not the default. build.rs does NO cross-crate scanning of foreign annotations (single-crate wall); contributor crates emit their own linkme rows in their own compilation.

LAYER 4 — RUNTIME ASSEMBLY in leaf-core/leaf-boot, around (never before) main. RegistryBuilder::from_slices(COMPONENTS, PROVIDERS, ...) lifts the flat const rows, runs the anti-DCE expected-vs-found self-check, evaluates CondExpr conditions, runs the deferred auto-config back-off batch, then freeze() materializes the dense BeanId registry (registry-substrate ADR). NOTHING async/IO/order-dependent happens at link or pre-main; it is one cold, synchronous, single-threaded assembly pass — async construction begins only after freeze, via the boxed-future Provider seam.

CONDITIONAL STRATEGY (the per-kind tier map this boundary owns as a kernel ABI). A CondExpr leaf declares an EarliestTier; the macro/codegen routes each kind to exactly one: TIER cfg (compile-time, OnClass/OnCapability) emitted as #[cfg(feature="redis")] ON the Descriptor row — an absent optional dependency physically does not compile, so naming it can never error; capability-true MUST also force-link (LAYER 3a). TIER const-fold (build-time, target/build-config-knowable: OnCloudPlatform/OnRustVersion/OnThreading-platform) folded by leaf-codegen/build.rs into meta as CondExpr::Const(bool). TIER runtime (OnBean/OnMissingBean/OnProperty/profiles, AND any dynamic/WASM-contributed definition with no cfg and no Rust type) — a Condition predicate over the frozen registry view in LAYER-4's deferred phase. Profiles are ALWAYS runtime (the coherence constraint). The runtime tier is MANDATORY (origin-agnosticism), so cfg/const are fast-PATHS over an always-present runtime planner. CondExpr.phase() is framework-INFERRED as max(leaf tiers) (no author phase declaration; an OnBean nested anywhere auto-defers the whole guard to the Register pass).

AUTO-CONFIG ASSEMBLY (owner-decisions #2/#3). R1 runtime-deferred-batch is the DEFAULT engine (faithful to opinionated-but-overridable back-off, origin-agnostic, substrate-neutral); the R3 build-time-frozen plan is the OPT-IN cargo leaf prepare mode for determinism/wasm. A "starter" = a dep:-hidden umbrella Cargo feature over an integration crate that (a) gates compilation of its #[auto_configuration] rows AND (b) triggers LAYER-3a force-link — both required because cfg gates compilation not linkage. The BOM analogue = [workspace.dependencies] internally for version unification + a version-pinned umbrella facade crate (leaf) downstream that re-exports core+macros+enabled integrations and force-links them.

## Rationale
- THIN MACROS (§2.10) is satisfied at the strongest altitude: the macro emits exactly ONE const Descriptor + one const ProviderSeed via absolute ::leaf_core paths — literally what a user writes by hand to RegistryBuilder::register. The cross-crate reference proves a proc-macro CANNOT build a cross-crate registry anyway (single-invocation scope, no state across invocations, no sibling visibility), so 'emit one hand-writable artifact and nothing more' is the only correct shape. All merge/index/freeze/condition/order/back-off logic lives in normal testable crates; cargo expand stays legible.
- It is the ONLY substrate the confirmed-stable envelope makes both zero-overhead AND origin-agnostic with zero nightly (§2.8). The nightly-tradeoffs doc verifies const TypeId::of (1.91), trait upcasting (1.86), the const-eval baseline (inline const{}, const fn hashers, const{assert!}), and linkme/inventory as STABLE — so the whole emitted const Descriptor is const-constructible and link-collectible on stable. linkme is the explicit cross-crate recommendation (zero runtime cost, no life-before-main, async-init-compatible, thin/data-only macros, no global locks).
- It honors OWNER DECISION #1 (maximal-magic discovery, no declared scan-scope) by making robust anti-DCE LOAD-BEARING exactly where the cross-crate reference says it must be: at the binary crate. The build.rs force-link (defeats DCE Layer 0: an unreferenced rlib is not linked), the >=1.89 / -z nostart-stop-gc guardrail (defeats Layer B), and the runtime expected-vs-found self-check (turns a silent empty registry into a loud NoSuchBean naming the missing source) are the three independent defenses the doc proves are each necessary. The registry being RECOGNITION not VALIDATION (registry-substrate ADR) is precisely why the self-check is non-negotiable.
- It honors OWNER DECISIONS #2 and #3 without re-opening them: the R1 runtime-deferred-batch engine is the default because back-off correctness REQUIRES a definite 'all user defs registered' moment that only a runtime deferred phase gives (Spring's DeferredImportSelector intent); a starter is the cross-crate doc's endorsed dep:-hidden umbrella feature + mandatory force-link; the BOM analogue is [workspace.dependencies] + a pinned umbrella facade — exactly the doc's realistic topology (Architecture B for servers, C for wasm, D as universal escape hatch).
- It keeps the cfg/const/runtime conditional tiers a COHERENT kernel ABI and resolves the 'likely a blend' the charter foresaw: cfg-gating is the only honest rust-native OnClass (a linked type is always present; an absent optional dep simply doesn't compile), build-decidable conditions const-fold, and OnBean/profiles/dynamic-WASM are intrinsically runtime. Making the runtime tier MANDATORY is forced by origin-agnosticism — a dynamic/WASM definition has no cfg and no Rust type, so it can ONLY be gated at runtime; cfg/const are accelerators over it. Profiles-always-runtime is the stated coherence constraint, preserved.
- It is substrate-SWAPPABLE behind one &'static [Descriptor] shape, the honest answer to the platform fragility the doc documents (LLD vs GNU ld, macOS-without-LTO, wasm has no link sections, staticlib-into-C). linkme is the server default; build-codegen (cargo leaf prepare emitting the same flat array) is the wasm/embedded/native-image path; explicit register/with is the universal fallback. The runtime API is identical across all three, so leaf never promises 'passive classpath scan everywhere' (not reliably achievable on stable across all targets) yet still delivers the maximal-magic feel on dominant server targets.
- It confronts overhead honestly (§2.5): link-time collection is zero-cost; the ONLY runtime cost is the one-shot cold assembly pass (lift rows, eval conditions, deferred back-off batch, freeze into the dense BeanId space) happening once around main, never on a hot path. The optional cargo leaf prepare build-time plan moves even the auto-config union/dedup/sort/build-prune to build time for determinism-sensitive deployments — the truest analogue of Boot's spring-autoconfigure-metadata fast-path — without forcing that machinery on every user.

## Options considered & rejected
- **Pure build.rs codegen as the DEFAULT collection engine (Architecture C everywhere, sqlx model)** — rejected: Maximally portable/deterministic, but the cross-crate reference proves a build.rs sees ONLY its own crate (info flows strictly downward; the kernel's build.rs can never enumerate its dependents). So foreign #[component]s are invisible to the binary's build.rs unless every contributor emits a build-time manifest by an unenforced convention — a forgetting contributor is silently absent. That trades the link-time DCE hazard for an equally-silent missing-manifest hazard while ADDING a mandatory build step and OUT_DIR hygiene for the common server case. Demoting it to the OPT-IN wasm/embedded/determinism path (cargo leaf prepare) captures its real upside (portability, --check-able artifact, native-image) exactly where linkme fails, without taxing the dominant target. Maximal-magic wants 'add a crate dependency and its components appear' — build.rs codegen cannot deliver that for a crate the binary's build doesn't already know to ask about.
- **inventory (life-before-main constructors) as the default substrate** — rejected: More DCE-robust and wasm-nominal than linkme, but life-before-main is hostile to async-first (§2.3): constructors run before main on an unknown thread, must not panic, cannot run async, std possibly not fully initialized, order unspecified. leaf's model is descriptors+factories assembled by an async runtime AFTER main, so a pre-main cost and global-constructor safety surface buys nothing the zero-cost linkme slice doesn't already give — and linkme's const-only constraint is a FEATURE (it forces the thin-data-only macro the charter demands). inventory is correctly reserved for the rare non-const registered value, not the default.
- **Runtime annotation MERGE / late-overlay (macro emits raw attributes; leaf-core merges meta-annotations and conditions on demand at runtime)** — rejected: Maximizes openness to dynamic/profile/WASM overlays but pays a per-element runtime merge cost on the cold path AND defeats earliest-richest diagnostics (§2.9): a malformed CondExpr or @AliasFor misconfig becomes a startup surprise instead of a compile/build error with a span. The annotation-model + registry-substrate ADRs already fix flat const AnnotationMetadata frozen at compile time; raw-attributes-merged-late would contradict them. The chosen design freezes the merge into const data at macro time (compile-time-legible errors) while STILL letting dynamic/WASM sources contribute their own already-merged Descriptor rows at runtime via RegistryBuilder.reopen_epoch — openness without the per-element runtime merge tax.
- **Build-time-frozen auto-config plan (R3) as the DEFAULT auto-config engine** — rejected: Lowest startup cost and earliest diagnostics, but it PRE-DECIDES the compile-vs-runtime split too hard and, fatally, cannot host the back-off correctness owner-decision #2 requires: OnMissingBean must see the FINAL user-bean set, inherently runtime, so even R3 keeps a runtime residual. It also cannot see a dynamic/WASM auto-config (no build presence), reintroducing a runtime engine for the dynamic case. The frozen-batch/runtime-residual seam is a new confusion surface ('why didn't my build plan change when I set the exclude property?'). Keeping R1 as the default preserves the opinionated-but-overridable soul faithfully and origin-agnostically; offering R3 as an OPT-IN cargo leaf prepare mode gives determinism/wasm/native-image users the fast-path without making the seam mandatory.
- **A heavier macro that does the merge/index/condition-fold itself (fat macro, fewer crates)** — rejected: Directly violates THIN MACROS (§2.10) and is technically impossible cross-crate: a proc-macro has single-invocation scope, no reliable state across invocations (rustc may or may not reuse the process, order unspecified), no sibling/downstream visibility — so it physically cannot build the merged cross-crate registry. Heavy macro logic also makes cargo expand illegible and hides behavior from the hand-writable API, breaking the escape-hatch guarantee. The decision puts all heavy logic in normal testable crates because that is both the charter mandate and the only sound option.

## Consequences
- FORCES leaf-core to host an ultra-stable, minimal registration ABI that emitted code hard-codes absolute paths to workspace-wide: Descriptor, TypeRow, ProviderSeed, ContractId, contract_hash (const fn), ScopeDef, AnnotationMetadata, CondExpr, the COMPONENTS/PROVIDERS/CONDITIONS/AUTO_CONFIGS slices, and a re-exported ::leaf_core::linkme path (so the macro references linkme through leaf-core, never assuming the user has linkme). Any change is a workspace-wide breaking change — they must be frozen early and versioned conservatively (registry-substrate ADR already commits to this).
- FORCES a strict crate topology: leaf-macros (proc-macro=true, exports only macros, host target, NO runtime API) separate from leaf-core (registration ABI + linkme slices + RegistryBuilder), separate from leaf-boot (auto-config engine + run pipeline) and leaf-codegen (build.rs/cargo-leaf-prepare logic). Integration crates depend on leaf-core, never the umbrella. The umbrella facade `leaf` re-exports and force-links. Dependencies point inward; no cycles. Matches charter §6 exactly.
- FORCES the binary crate (via #[leaf::main] + build.rs) to own three responsibilities the kernel structurally cannot: force-link references (use crate as _;) for every opted-in integration crate, the expected-crate manifest for the anti-DCE self-check, and (optionally) the cargo leaf prepare build-time auto-config plan. The kernel can never force its plugins to link. This makes 'which crates participate' an explicit, scoped, predictable decision (feature set + scan! list), mitigating the implicit-global-injection concern per owner-decision #1 without requiring per-component declaration.
- FORCES the conditional per-kind tier map (cfg / const-fold / runtime) into the kernel ABI so macros across the workspace agree on what is static vs deferred. cfg-tier conditions MUST be paired with force-link or the gated-in registration silently vanishes. The runtime tier is mandatory (origin-agnosticism); profiles always runtime. CondExpr.phase() is framework-inferred (max over leaves), eliminating the forgot-to-declare-REGISTER footgun. Couples the codegen boundary tightly to the cross-crate substrate (cfg+force-link) and the registry freeze ordering (Register-phase reads the complete definition set).
- FORCES toolchain guardrails as a shipped requirement, not advice: MSRV >= 1.89 for the ELF #[used(linker)]/SHF_GNU_RETAIN default, OR a generated .cargo/config.toml injecting -Clink-arg=-Wl,-z,nostart-stop-gc; linkme >= 0.3.24 pinned in [workspace.dependencies] (RUSTSEC-2024-0407); resolver = '3' set explicitly at the workspace root (so sibling/test/build features don't leak a capability into the production component set). CI MUST test the final-binary link (release + LTO + --gc-sections), not lib tests, across the target/linker matrix — because DCE failures are silent and configuration-dependent.
- ACCEPTS that auto-discovery is NOT a true classpath scan and NOT uniformly passive across all platforms: it is link-time collection WITHIN each linked crate + explicit opt-in of WHICH crates participate (feature set + scan! list). On wasm/embedded/staticlib the substrate degrades to build-codegen (Arch C) or explicit registration (Arch D) behind the identical &[Descriptor] runtime API. leaf must DOCUMENT this honestly and never promise 'a transitive dep silently injects beans' — also the predictability posture owner-decision #1 endorses (back-off + exclusions, activated by a starter/feature, never an arbitrary transitive dep).
- ACCEPTS that generics cannot be link-registered: a generic #[component] Foo<T> has no monomorphized symbol/TypeId, so the macro MUST hard-error on a generic bean and offer a per-instantiation register_component!(ConcreteType) shim. Inherent to erasure-without-reflection; matches the cross-crate doc and the registry-substrate ADR's concrete-TypeId keying.
- ACCEPTS a small, bounded const-data ABI surface (CondExpr enum, AnnotationMetadata tables, TypeRow) as load-bearing: emitted const trees hard-code these shapes, so evolving them is costly. Mitigated by keeping them flat and minimal (registry-substrate/annotation-model ADRs already constrain this) and by the const{assert!} ContractId-collision guard at freeze catching identity clashes loudly.
- FORCES the descriptor format to be migration-ready for a future compiler-native #[global_registration] (rust#125119): because the macro emits a const Descriptor into a linkme slice and the runtime reads a flat &[Descriptor], leaf could later swap linkme for #[global_registration] with no macro/ABI change. The decision does NOT block on that unstable feature (it ships on stable linkme) but is shaped to adopt it later.

## API / type sketch

// ═══ leaf-core: the ultra-stable registration ABI emitted code hard-codes ::leaf_core paths to ═══

// ── const fn-pointer factory seed (NOT a live object; the Provider is BUILT at freeze) ──
pub struct ProviderSeed {
    pub contract: ContractId,
    pub make:     fn(&Registry) -> std::sync::Arc<dyn Provider>,  // const fn-pointer; boxed-future seam built here
}

// ── ONE substrate, many typed slices (link-time collection, zero pre-main cost) ──
#[linkme::distributed_slice] pub static COMPONENTS:   [Descriptor]   = [..];
#[linkme::distributed_slice] pub static PROVIDERS:    [ProviderSeed] = [..];
#[linkme::distributed_slice] pub static AUTO_CONFIGS: [Descriptor]   = [..];  // auto_config flag in meta
#[linkme::distributed_slice] pub static CONDITIONS:   [CondImplRow]  = [..];  // ConditionId -> &'static dyn Condition
#[linkme::distributed_slice] pub static LISTENERS:    [Descriptor]   = [..];  // every discovery feature = another slice

// ── conditional strategy: the per-kind tier map is a kernel ABI ──
pub enum EarliestTier { Cfg, ConstFold, Runtime }            // declared per ConditionId kind
pub enum CondExpr {                                          // const-constructible tree (annotation-model ADR)
    Const(bool),                                             // a build/cfg-folded leaf collapses to this
    Leaf { id: ConditionId, attrs: &'static [Attr] },        // runtime-evaluated leaf
    All(&'static [CondExpr]), Any(&'static [CondExpr]), Not(&'static CondExpr),
}
impl CondExpr { pub const fn tier(&self) -> EarliestTier; } // = max over leaves; drives phase()

// ── compile-time identity: stable across builds (TypeId is in-process only — registry-substrate ADR) ──
pub const fn contract_hash(stable_ident: &str) -> u64;       // const FNV/Fx; const{assert!} collision-guarded at freeze
#[derive(Clone, Copy, PartialEq, Eq, Hash)] pub struct ContractId(pub u64);

// ── LAYER 4: runtime assembly, around main, synchronous cold pass (NOTHING async/IO pre-freeze) ──
pub struct RegistryBuilder { /* append-only */ }
impl RegistryBuilder {
    pub fn from_slices() -> Self;                                                   // lifts COMPONENTS/PROVIDERS/...

    pub fn evaluate_conditions(&mut self, ctx: &ConditionCtx) -> Result<(), AssemblyError>; // cfg pruned/const-fold decided; runs runtime tier
    pub fn run_autoconfig(&mut self, env: &Env) -> Result<(), AutoConfigError>;          // R1 deferred back-off batch (default)
    pub fn freeze(self) -> Result<Registry, AssemblyError>;                              // dense BeanId; ContractId collision guard

}

## Forces on other concepts
- **Type-Erasure & Registry Substrate (upstream registry-substrate ADR)**: Honored and built upon unchanged: the macro emits exactly that ADR's Descriptor (self_type via const TypeId::of, ContractId via const contract_hash, provides[] TypeRows, ScopeDef, AnnotationMetadata, Origin) into the COMPONENTS slice; RegistryBuilder::freeze() builds the dense BeanId space and runs the ContractId collision guard. This boundary owns HOW the rows are emitted/collected/assembled; the registry-substrate ADR owns their KEYING/STORAGE. The reopen_epoch second-epoch seam this ADR exposes is the registry-substrate ADR's dynamic/WASM lane.
- **Scope / Lifetime / Ownership Model (upstream ownership-model ADR)**: Honored unchanged: the ProviderSeed.make const fn-pointer builds the ADR's dyn Provider (origin-agnostic, BoxFuture seam) at freeze — never pre-main, so no live object exists at link time (the cross-crate doc's 'register descriptors/factories, never live objects' rule). Published/ErasedBean/Ref<T> are the runtime currency; the macro emits only the const seed. The boxed-future provide() is built in LAYER 4, satisfying the ADR's async-across-dyn standard.
- **Cross-Crate Composition & Anti-DCE (register §7, owner-decision #1)**: This ADR IS the realization: linkme is the default substrate; the binary-crate build.rs owns force-link + the expected-crate manifest; freeze()'s self_check turns a DCE-dropped crate into a loud NoSuchBean naming the missing source. Guardrails (>=1.89 / -z nostart-stop-gc, linkme>=0.3.24, resolver=3) mandated. Architecture C (build-codegen) is the wasm/embedded path; D (explicit register/with) the universal escape hatch; all behind one &[Descriptor] API. 'Which crates participate' = feature set + scan! list (scoped, not global).
- **Conditional Strategy + Profiles + Auto-Config Back-off (register §8)**: The per-kind tier map (cfg / const-fold / runtime) is fixed here as a kernel ABI: OnClass/OnCapability->cfg (+force-link), target/build-config->const-fold, OnBean/OnProperty/profiles/dynamic-WASM->runtime. Runtime tier MANDATORY (origin-agnosticism). Profiles always runtime. CondExpr.phase() framework-inferred (max over leaves). Auto-config = R1 deferred runtime batch by default (back-off needs the final user-def set), R3 build-frozen plan opt-in via cargo leaf prepare. Starter = dep:-hidden umbrella feature + force-link (owner-decisions #2/#3).
- **Annotation / Merged-Annotation Model**: The merge is frozen into flat const AnnotationMetadata at MACRO time (compile-time-legible errors for @AliasFor/stereotype misconfig), not merged at runtime. The macro emits that const meta into Descriptor.meta; the CondExpr tree lives in meta. The MarkerId/AttributeEntry/CondExpr ABI must stay ultra-stable (emitted const trees hard-code paths). This forbids a runtime-late-overlay annotation model for native beans, but dynamic/WASM sources contribute their own already-merged meta via reopen_epoch.
- **BeanDefinition Metamodel & @Configuration/@Bean**: The SEED is a flat const Descriptor (the register's flat-const-RawBeanDefinition option, not builder-attachment) — fixing the metamodel's const-friendliness. @Configuration is a compile/codegen-time grouping that lowers each @Bean into its own Descriptor+ProviderSeed row; no runtime config object on the default path. FactoryBean is just another ProviderSeed whose product type appears as a provides[] row pointing at the factory's contract.
- **Container Refresh Lifecycle & Compile-vs-Runtime Split**: Forces a definite phase progression: link-time collection -> (build-time force-link + manifest + optional plan) -> around-main RegistryBuilder lift -> anti-DCE self-check -> condition eval (cfg pruned, const-fold decided, runtime tier evaluated) -> deferred auto-config back-off batch -> freeze (dense BeanId, safe-publication-for-free) -> async eager instantiation. Post-freeze edits are a typestate/diagnostic error; the dynamic/WASM second epoch (reopen_epoch->re-freeze) is the only sanctioned later mutation.
- **WASM Optional Layer / Two-Source Duality**: The static linkme/build-codegen slice is the default 'classpath'; the dynamic directory-scanned WASM source enters via reopen_epoch (controlled second-epoch re-freeze), contributing identical Descriptor+Provider rows (origin=DynamicWasm, diagnostic-only). A WASM-contributed definition has no cfg and no Rust type, so it is gated ONLY by the runtime condition tier — why that tier is mandatory. WASM contracts key on ContractId, never TypeId. On wasm targets the native substrate is build-codegen (Arch C), not linkme.
- **Unified Error / Diagnostics Model (register §6)**: Errors split by timing for earliest-richest context (§2.9): malformed CondExpr / @AliasFor / unknown feature / generic-bean = compile error with a span; ContractId collision / ordering cycle / anti-DCE expected-vs-found mismatch / cargo-leaf-prepare --check staleness = build-or-freeze-time error naming the source; condition-not-met = a silent recorded report entry (NOT an error), kept distinct from NoSuchBean (absent) and condition-gated-out (no slot minted). The macro owns #[diagnostic::on_unimplemented] doctrine messages for the Send+Sync bound.

## Open sub-questions (→ Phase 3)
- The exact macro supertrait-emission policy for provides[] TypeRows (emit an upcast row for every declared injectable supertrait vs only those used at an injection point — which the macro can't see cross-crate). Likely 'all declared injectable supertraits' for cross-crate safety, measured for row-count bloat. Shared with the registry-substrate ADR's identical open question. Phase 3.
- The precise contract_hash inputs and collision policy: which author-stable string feeds the const FNV/Fx hash (cargo package name + module path + ident + declared-name?), how re-exports/renames affect it as a semver surface, and whether the freeze-time collision guard hard-errors or salts. Co-decided with auto-config exclusion identity and the registry-substrate ADR. Phase 3.
- Whether cargo leaf prepare (the build-time auto-config plan + --check-able artifact) ships in the first release or is deferred: it is a real maintenance surface (manifest convention, cargo-metadata aggregator, OUT_DIR hygiene, deterministic codegen) justified mainly by wasm/native-image/determinism. Measure whether the default R1 runtime batch is fast enough that the plan is only worth it for hostile targets. Phase 3, measured before built.
- The exact scan!{ ... } / #[leaf::main] DX for expressing 'which crates participate' and how it interfaces with the dep:-hidden feature set (does enabling a feature auto-add to the scan list, or must both be stated?). The force-link emission detail (use crate as _; vs a linkme-anchor reference) and the generated .cargo/config.toml content. Phase 3.
