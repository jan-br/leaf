# ADR 08 — Proxy / Interception Substrate (no CGLIB)  `[proxy-substrate]`

> **Note (2026-06-14):** the optional *dynamic-WASM source / WASM-"classpath"* mentioned below is **DROPPED** — leaf autodiscovery is native link-time registration only. Ignore WASM/dynamic-module references here; they are struck from the active design.


## Decision
ADOPT a COMPILE-TIME-GENERATED, PER-SERVICE-TRAIT transparent delegating newtype as leaf's single proxy shape (the rust-native "CGLIB analogue moved to codegen"), routing every external call through ONE ordered, replayable, short-circuitable `Interceptor` chain whose innermost step is a `TargetSource` (the per-call target supplier). There is exactly ONE wrap primitive; AOP advice, scoped re-resolution, @Lazy, @Async, tx, cache, validation, retry, and exception-translation are all just chain entries and/or TargetSource choices over it. Concretely, in `leaf-core` (the ultra-stable kernel ABI macros hard-code `::leaf_core` paths to):

PROXY MECHANISM = generated transparent newtype, NOT a runtime method-table, NOT tower-per-edge. For a bean `OrderSvc` exposing injectable service trait `dyn Order`, `leaf-macros` (thin) emits one `struct OrderProxy { chain: Arc<AdviceChain>, source: Arc<dyn TargetSource> }` and `impl Order for OrderProxy { async fn place(&self, a: Args) -> R { self.chain.invoke(Call::new::<{Order::place}>(&self.source, ErasedArgs::pack((a,))) ).await.unpack() } }` — one delegating method per advisable trait method, each packing typed args into the chain and unpacking the typed return. Because `OrderProxy: Order`, it upcasts to `Arc<dyn Order>` (trait upcasting, stable 1.86) and is injected wherever `dyn Order` is wanted — DI never sees the raw target, autowiring is untouched, Spring's invisibility is preserved. The chain/`Interceptor`/`Call`/`AdviceChain` is hand-writable library code (the generated newtype is exactly what a user could hand-write — thin macros, charter §2.10). The DYNAMIC/WASM-sourced lane (a bean with no compile-time type) gets the runtime ERASED fallback: a single `ErasedProxy { chain, source, methods: &'static MethodTable }` driving downcast thunks — used ONLY where no generated type can exist; native/test/FactoryBean beans always use the generated newtype.

TRANSPARENT (`dyn Svc`), NOT a visible handle. The advice proxy injects as the SAME type the target satisfies. The honest visible-handle shape (`ScopedRef<dyn T>`, `Provider<T>`, `@Lazy`) is the DEFERRAL-PRIMITIVES family's surface (a separate ADR), NOT this substrate's. This is mandatory for the co-advised-AND-scoped case to compose into ONE object (a `@Transactional` request-scoped bean cannot be both a visible handle and a transparent proxy).

SERVICE TRAIT IS MANDATORY for transparent interception (Rust has no subclass-a-concrete-class). A `#[component]` that is `#[advisable]`/targeted by any advisor MUST expose a service trait; the macro hard-errors with a `#[diagnostic::on_unimplemented]`-style message ("type `Foo` is advised by `@Transactional` but exposes no service trait; declare `impl Order for Foo` and inject `dyn Order`, or move the concern to a collaborator"). Concrete-only beans are injectable but un-advisable — a deliberate, diagnosed divergence from Spring.

THE CALL-ROUTING PRIMITIVE — one trait, BoxFuture seam (forced by async-context-model 5b; AFIT/RPITIT not dyn-compatible regardless of nightly):
```
pub trait Interceptor: Send + Sync {
    fn intercept<'a>(&'a self, call: Call<'a>, next: Next<'a>) -> BoxFuture<'a, Result<ErasedRet, AdviceError>>;
}
```
`next.proceed().await` advances to the next interceptor or finally to `TargetSource::get(cx).await` + the real method. `Next` is REPLAYABLE (calling `proceed()` more than once re-runs the tail as a FRESH future — retry's requirement) and SKIPPABLE (an interceptor may return `Ok(ret)` WITHOUT calling `proceed()` — caching's short-circuit requirement). `Call` carries the method identity (`MethodKey` = stable name-hash), the role/order context, and the typed-but-erased args (`ErasedArgs`); typed around-advice downcasts at the seam; the generated newtype provides per-method typed pack/unpack glue so the common path stays typed and the cost is a move, not a per-arg box.

THE TARGETSOURCE SEAM (the TargetSource concept the Phase-1 coherence note demanded built-in): the chain's innermost step does not assume a fixed inner target — it calls
```
pub trait TargetSource: Send + Sync {
    fn get<'a>(&'a self, cx: &'a ResolveCtx<'a>) -> BoxFuture<'a, Result<ErasedBean, ResolveError>>;
}
```
A SINGLETON-advised proxy holds `FixedTarget(ErasedBean)`; a SCOPED proxy holds `ScopeTarget { scope, key, weak: Weak<ContainerCore> }` that re-resolves the live instance from the ambient `Cx` per call (scoped-proxies R3 LiveTargetProvider, unified into the substrate). So a request-scoped + `@Transactional` bean is ONE proxy: the `ScopeTarget` yields the live target, then the unchanged advice chain wraps it. This keeps proceed()'s contract un-distorted (scope re-resolution is NOT a fake advisor) and is the duplicated-wrap unification both Phase-1 designs asked for.

ADVICE COMPOSITION & ORDERING = ONE `AdviceChain` per bean carrying the UNION of matching advice, sorted by the SINGLE canonical comparator `(RoleTier, Order:i32, stable-tiebreak ContractId)` — RoleTier FIRST (infrastructure advice positioned deterministically relative to application aspects), then the workspace-wide `@Order`/`@Priority` comparator (lower-wins; shared with autowiring tie-break / collection order / lifecycle phases — owned by the ordering ADR, CONSUMED here at correctness grade). The assembled order is INSPECTABLE (`Container::advice_chain_of(BeanKey) -> &[AdvisorRef]` + `leaf doctor`) because misordering (cache-outside-vs-inside-tx) is invisible in object identity.

AUTO-PROXY-CREATOR = ONE per-context processor computing a frozen `ProxyPlan` at the registry FREEZE boundary (registry-substrate): match runs once over declared method-metadata + advisor descriptors → `BeanId -> &[ordered AdvisorRef]`, role-filtered by the active `CreatorPolicy` (infra-floor capability lattice, UPGRADED in place to admit application aspects — never racing processors; assembled at the binary/app-root where the full feature set is visible). At the instance-decoration seam (the ownership-model `after_init`/post-process hook returning a possibly-different `ErasedBean`) the processor does an O(1) plan lookup and installs the proxy. Eligibility gate: un-advised beans pass through unwrapped (bare-engine parity — no creator installed = advice inert). getEarlyBeanReference consults the SAME frozen plan (early-ref == final proxy by construction; no dedup set). The early-proxy sub-tier is RUNTIME-ONLY and trivial here because the proxy holds a `TargetSource`, not a live target — it is materializable over a not-yet-built target with no `.await`.

ADVISORS are FLAT const descriptors (NOT first-class beans), avoiding the bootstrap chicken-and-egg: `AdvisorDescriptor { id: ContractId, order: i32, role: Role, pointcut: &'static dyn Pointcut, make_interceptor: fn(&Container) -> BoxFuture<Arc<dyn Interceptor>> }` discovered via the shared cross-crate substrate (registry-substrate / anti-DCE). `make_interceptor` resolves the aspect bean (so advice can inject collaborators) at refresh; a missing advisor crate surfaces via the mandatory expected-vs-found self-check (NOT a silent no-op — and for @Async, NOT a silent sync run).

ASPECT AUTHORING: `#[aspect]` is a `#[component]` whose `#[advice(around|before|after|after_returning|after_throwing, pointcut = …, order = …)]` methods lower to `AdvisorDescriptor`s; before/after collapse to one around-shaped `Interceptor` at lowering (the chain sees ONE kind). Pointcuts are TYPED COMBINATORS primary (`within::<dyn Order>() & annotated::<Transactional>() & returns::<Result<_,_>>()`) over declared `JoinPointMeta`; an OPTIONAL compile-time-parsed expression front-end (`#[pointcut("execution(..) && @annotation(Transactional)")]`) lowers to the same predicate (Spring-feel, zero runtime parser). leaf's own tx/cache/validation/async/retry/exception-translation author against the SAME vocabulary, emitting `Role::Infrastructure` descriptors (dogfooding, charter §2.11).

SELF-INVOCATION = honest boundary, UNIFORM across ALL advisors including @Async (reject the @Async body-rewrite as the default precisely to avoid per-advisor inconsistency — `self.foo()` must mean the same thing for `@Async` and `@Transactional`). `this.method()` bypasses the proxy because the wrapper is external to the target's control flow — reproduced honestly, with the same Spring-sanctioned workarounds (refactor to a collaborator; inject `Ref<dyn Self>` self-reference) and a richer compile-time diagnostic note. No `AopContext.currentProxy()` ThreadLocal analogue (it would need an ambient holder and re-introduce the staleness clash). Each declarative feature's imperative escape (e.g. `RetryTemplate::execute`) remains the proxy-free path.

CANCEL-SAFETY without async Drop (async-context-model 5d): around-advice holding a resource across `proceed()` (a tx, a permit, a `Cx` scope guard) releases via SYNC RAII Drop on the cancel path (drop = roll back / release / pop); genuinely async finalization (a flush) is the container-driven teardown ledger's job, never Drop. `Cx`/ambient binding for advice rides the poll-re-installing combinator (NOT a ThreadLocal); the `ConcurrencyGate` permit (async-context-model 5c) is the shared limit primitive for `@ConcurrencyLimit`.

## Rationale
- Intent parity + DX#1 (charter §2.1/§2.2): the generated transparent newtype is the ONLY shape that preserves Spring's defining proxy property — the holder injects `Arc<dyn Order>` and never knows it holds a wrapper, autowiring-resolution is untouched, and a co-advised bean is ONE object. The runtime method-table (aop-proxying R2) forces either a generated typed shell anyway (so not zero-codegen) or a VISIBLE `Proxied<dyn T>` that breaks invisibility and ripples injection-point types cross-crate; tower-per-edge (R3) breaks Spring's load-bearing 'wrap once, globally — every external caller gets advice' guarantee and mismatches tower's single-call `Service` shape against a multi-method trait. The generated newtype is also the lowest steady-state overhead (charter §2.5 #2): monomorphized delegation, no per-call erased-args boxing, no reflection mini-layer.
- It is buildable on the confirmed-stable envelope with ZERO nightly (charter §2.8): trait upcasting (1.86) injects `OrderProxy` as `Arc<dyn Order>` as a cheap vtable adjust; AFIT/RPITIT (1.75) author the advice traits on the static path; the dyn `Interceptor` seam boxes its future (`BoxFuture`) exactly as the upstream async-context-model ADR already locked (5b) — so this decision inherits the boxing standard rather than re-litigating it. Async closures (1.85) author `Advice::around(|call| async { … })` ergonomically. No part needs RTN (correctly 'avoid' per the nightly doc) — the `trait_variant`/`dynosaur`/hand-emitted-BoxFuture layer is an internal seam.
- The TargetSource seam is the unification the Phase-1 coherence note explicitly demanded ('whichever feature defines the call-routing primitive must build in the where-does-the-target-come-from seam from the start, or the unification both designs want becomes a retrofit'). Making proceed()'s innermost step a `TargetSource` rather than a fixed inner target collapses scoped-proxies (R3 LiveTargetProvider), @Lazy deferral, and AOP into ONE wrap primitive: scope re-resolution is the innermost target supplier feeding the UNCHANGED advice chain, so a request-scoped `@Transactional` bean composes as 'resolve live target, then run the chain' with proceed()'s fixed-inner contract un-distorted (the explicit defect of scoped-proxies R1).
- REPLAYABLE + SKIPPABLE `Next` is a hard, named requirement of two consumers the register flags: retry-resilience needs `proceed()` to be FnMut→fresh-future (it calls the real method multiple times), and caching needs to return WITHOUT calling proceed() (cache-hit short-circuit). Designing these into the `Next` contract from the start (rather than a fixed once-only proceed) is what lets retry/cache be plain chain entries instead of bespoke machinery — maximal shared machinery (charter §2.11).
- Advisors-as-flat-descriptors (NOT first-class beans) deliberately avoids the bootstrap chicken-and-egg both auto-proxy-creator R3 and aspect-model R3 conceded is real: advisors-as-beans must be fully built (async, possibly cyclic) BEFORE any advisable bean's decoration, forcing a fragile line-950 early-phase carve-out that gets worse for the SYNC mid-cycle getEarlyBeanReference path. The descriptor carries a `make_interceptor` factory that resolves the aspect bean at refresh — so advice STILL injects collaborators (Spring's '@Aspect is a bean' intent) without making the matchable contribution itself a bean. The descriptor is also the flat const shape the cross-crate substrate (registry-substrate) and anti-DCE self-check already handle uniformly.
- The frozen ProxyPlan at the registry freeze boundary (registry-substrate) gives the strongest §2.9 diagnostics ('why is bean X advised with [tx,cache,audit]?' is answerable by reading the plan), eliminates O(beans×advisors) per-decoration match cost, and makes early-ref==final-proxy automatic (same plan, same answer — dissolving Spring's earlyProxyReferences dedup set). It composes with the registry-substrate's freeze/seal lifecycle (descriptors append cold, seal once, optional dynamic second epoch for the WASM lane) — the dynamic source appends plan entries pre-freeze or re-freezes; it cannot retroactively advise already-built beans, an honest, bounded limitation.
- UNIFORM self-invocation across ALL advisors (rejecting @Async body-rewrite as default) resolves the sharpest cross-feature inconsistency the coherence note names: if @Async killed the gotcha via body-rewrite while @Transactional preserved it, `self.foo()` would silently run-sync-but-advised for one and run-unadvised for the other on the same method shape — a WORSE trap than Spring's uniform 'self-calls bypass advice'. Keeping @Async a chain advisor (already locked by async-context-model 5c as a `Role::Infrastructure` dispatch advisor) means its order relative to @Transactional is governed by the one deterministic chain sort (a tx must not span the async hop) — correctness the body-rewrite sacrifices by leaving the chain.
- Origin-agnosticism is preserved (the hard brief constraint + WASM 'WASM-ness stops at the proxy'): the chain wraps an `ErasedBean` via the `TargetSource`, so native, test-double, and FactoryBean targets use the generated newtype identically; only the genuinely-no-compile-time-type dynamic/WASM bean falls back to the runtime `ErasedProxy` shim — a localized, diagnosed split, not a contaminating second model. The advisor descriptor is a flat const with a fn-ptr factory, so a WASM-module advisor collects into the same stream.
- Cancel-safety via sync RAII Drop + the container teardown ledger (no async Drop) rides the upstream async-context-model 5d decision unchanged: a cancelled call mid-`proceed()` drops the around-future, whose RAII guard rolls back the tx / releases the `ConcurrencyGate` permit / pops the `Cx` scope synchronously; async finalization is never on Drop. This keeps the around-advice contract honest under cooperative cancellation without inventing per-feature unwind machinery.

## Options considered & rejected
- **Runtime metadata method-table + `ProxyHandle` (aop-proxying R2, the JDK-dynamic-proxy analogue) as the primary mechanism** — rejected: It still needs a generated typed shell (`impl Order for ProxyHandle`) for transparency — so not zero-codegen — OR a visible `Proxied<dyn T>` that breaks Spring's invisibility and changes injection-point types cross-crate. Worse, it pays a per-call erased-args/erased-return boxing tax (alloc + downcast on the HOT advised path) that the generated newtype avoids with monomorphized typed pack/unpack, and it re-imports a hand-rolled mini-reflection layer (`erased_invoke` thunks keyed by table index) — the exact JVM reflection capability the charter warns against re-importing for its own sake (§2.2). Its one genuine advantage (smooth dynamic/WASM blend) is captured by demoting the erased shim to the dynamic-source-only fallback (`ErasedProxy`), where there is no compile-time type to generate against — so the upside is kept without making every native bean pay the erased tax.
- **tower::Layer/Service resolution-seam middleware (aop-proxying R3) — interception as a property of the A→B edge, no proxy object** — rejected: Breaks Spring's load-bearing 'wrap the bean once, globally — every external caller gets advice' guarantee: an edge-based model risks advising some injection points and not others, re-introducing the 'is this reference advised?' ambiguity Spring eliminated. tower's `Service` is request/response-shaped (one call type per Service) while a bean trait has many methods of different signatures — forcing an enum-of-calls or per-method Service that re-introduces codegen anyway. The early-reference/cycle case is awkward (what does a mid-cycle collaborator inject if the 'proxy' only exists at the edge?). The `Interceptor` chain adopted here IS the tower-spirit composition (ordered async middleware with a replayable `Next`) but anchored to a per-bean transparent object, getting tower's composability without sacrificing wrap-once-globally or transparency.
- **Visible-handle injection (`Advised<dyn T>` / `Proxied<dyn T>`) for advice — honest, zero dispatch codegen** — rejected: Fatal for the co-advised-AND-scoped composition the cluster explicitly promises: you cannot wrap a visible handle and a transparent proxy as ONE object, so a request-scoped `@Transactional` bean becomes incoherent. It also changes the holder's public injection-point type, rippling cross-crate (consumers in other crates must name `Advised<dyn Order>`) and through autowiring/injection-styles — cutting against DX#1's Spring-grade transparency. The honest visible-handle shape is genuinely right for the DEFERRAL family (`ScopedRef`/`Provider`/`@Lazy`, where re-resolution/optionality SHOULD be in the type and the call is `.get().await?`) — so it is adopted THERE (a separate ADR), and advice keeps the transparent shape. This splits the transparent-vs-honest dilemma along the correct seam (deferral = honest; advice = transparent) rather than forcing one answer everywhere.
- **@Async (and cache/validation/retry) via macro body-rewrite (async-execution R2), eliminating the self-invocation gotcha** — rejected: Produces INCONSISTENT self-call semantics across cross-cutting concerns that all claim the same substrate: `self.foo()` would dispatch-but-not-tx for @Async-rewritten and run-unadvised for @Transactional-proxied on the same method shape — a worse trap than Spring's uniform bypass. It removes @Async from the shared ordered chain, making its order relative to @Transactional ill-defined (does the tx span the async hop?) — the invisible-misordering bug the shared chain prevents. It also pre-judges the metadata/codegen boundary toward heavy compile-time rewrite and bakes a hidden injected-executor field into the bean. Kept ONLY as a deferred Phase-3 opt-in hybrid (body-rewrite for dispatch PLUS a chain marker so ordering stays governed), never the default.
- **Advisors as first-class beans collected via collection-injection (auto-proxy-creator R3 / aspect-model R3)** — rejected: Maximal unification on paper, but inherits a real bootstrap chicken-and-egg: advisor beans must be fully built (async, possibly cyclic, injecting their own deps) BEFORE any advisable bean's decoration, forcing a line-950 early-phase carve-out that becomes a re-entrancy trap at the SYNC mid-cycle getEarlyBeanReference point. It also entangles AOP role-isolation with the qualifier subsystem and conflates collection-injection's loose preference-grade ordering with correctness-grade advice ordering. The flat-descriptor-with-make_interceptor-factory keeps the genuine win (advice injects collaborators, conditions/profiles gate activation via the descriptor's condition predicate) without making the matchable contribution itself a bean in the same build graph.
- **Per-bean runtime matching at the decoration seam (auto-proxy-creator R1) instead of a frozen ProxyPlan** — rejected: Pays O(beans×advisors) match cost per decoration and needs a mutable `early_proxied` dedup set to keep early-ref and final proxy identical (shared mutable state on the cycle path). The frozen-plan approach (computed once at the registry freeze boundary the registry-substrate already defines) makes early-ref==final-proxy automatic, gives a single inspectable artifact for diagnostics, and reduces decoration to an O(1) lookup. Runtime conditions are handled by the descriptor's condition predicate evaluated at freeze (when container state is known), so the plan reconciles with 'what DOES wire up' without plan-per-profile explosion.

## Consequences
- FORCES a service trait to be MANDATORY for any advised/scoped bean (no subclass-a-concrete-class in Rust). The macro hard-errors on an advised concrete-only bean with a doctrine-citing diagnostic steering to 'declare a service trait or move the concern to a collaborator.' This is a real, named divergence from Spring (which CGLIB-proxies trait-less classes) and ripples into component-stereotypes, persistence-exception-translation (@Repository must expose a trait), and validation. Concrete-only beans stay injectable but un-advisable.
- FORCES every advisable trait method through the generated newtype's per-method typed pack/unpack glue plus the boxed-future-per-interceptor-hop on the HOT advised-call path (async-context-model 5b standard). Cost: one box per chain hop + the chain walk; mitigated because (a) un-advised beans are NOT wrapped (zero cost), (b) the typed pack/unpack is a move not a per-arg box, (c) the deferred Phase-3 monomorphized single-advisor fast-path remains available. The dynamic/WASM `ErasedProxy` fallback additionally pays per-call erased-args boxing — accepted only where no compile-time type exists.
- FORCES the metadata/codegen boundary (registry-substrate) to capture, as DATA, each advisable type's method table (stable `MethodKey` name-hash per method) and per-method attribute-presence flags (in `AnnotationMetadata`), because pointcut matching is a pure predicate over declared metadata, never a reflective method walk. Method identity must be stable-as-data or method-keyed advice mis-matches silently across refactors — the typed `within::<T>`/`annotated::<A>` combinators mitigate this (a rename is a compile error, unlike a stale string).
- FORCES the auto-proxy ProxyPlan to be computed at the registry FREEZE boundary (registry-substrate) and the `CreatorPolicy` capability lattice (infra-floor upgraded to admit application aspects) to be assembled at the binary/app-root where the full enabled-feature set is visible — never as racing post-processor beans (link/registration order is unspecified and may be randomized). The dynamic/WASM second epoch may append plan entries pre-freeze or re-freeze, but cannot retroactively advise already-built beans.
- FORCES the mandatory anti-DCE expected-vs-found self-check (cross-crate / owner-decision #1) to cover the advisor-descriptor stream AND the auto-proxy-creator/executor presence: a DCE'd advisor crate must surface as a loud diagnostic naming the missing source, NOT a silent no-op — and for @Async, NOT a silent synchronous run on the wrong context. `leaf doctor` must list 'advisors discovered by role/crate', 'assembled chain order per bean', and 'is the auto-proxy-creator + execution facility present'. The erased recognition (downcast) makes this self-check load-bearing.
- FORCES the ONE canonical `@Order`/`@Priority`/`Ordered` comparator (owned by the ordering ADR) to serve advice ordering at CORRECTNESS grade (RoleTier first, then lower-value-wins, then stable `ContractId` tiebreak), and FORCES the assembled chain order to be an inspectable §2.9 diagnostic (`Container::advice_chain_of`) because misordering (cache-outside-vs-inside-tx) is invisible in object identity. Collection-injection's preference-grade use of the same comparator must not weaken the determinism advice relies on.
- FORCES the context-propagation model (async-context-model 5a) to be the ONLY ambient mechanism advice reads: scoped-proxy `TargetSource` re-resolution and tx/locale/tracing binding read the poll-re-installing `Cx`, never a ThreadLocal/`AopContext.currentProxy()` analogue. The transaction resource binding is an `Isolate` `Cx` key (never auto-captured across the @Async spawn hop). Around-advice cancel-safety is sync RAII Drop on cancel + the container teardown ledger for async finalization (no async Drop).
- FORCES the deferral-primitives ADR to own the HONEST visible-handle shapes (`ScopedRef<dyn T>`, `Provider<T>`, `@Lazy`): this substrate deliberately does NOT make the advice proxy a visible handle, so the transparent-vs-honest split is resolved as 'advice = transparent `dyn Svc`; deferral = honest handle in the type.' The two must agree that a scoped-AND-advised bean uses the transparent proxy with a `ScopeTarget`, while a purely-deferred dependency uses the visible handle.
- ACCEPTS that the early-proxy (getEarlyBeanReference) sub-tier is RUNTIME-ONLY (AOT contexts disallow cycles) and trivial: the proxy holds a `TargetSource`, not a live target, so it materializes over a not-yet-built target with no `.await`. This collapses circular-references' three-level cache question — the middle/early slot earns its keep only for the runtime cycle case, and the frozen plan makes early==final automatic.
- ACCEPTS a generated-type-per-advisable-trait codegen volume and a generics hard-stop: a generic `Repo<T>` cannot generate a proxy without a concrete instantiation; the macro hard-errors with a `register_proxy!(Repo<ConcreteT>)` per-instantiation escape (mirroring generics-can't-register). Cross-crate, the proxy for a bean in crate X must be generated where X's trait set is visible while advisors may be contributed in crate Y — the two rendezvous at the binary/app-root via the ProxyPlan, an accepted, bounded coupling.

## API / type sketch

// ── leaf-core: the ONE interception substrate over the ownership-model ErasedBean / Provider ──

// CALL-ROUTING PRIMITIVE — boxed-future seam (async-context-model 5b: AFIT/RPITIT not dyn-compatible)
pub type BoxFuture<'a, T> = std::pin::Pin<Box<dyn std::future::Future<Output = T> + Send + 'a>>;

pub struct Call<'a> {
    pub method: MethodKey,                 // stable name-hash (data, not reflection)
    pub bean:   BeanKey,                   // who is advised (diagnostics, ordering context)
    pub args:   ErasedArgs,                // typed-but-erased; generated glue packs/unpacks
    pub source: &'a dyn TargetSource,      // the per-call target supplier (innermost)
    cx:         &'a ResolveCtx<'a>,
}
pub struct Next<'a> { /* remaining interceptors + tail to TargetSource + real method */ }
impl<'a> Next<'a> {
    pub fn proceed(&mut self) -> BoxFuture<'a, Result<ErasedRet, AdviceError>>; // REPLAYABLE: FnMut -> fresh future (retry)
    // an interceptor may simply NOT call proceed() and return Ok(ret) -> SHORT-CIRCUIT (cache hit)
}
pub trait Interceptor: Send + Sync {       // ONE advice kind; before/after lower to this
    fn intercept<'a>(&'a self, call: Call<'a>, next: Next<'a>)
        -> BoxFuture<'a, Result<ErasedRet, AdviceError>>;
}
pub struct AdviceChain { ordered: Box<[Arc<dyn Interceptor>]> } // union of matching advice, sorted by the ONE comparator
impl AdviceChain { pub fn invoke<'a>(&'a self, call: Call<'a>) -> BoxFuture<'a, Result<ErasedRet, AdviceError>>; }

// THE TARGETSOURCE SEAM — proceed()'s innermost step; NOT a fixed inner target (the TargetSource concept)
pub trait TargetSource: Send + Sync {
    fn get<'a>(&'a self, cx: &'a ResolveCtx<'a>) -> BoxFuture<'a, Result<ErasedBean, ResolveError>>;
}
pub struct FixedTarget(ErasedBean);                                   // singleton-advised
pub struct ScopeTarget { scope: ScopeKind, key: BeanKey, weak: std::sync::Weak<ContainerCore> } // scoped re-resolve (unifies scoped-proxies)
impl TargetSource for FixedTarget { /* yields the held ErasedBean */ }
impl TargetSource for ScopeTarget { /* reads ambient Cx -> InstanceStore::get_or_init, live per call */ }

// THE PROXY — generated transparent newtype per advisable service trait (primary mechanism)
// leaf-macros (thin) emits this; it is exactly hand-writable; `cargo expand` shows it:
//   pub struct OrderProxy { chain: Arc<AdviceChain>, source: Arc<dyn TargetSource> }
//   impl Order for OrderProxy {
//       async fn place(&self, a: Args) -> R {
//           self.chain.invoke(Call::method::<{Order::place}>(&*self.source, ErasedArgs::pack((a,)))).await.unpack()
//       }
//   }
//   // upcasts to Arc<dyn Order> (trait upcasting 1.86) -> injected transparently; autowiring untouched.

// DYNAMIC/WASM-ONLY FALLBACK — runtime erased shim where no compile-time type exists ("WASM-ness stops at the proxy")
pub struct ErasedProxy { chain: Arc<AdviceChain>, source: Arc<dyn TargetSource>, methods: &'static MethodTable }
pub struct MethodTable(&'static [MethodEntry]);
pub struct MethodEntry { pub key: MethodKey, pub invoke: fn(&ErasedBean, ErasedArgs, &ResolveCtx) -> BoxFuture<Result<ErasedRet, AdviceError>> }

// ADVISOR DESCRIPTOR — flat const (NOT a first-class bean); avoids bootstrap chicken-and-egg
pub struct AdvisorDescriptor {
    pub id:        ContractId,             // stable identity + tiebreak (registry-substrate)
    pub order:     i32,                    // the ONE @Order/@Priority comparator (ordering ADR)
    pub role:      Role,                   // Application | Support | Infrastructure (data-only, no cascade)
    pub pointcut:  &'static dyn Pointcut,  // pure predicate over declared metadata
    pub condition: Option<&'static dyn Condition>, // conditions/profiles gate activation at freeze
    pub make_interceptor: fn(&Container) -> BoxFuture<'_, Result<Arc<dyn Interceptor>, ResolveError>>, // resolves aspect bean
}
pub enum Role { Application, Support, Infrastructure } // RoleTier is the PRIMARY ordering key

## Forces on other concepts
- **Scope / Lifetime / Ownership Model (upstream ownership-model ADR)**: Honored unchanged: the proxy wraps an `ErasedBean` via `TargetSource`; the wrap-and-swap IS the after_init/post-process hook returning a possibly-different `ErasedBean`; the generated `OrderProxy` upcasts to `Arc<dyn Order>` (trait upcasting 1.86). The scoped `ScopeTarget` captures `Weak<ContainerCore>` (no Arc cycle) and re-resolves the live instance via `InstanceStore::get_or_init` — the Arc strong-count governs the real free, so in-flight advised work past a scope boundary stays sound. The proxy itself is a safely-published singleton-shaped non-owning re-resolver.
- **Type-Erasure & Registry Substrate (upstream registry-substrate ADR)**: The macro emits each advisable type's `MethodTable` (stable `MethodKey` name-hashes) and per-method attribute-presence into `AnnotationMetadata` (Descriptor.meta) so pointcuts match over DATA, never reflection. The `ProxyPlan` is computed at the registry FREEZE boundary, keyed by `BeanId`. Advisor identity/tiebreak is `ContractId` (stable, collision-guarded), never serialized `TypeId`. The dynamic/WASM lane's `ErasedProxy` is a Provider of the same `ErasedBean`; the registry stays origin-blind on the resolution path.
- **Async Execution & Context-Propagation (upstream async-context-model ADR)**: Builds on 5b (BoxFuture at every dyn `Interceptor`/`TargetSource` seam — no monomorphized default), 5c (@Async stays a `Role::Infrastructure` chain advisor; `ConcurrencyGate` is the shared `@ConcurrencyLimit` primitive), 5a (advice reads the poll-re-installing `Cx` ambient, NOT a ThreadLocal; tx resource is an `Isolate` key never crossing the spawn hop), and 5d (around-advice cancel-safety = sync RAII Drop + container teardown ledger; no async Drop). This ADR must not re-decide any of these — it consumes them.
- **Deferral Primitives & @Lazy / Scoped Proxies**: The HONEST visible-handle shapes (`ScopedRef<dyn T>`, `Provider<T>`, `@Lazy`) live HERE, not in the advice proxy — the transparent-vs-honest split is 'advice = transparent `dyn Svc`; deferral = honest handle in the type.' A purely-deferred dependency uses the visible handle; a scoped-AND-advised bean uses the transparent proxy whose innermost `ScopeTarget` IS the scoped re-resolution (scoped-proxies unified into the substrate). Deferral re-resolution yields `Published` exactly as the ownership-model ADR defined.
- **Auto-Configuration / Cross-Crate Composition & Anti-DCE (owner-decisions #1/#2/#3)**: Advisor descriptors are discovered via the shared cross-crate substrate with NO bespoke registry; the mandatory expected-vs-found self-check MUST cover advisors and the auto-proxy-creator/execution-facility presence (a DCE'd advisor crate = loud diagnostic, never a silent no-op or silent-sync @Async). The `CreatorPolicy` capability lattice (infra-floor upgraded to aspects) is assembled at the binary/app-root where the full enabled-feature set is visible; auto-config back-off and the exclusions mechanism gate which advisors activate (descriptor `condition`).
- **Transaction Management / Caching / Validation / Retry / Exception-Translation (declarative cross-cutting features)**: Each is ONE `Role::Infrastructure` `Interceptor` in the shared ordered chain authored via the SAME `#[advice]` vocabulary (dogfooding). They depend on: `Next::proceed()` being REPLAYABLE (retry calls it N times) and SKIPPABLE (cache short-circuits on hit); typed-arg access via the generated pack/unpack glue (validation/cache-key from a param); sync-RAII cancel-safety across proceed() (tx rollback, permit release); and the correctness-grade chain order (cache-outside-vs-inside-tx, innermost exception-translation) being deterministic and inspectable. Each keeps a proxy-free imperative escape (e.g. `RetryTemplate::execute`).
- **Ordering Primitive (the ONE @Order/@Priority/Ordered comparator)**: Advice ordering consumes the single canonical comparator at CORRECTNESS grade with `RoleTier` as the primary key, then lower-value-wins, then stable `ContractId` tiebreak. The comparator must be deterministic (never link/section/registration order, which is unspecified). Collection-injection's preference-grade use of the same comparator must not relax the determinism advice ordering relies on; the resolved order is surfaced as a §2.9 diagnostic.
- **Circular References & the bean-registry early-reference cache**: The early-proxy (getEarlyBeanReference) sub-tier is RUNTIME-ONLY (AOT disallows cycles) and trivial because the proxy holds a `TargetSource`, not a live target — it materializes over a not-yet-built target with no `.await` and no re-entry into async creation. The frozen `ProxyPlan` makes early-ref == final-proxy automatic (no `earlyProxyReferences` dedup set). The middle/early cache slot earns its keep ONLY for the runtime cycle case; for an advised cyclic bean the early reference handed mid-cycle must be the proxy.
- **Metadata / Codegen Boundary & thin macros**: `#[component]`/`#[advisable]`/`#[aspect]`/`#[advice]`/`#[pointcut]` are THIN: they emit (a) the generated transparent newtype (exactly hand-writable, `cargo expand`-legible), (b) flat const `Descriptor`/`MethodTable`/`AdvisorDescriptor` rows via absolute `::leaf_core` paths, and (c) the `make_interceptor` factory closure — with ALL chain/plan/match/order logic in `leaf-core`. A generic advisable type hard-errors with a `register_proxy!(Concrete)` per-instantiation escape (generics-can't-register). The optional pointcut-expression parser lives in a normal crate, never the macro's hidden logic.
