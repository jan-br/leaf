# ADR 07 — Async Execution & Context-Propagation Model  `[async-context-model]`

> **Note (2026-06-14):** the optional *dynamic-WASM source / WASM-"classpath"* mentioned below is **DROPPED** — leaf autodiscovery is native link-time registration only. Ignore WASM/dynamic-module references here; they are struck from the active design.


## Decision
ADOPT a single async-first execution spine in leaf-core (runtime-naming-free) with five locked sub-decisions. (5a) AMBIENT CONTEXT = ONE poll-re-installing future combinator over ONE leaf-owned task-local, with an explicit Cx value as the always-present escape hatch and the mandatory bridge at AOP/event/schedule seams; NOT a ThreadLocal, NOT tokio::task_local in core. Cx is one immutable cheap-clone Arc-backed bundle (Send+Sync+'static) of typed CxKey slices; an AmbientStore trait is the ONE storage seam (runtime backs it); Scoped<F> re-installs the bundle on EVERY poll (the ThreadContextElement/ScopedValue analogue, correct across work-stealing migration); propagation across a hop is child.scoped(Cx::current()) = one Arc clone + a poll-wrapper. Default ISOLATE; INHERIT keys auto-captured by the executor CxDecorator only when propagation is enabled. The Micrometer accessor SPI is DEMOTED to a CxBridge seam-adapter (tracing scope-stacks, WASM marshalling), not the primary mechanism. Per-key Propagation::Inherit|Isolate realizes the tx-connection never-inheritable rule as a typed default. (5b) ASYNC-ACROSS-DYN = BoxFuture is THE standard at every origin-agnostic dyn seam (forced by the upstream ownership-model Provider::provide -> BoxFuture; AFIT/RPITIT not dyn-compatible regardless of nightly), authored via trait_variant::make(:Send) + dynosaur bridge or thin-macro-emitted Pin<Box<dyn Future+Send>>; monomorphized fast-path at known-concrete sites as a Phase-3 escape; nothing gated on RTN. (5c) EXECUTOR SHAPE = CAPABILITY-SPLIT traits Spawner / BlockingOffload / ConcurrencyGate bundled under one ExecutionFacility role bean, runtime-agnostic in core (impls in leaf-tokio/leaf-smol), selected by primary/qualifier + force-mode; scheduling is a time-aware capability over the SAME Spawner (Trigger SPI + one reactive SchedulerCore), NOT a second executor; #[async] is a Role::Infrastructure dispatch advisor in the shared auto-proxy chain. (5d) CANCELLATION/DRAIN/TEARDOWN-WITHOUT-async-Drop = cooperative cancellation + structured-concurrency drain with per-unit timeouts + explicit container-driven shutdown().await; awaitable SpawnHandle DROP = abort, fire-and-forget detaches; sync cleanup rides RAII Drop (Cx restore, Permit release), async cleanup is NEVER Drop but an awaited TeardownLedger entry drained LIFO by context-close after stop_all(); the upstream Arc strong-count governs the real free; uncaught fire-and-forget failures route to a container-level AsyncUncaughtFailureHandler. (5e) BOUNDS = uniform Send+'static for spawned work and Send+Sync+'static for Cx-propagated values and shared-scope beans (rides the upstream Bean/ErasedBean bound); NO ?Send carve-out in core, steered by a doctrine #[diagnostic::on_unimplemented]; single-threaded ?Send lane deferred to the pluggable-runtime question. (5f) HOT-vs-COLD = reactive on hot paths (wakers/watch/Notify/readiness), polling ONLY on cold paths (retry/backoff/reconnect/refresh/@Scheduled await a reactive timer-wheel, never a spin); run-state observed via watch<RunState>, never an is_running() poll-loop. Cluster 5 is registered.

## Rationale
- 5a: the poll-re-installing combinator over a single task-local is the only model correct across .await under a work-stealing runtime (register 5a / context-propagation R2, the keystone clash charter 3 names); thread_local read-after-await is actively wrong and capture-once-at-submit (R1) leaks/desyncs. It costs one Arc clone per hop and a cheap sync install/restore per poll, far cheaper than R1's per-hop box-dance on hot fan-out (charter 2.5).
- 5a: keeping Cx::with/Cx as an always-present explicit value answers R3's abuse-resistance insight without R3's viral-signature DX tax; the explicit value is the escape hatch and the cleanest WASM-marshal form, while the ambient task-local serves AOP/event/schedule seams that cannot thread a param. This merges R1 storage-SPI, R2 combinator, and R3 value into one composable substrate.
- 5a: ONE opaque bundle over N discovered accessors kills the silent-DCE empty-registry bug (a dropped accessor crate stops propagating with no error) and the baffling-why-context-moves hazard, giving one inspectable bundle schema for the startup self-check (charter 2.9). ISOLATE-by-default plus per-key Propagation realizes propagate-context=false and the tx-connection never-inheritable rule as a typed default.
- 5b: BoxFuture-at-dyn is forced by the upstream ownership-model (origin-agnostic Provider returns BoxFuture; AFIT/RPITIT not dyn-compatible regardless of nightly). Making it framework-wide via trait_variant+dynosaur keeps a zero-nightly core (charter 2.8) and honors thin macros (charter 2.10); gating nothing on RTN respects its avoid status.
- 5c: the capability split makes execution context a type-level property (charter 2.3) so a bean depending on BlockingOffload advertises that it blocks; ConcurrencyGate becomes a first-class injectable reused by retry/scheduling (charter 2.11). Bundling under one role bean preserves Spring's single applicationTaskExecutor identity and the swap-by-config one-switch with the trait naming no runtime (charter 2.6).
- 5c: folding scheduling into a time-aware capability over the same Spawner is the unification the register asks for (one spawn substrate, one shared reactive timer-wheel) and structurally defeats the single-thread-serialization gotcha by separating WHEN from WHERE. #[async] staying an advisor in the shared chain preserves correctness-critical ordering vs #[transactional] that body-rewrite would make ill-defined.
- 5d: async Drop is experimental-incomplete with a tokio-shutdown crash and no dyn story; the charter mandates designing around it. Explicit shutdown().await draining the upstream TeardownLedger LIFO after stop_all() gives deterministic stop-before-destroy order, per-bean timeouts, and a ShutdownReport that Drop cannot. Cooperative cancellation + JoinSet drain (never a parked thread/latch) honors 2.3; RAII handles only the safe sync case.
- 5d: the upstream Arc strong-count (not an arena slot) governing the real free is what makes in-flight work referencing a scoped bean past its boundary sound; the ledger drains when straggler clones release. Ownership supplies the soundness, this model supplies the awaited drain, so the two are co-decided.
- 5e: uniform Send+Sync+'static resists the sync/async bimodality charter 2.3 forbids and rides the bound where the upstream ownership-model put it (one leaf-core location), so the safe-publication contract is satisfied by the handle type. A multi-thread runtime is the only contract a swap-by-config can preserve without compile-breaking on a ?Send carve-out; the wall is steered by a doctrine diagnostic rather than a raw solver error.
- 5f: section 1.9 / charter 2.4 is enforced by primitive choice not convention: wakers/watch/Notify on hot paths and a reactive timer-wheel (never a spin) on sanctioned cold paths. Load-bearing for scheduling (trigger must be sleep_until+waker), retry (backoff awaits a timer), and run-state readiness (subscribe to watch<RunState>, never poll is_running).

## Options considered & rejected
- **5a: Micrometer-faithful runtime accessor-registry + ContextSnapshot + RAII restore-Scope as the PRIMARY mechanism (context-propagation R1)** — rejected: A one-shot restore-before-run is insufficient across work-stealing migration (a value read after an .await that resumed elsewhere is wrong unless re-installed per poll, pushing toward R2 anyway), and the N-accessor registry is DCE-prone (a dropped accessor crate silently stops propagating with no error). It also pays a per-hop box-dance over a Vec. Retained only as the demoted CxBridge seam-adapter for scope-based holders (tracing) and WASM marshalling.
- **5a: Explicit Cx-everywhere as the PRIMARY model, ambient only at seams (context-propagation R3 / locale-context R2)** — rejected: Viral signatures: every method that needs or merely passes through the current X must take a Cx param, the API pollution Spring's holder was invented to avoid (DX is priority 1). The seams that most need context (AOP proxies, async events, scheduling) cannot thread a param, so R3 cannot escape the ambient mechanism and maintains two models that can desync. Kept as the escape hatch and WASM-marshal form, not the default.
- **5a: tokio::task_local! named directly in leaf-core** — rejected: Violates charter 2.6 runtime-agnosticism (core must name no runtime) and forecloses smol/single-thread backings. The AmbientStore trait with runtime-provided storage costs one trait + mild indirection but keeps the swap-by-config promise.
- **5b: Monomorphized zero-cost async as the PRIMARY surface, dyn only as escape** — rejected: Impossible at the origin-agnostic seams that define leaf (Provider, executor, context bridge, lifecycle participants, WASM proxy are all dyn) because AFIT/RPITIT are not dyn-compatible regardless of nightly. The boxed future is unavoidable there; monomorphization is the Phase-3 fast-path, not the default, which would abandon origin-agnosticism.
- **5c: One fat dyn TaskExecutor trait (task-execution R1)** — rejected: Conflates three distinct execution contexts (spawn / blocking offload / concurrency cap) with different safety contracts, hiding may-block behind a do-everything interface rather than the type signature, and cannot carry generic submit ergonomically. The split keeps the single named role bean while making each capability independently injectable and reusable.
- **5c: Execution-policy-as-data dispatcher + named ExecutionProfile config (task-execution R3)** — rejected: A missing/misnamed profile is a runtime/startup error, the executor stops being a primary/qualifiable bean (selection moves out of DI into a config table), and a per-spawn lookup+branch adds hot-path indirection. The config-tuning win is recovered by binding the ExecutionFacility knobs from the Environment without data-routing the dispatch itself.
- **5c: @Async via macro body-rewrite (async-execution R2), killing the self-invocation gotcha** — rejected: It leaves the shared auto-proxy chain, making #[async] ordering vs #[transactional]/#[cacheable] ill-defined (does the tx wrap the spawn or run inside it), and silently changes Spring-ported code relying on self-calls running sync. Kept as a possible Phase-3 hybrid (body-rewrite for dispatch + a chain marker for ordering), not the committed default.
- **5d: Ride RAII Drop for async teardown (async @PreDestroy via AsyncDrop)** — rejected: async Drop is experimental-incomplete with a reported tokio-shutdown crash and no dyn-Trait story; leaf's destroyers are dyn-erased and sync Drop cannot await a pool flush / socket close. Hence explicit container-driven shutdown().await over the awaited TeardownLedger; sync-only cleanup (Cx restore, Permit release) does ride Drop where it is cancellation-safe.
- **5e: Two surfaces (Send and ?Send) or accept compile-breaking runtime swaps** — rejected: A ?Send carve-out re-introduces the sync/async bimodality charter 2.3 forbids and doubles the trait surface; accepting that a swap can break #[async] at compile time contradicts swap-by-config. RTN would express the bound generically but is avoid (ownerless, PR closed). Uniform Send+Sync+'static now, with the single-threaded lane deferred behind the pluggable-runtime seam, is the only consistent posture.

## Consequences
- FORCES every async dyn seam to be authored through the BoxFuture standard (trait_variant + dynosaur bridge, or thin-macro-emitted Pin<Box<dyn Future+Send>>): the upstream Provider, the Spawner/BlockingOffload/ConcurrencyGate capabilities, Lifecycle::start/stop_graceful, the AOP interceptor chain, the Shutdown trait, and the WASM host-proxy. Monomorphized fast-path only at known-concrete sites (Phase-3). Cost: one box per spawn / advised call / prototype-or-scoped resolution; cold for singletons.
- FORCES a leaf-owned AmbientStore trait in core whose storage is runtime-provided (leaf-tokio via tokio::task_local!, leaf-smol via its own): a mandatory-but-degraded Layer-0 crate. If the runtime crate is DCE'd, propagation has no backing store (context stops moving), so the startup self-check must list active context backing + the inspectable Cx bundle schema and WARN loudly (degraded-not-fatal, unlike the missing executor).
- FORCES the executor/runtime integration crate to be a force-linked, self-checked, MANDATORY dependency: an async-first framework with no ExecutionFacility cannot run, so the facade self-check must HARD-FAIL at refresh if no primary application facility is found. leaf-boot should force-link the default runtime crate. Couples to the cross-crate anti-DCE mechanism and owner-decision 1 (maximal-magic discovery, no declared scan-scope).
- FORCES uniform Send+'static on spawned work and Send+Sync+'static on Cx-propagated values and shared-scope beans (stricter than Spring): a non-Send #[async] body, non-Sync singleton, or non-Send-Sync context value is a compile error, absorbed by doctrine #[diagnostic::on_unimplemented] messages (make it prototype/request-scoped; move blocking work to BlockingOffload), never a raw solver error. The single-threaded ?Send lane is deferred.
- FORCES the cancellation contract framework-wide: awaitable SpawnHandle DROP = abort (cancel at next await); fire-and-forget #[async]/async-event/scheduled-fire detaches (outlives caller, drained at shutdown, NOT cancelled on caller-drop), documented because a detached task can outlive the drain window and is then aborted. Uncaught fire-and-forget failures route to a single container-level AsyncUncaughtFailureHandler, composing with the separately-decided error model.
- FORCES teardown to be an explicit awaited pass, never Drop: context-close runs CAS-close-once, publish Closed event, runtime-lifecycle.stop_all() (reverse-phase, per-phase timeout, JoinSet drain), then drain the upstream TeardownLedger LIFO under per-bean timeouts with ShutdownReport aggregation. Every bean/resource with async cleanup puts it in its destroyer (not Drop) and tolerates destroy-without-stop. The Arc strong-count governs the real free, so context-scope teardown waits out straggler clones up to the deadline.
- FORCES the 1.9 reactive/poll rule onto every feature's primitive choice: scheduling triggers, retry/backoff, reconnect, periodic refresh must await a reactive timer (one shared timer-wheel), never an interval spin; run-state readiness must be watch<RunState> subscription, never an is_running() poll-loop; per-phase/per-bean drains await completion notifications, never poll. A busy-wait on a hot path is a charter violation.
- FORCES the per-key propagation policy to be a typed property of each CxKey (Inherit vs Isolate): locale/trace are Inherit; the transaction/connection resource is Isolate (the never-inheritable rule as a typed default, so it cannot silently ride to a child task and shatter isolation). Transaction-management's resource binding lives in an Isolate CxKey read ambiently within one logical task, never auto-captured by CxDecorator.
- ACCEPTS that scope-based third-party concerns (tracing/observation: open-scope, NullObservation masking) do not map onto a value bundle; they are served by CxBridge adapters that own their scope push/pop inside the bridge, not the core Cx bundle. Full scope-masking parity lives in the bridge layer, an explicit non-uniformity bounded to observability integration.
- ACCEPTS a real allocation cost on hot paths (box-per-spawn, box-per-advised-call, Arc-clone-per-hop, per-poll Cx install/restore), named not hidden. Mitigated in order: (a) constructor-injected collaborators resolved once and stored typed (steady-state calls touch no box); (b) prototype exempt from teardown/refcount (upstream owned-move); (c) skip-the-install when Cx unchanged across a poll; (d) the Phase-3 monomorphized fast-path. The dyn default is not eliminated.

## API / type sketch

// leaf-core (names NO runtime; ultra-stable)
// 5a. AMBIENT CONTEXT - one cheap-clone bundle (NOT N accessors), task-bound (NOT ThreadLocal)
#[derive(Clone)] pub struct Cx(std::sync::Arc<CxNode>);              // Send + Sync + 'static
pub trait CxKey: 'static {
    type Value: Send + Sync + 'static;
    const NAME: &'static str;                                       // 'no current X' diagnostics + inspection
    const POLICY: Propagation;                                      // Inherit (locale/trace) | Isolate (tx/conn)
}
pub enum Propagation { Inherit, Isolate }
impl Cx {
    pub fn get<K: CxKey>(&self) -> Option<&K::Value>;               // typed read, no alloc
    pub fn with<K: CxKey>(&self, v: K::Value) -> Cx;               // derived child (one-directional shadow)
    pub fn current() -> Option<Cx>;                                // read ambient (clone of bundle)
    pub fn enter<R>(&self, f: impl FnOnce() -> R) -> R;            // explicit ambient install for ONE sync region
}
pub trait AmbientStore: Send + Sync {                              // the ONE storage seam; runtime backs it
    fn with_installed<R>(&self, cx: &Cx, f: &mut dyn FnMut() -> R) -> R;  // sync install/restore for ONE poll
    fn current(&self) -> Option<Cx>;
}
pub struct Scoped<F> { /* cx + inner; poll = store.with_installed(&cx, || inner.poll(cx)) - EVERY poll */ }
pub trait CxFutureExt: std::future::Future + Sized { fn scoped(self, cx: Cx) -> Scoped<Self>; }
pub trait CxBridge: Send + Sync { fn project(&self, cx: &Cx) -> BridgeGuard; }   // demoted seam-adapter; sync Drop = pop

// 5b. ASYNC-ACROSS-DYN - BoxFuture is the standard at every origin-agnostic seam
pub type BoxFuture<'a, T> = std::pin::Pin<Box<dyn std::future::Future<Output = T> + Send + 'a>>;
// authored via #[trait_variant::make(Spawner: Send)] + dynosaur bridge, OR thin-macro-emitted BoxFuture.
// (consistency w/ upstream ownership-model: Provider::provide(..) -> BoxFuture<Published>.)

// 5c. EXECUTOR - capability split, bundled under ONE role bean; runtime-agnostic
pub trait Spawner: Send + Sync { fn spawn(&self, fut: BoxFuture<'static, ()>) -> SpawnHandle; }
pub trait BlockingOffload: Send + Sync { fn run_blocking(&self, f: Box<dyn FnOnce() + Send + 'static>) -> BlockingHandle; }
pub trait ConcurrencyGate: Send + Sync { fn acquire(&self) -> BoxFuture<'static, Permit>; }  // Permit RAII Drop releases (cancel-safe)
pub trait CxDecorator: Send + Sync { fn capture(&self) -> Cx; }                              // opt-in propagation at submit
pub struct ExecutionFacility { /* impl Spawner+BlockingOffload+ConcurrencyGate over one runtime; primary application role bean */ }
pub trait Trigger: Send + Sync { fn next_fire(&self, last: TriggerContext) -> Option<Instant>; }  // SYNC timing (no boxing)
// scheduling = time-aware capability over the SAME Spawner (NOT a 2nd executor): Trigger SPI + one reactive SchedulerCore.

// 5d. CANCELLATION / DRAIN / TEARDOWN - explicit, no async Drop
pub struct SpawnHandle { /* await => Result<T, JoinError>; DROP => ABORT (cancel at next await) */ }
pub enum DropPolicy { Abort, Detach }                                         // fire-and-forget #[async] = Detach
pub trait Shutdown { fn shutdown(&self, deadline: Deadline) -> BoxFuture<'_, ShutdownReport>; }
pub trait Lifecycle: Send + Sync {
    fn start(&self) -> BoxFuture<'_, Result<(), LifecycleError>>;
    fn stop_graceful(&self) -> BoxFuture<'_, Result<(), StopError>>;          // future resolving = drained (no Runnable/latch)
    fn is_running(&self) -> bool;                                             // cold-path point query
}
// context-close drains the upstream ownership-model TeardownLedger LIFO under per-bean timeouts, after stop_all().
pub struct AsyncUncaughtFailureHandler(/* container-level sink for detached-task panics/errors - never swallowed */);

// 5e. BOUNDS - uniform; rides the upstream Bean/ErasedBean bound; doctrine diagnostic, not a raw solver error
#[diagnostic::on_unimplemented(
  message = "async work `{Self}` must be Send + 'static to run on leaf's executor; \

pub trait SpawnableWork: Send + 'static {}

pub fn watch_run_state() -> WatchReceiver<RunState>;                          // subscribe; NEVER poll is_running in a loop
// retry/backoff/scheduling await the shared reactive timer-wheel (sleep_until + waker), never an interval poll.

## Forces on other concepts
- **Scope / Lifetime / Ownership Model (upstream ownership-model)**: This model consumes that ADR and must not contradict it: the per-context InstanceStore is reached via the Cx ambient (NOT ThreadLocal); the Published/Provider::provide -> BoxFuture seam IS the 5b boxing standard; teardown is the explicit async drain of that ADR's TeardownLedger (no async Drop); deferral handles capture Weak<ContainerCore>. The Arc strong-count (not a slot index) governing the real free is what makes in-flight scoped work past a boundary sound under cooperative cancellation - co-decided.
- **Container Concurrency Contract (06 concurrency-contract)**: The uniform Send+Sync+'static bound (5e) rides ErasedBean/Bean in one leaf-core location - the safe-publication contract is satisfied by the handle type, not re-expressed per feature; the atomic Arc clone is the happens-before edge. The doctrine #[diagnostic::on_unimplemented] for the spawn/shared bound is owned here. No global lock anywhere: serialization is by single-task ownership / atomic CAS, never a mutex held across an await.
- **Bean Scopes and context-scope SPI**: Request/session/custom context-scope InstanceStores are bound to the Cx ambient and survive .await via the Scoped poll-wrapper; a scope opens at a Cx::with/scope boundary and its ledger drains at scope-end via the explicit async teardown. Scope teardown-on-cancellation must run destruction callbacks even when a straggler Arc clone outlives the boundary (bounded by the deadline). Custom scopes pick where instances live; they never define a new ambient mechanism.
- **AOP / Interception Substrate and Declarative Async**: #[async] is a Role::Infrastructure dispatch advisor in the shared auto-proxy chain (not a body-rewrite by default), so its order vs #[transactional]/#[cacheable] is governed by the chain's deterministic sort (a tx must not span an async hop). The advised-call path eats the 5b box-per-hop. Interception seams cannot thread a Cx param, so they read the ambient Cx (the explicit-value escape hatch is for user call graphs). Cancel-safety across proceed() mid-chain follows the cooperative-cancel contract.
- **Events and Event Multicaster**: Sync (default) dispatch runs on the caller's task and preserves Cx naturally; async dispatch routes listeners through the Spawner and carries context only when the CxDecorator is enabled (ISOLATE-by-default). Per-listener async failures route to the same container-level AsyncUncaughtFailureHandler. No return-value chaining on async listeners (the result is produced after the publisher returned) - the fire-and-forget detach semantic.
- **Runtime Lifecycle and Context Close**: Lifecycle::stop_graceful -> BoxFuture (the future resolving = drained, no Runnable/latch); phase groups drain via structured-concurrency JoinSet under per-phase timeout (never a parked thread). Context-close: CAS-close-once, Closed event, stop_all() (stop-before-destroy), LIFO TeardownLedger drain under per-bean timeouts, ShutdownReport. Every Lifecycle/disposable bean must tolerate destroy-without-stop. Run-state observed via watch<RunState>, never an is_running() poll-loop (5f).
- **Transaction Management and the never-inheritable resource**: The transactional connection/resource binding is an Isolate CxKey read ambiently within one logical task and never auto-captured by CxDecorator (the never-inheritable rule as a typed default) - propagating it to a child task would shatter isolation. Transaction finalization (rollback) under future-drop has no async-Drop RAII: it is explicit awaited cleanup driven by the interceptor's around-advice / the teardown ledger, bounded by cooperative cancellation.
- **Cross-Crate Composition and Anti-DCE (owner-decision 1)**: The runtime/executor integration crate is a mandatory force-linked Layer-0 dependency (missing executor = fatal at refresh self-check); the AmbientStore backing is mandatory-but-degraded (missing = context stops moving, WARN). With maximal-magic discovery and no declared scan-scope, both must surface via the expected-vs-found self-check naming the missing source. The Cx bundle schema + active executor/decorator set must be startup-inspectable (counters the baffling-why-context-moves hazard).
- **Metadata / Codegen Boundary and thin macros**: The thin #[async]/#[holder]/#[transactional] macros emit only flat const descriptors + the hand-writable BoxFuture spawn via absolute ::leaf_core paths; the dispatch/decorator/drain logic lives in leaf-core (charter 2.10). CxKey declarations and the executor role bean are ordinary code a user could hand-write. No public API is gated on nightly RTN - the trait_variant/dynosaur/Box layer is an internal seam for a later localized swap.
- **Retry / Resilience and Scheduling timers**: ConcurrencyGate is the ONE bounded-permit primitive shared by task-execution, retry (@ConcurrencyLimit, limit-1 = declarative instance lock), and scheduled-fire overlap - not solved per-feature. Retry/backoff, reconnect, periodic refresh, and @Scheduled triggers share ONE reactive timer-wheel and are the only sanctioned cold-path poll loops (5f); the permit releases on cancel via RAII even across backoff sleeps.

## Open sub-questions (→ Phase 3)
- The single-threaded / current-thread ?Send lane: whether to offer a relaxed bound at all for a current-thread runtime backing, and how to do so behind the pluggable-runtime seam without re-introducing the sync/async bimodality charter 2.3 resists. Deferred to the pluggable-runtime decision; measure demand before building.
- The skip-Cx-install-when-unchanged optimization: whether the per-poll with_installed swap can be elided when the bundle is identical to the currently-installed one (a cheap Arc-ptr compare), and whether that is measurable as worth it on deep-await hot paths. Performance-driven, Phase 3.
- The @Async body-rewrite hybrid: whether a Phase-3 variant compiles dispatch into the method body (killing the self-invocation gotcha) while still emitting a chain marker so ordering vs #[transactional] stays governed by the shared sort. A DX-vs-parity call deferred until the full AOP cluster is re-unified.
- Exact nested-context-scope semantics (request nested in session): how the Cx/AmbientStore models parenting of stores and the precise teardown-on-cancellation ordering so request- and session-destruction callbacks still run when in-flight work holds a straggler Arc clone past the boundary. Feature-local to scopes.
- WASM host-proxy context marshalling + teardown: how the proxy reads the ambient Cx and lowers INHERIT keys into the WIT call, and how a stateful guest resource's destruction interacts with the TeardownLedger/shutdown ordering and trap/failure semantics. Feature-local to the WASM layer; the erased-uniformity must not assume native-only teardown.
- The cron calendar engine + missed-fire semantics (dialect, timezone/DST, fire-all-missed vs once vs skip after a clock-jump) and whether the cron parse lives in a macro / build.rs / startup pass - the rare CPU-bound bit on an otherwise reactive cold path. Feature-local to scheduling.
- Whether the single shared reactive timer-wheel is one global driver (maximally reactive, single point of failure) or per-scheduler, and the precise composition of fixedDelay completion-feedback across the spawn boundary. Feature-local to scheduling/retry, measured.
- The AsyncUncaughtFailureHandler granularity (single container-level vs per-executor vs per-#[async]-method) and exactly how it composes with the separately-decided unified error/diagnostics model. Deferred to that ADR's resolution.
