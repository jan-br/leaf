# leaf Architecture Toolkit (Phase 2)

## The shape of leaf in one page

leaf is a Spring-grade dependency-injection and application framework for Rust whose entire architecture hangs off **one shared-bean currency**. Everything that is shared inside the container is an `ErasedBean = Arc<dyn Any + Send + Sync>`, sugared at typed boundaries as `Ref<T>` (over `dyn Svc` or a concrete type, using trait upcasting stable since 1.86). There is exactly **one** shared-handle type; the genuine ownership divergence rides a tiny total `Published` enum produced at the publish step — `Shared(ErasedBean)` for singleton + every context-scope instance, `Owned(Box<dyn Any + Send>)` for prototype (a real owned move, never refcounted, never stored, no teardown). Scope is **data on three orthogonal axes** — `Multiplicity{Once,PerResolution,PerContextKey}` × `StoreSource` × `TeardownPolicy` — never a second handle type. The `Send + Sync + 'static` safe-publication contract rides `ErasedBean`/`Bean` in *one* leaf-core location; the atomic Arc clone *is* the happens-before edge, so there is no singleton mutex and no global lock.

Construction is origin-agnostic through a single dyn seam: `Provider::provide(&cx) -> BoxFuture<Result<Published, ResolveError>>` (boxed because AFIT/RPITIT are not dyn-compatible — true regardless of nightly). Native beans, `FactoryBean` products, test doubles, and the WASM host-proxy *all* implement `Provider` and publish the identical `ErasedBean` — **"WASM-ness stops at the proxy"**; the container cannot tell origins apart because the stored shape is one type.

Those erased values live in a **frozen, dense-`BeanId(u32)`-keyed `Registry`**. Keying is **TypeId-primary, name-overlay**: by-type resolution is the common string-free path; a small name overlay (`IndexMap<BeanName, BeanId>` + aliases) serves the five Spring semantics that genuinely need a string identity (aliases, name-collision-as-error, name-keyed `Map<String,T>` injection, `&FactoryBean` deref, name-as-implicit-qualifier). Cross-build / cross-crate / persisted identity keys on a **stable author-assigned `ContractId(u64)`** (a const hash, collision-guarded at freeze) — *never* a `TypeId`, which is not stable across compiler builds. The singleton store is `singletons: Box<[OnceCell<ErasedBean>]>` indexed by `BeanId.0` (lock-free ready reads, at-most-once init by data shape). "Injectable as any supertype" is macro-emitted `TypeRow` upcast rows; a concrete handle matches by exact `TypeId` only.

The pipeline that fills that registry is **"emit data, never logic; collect by link; assemble at runtime; codegen only at the binary crate."** A thin proc-macro (`#[component]`/`#[bean]`/`#[configuration]`/`#[auto_config]`) expands to exactly **one hand-writable const `Descriptor` row + a const `ProviderSeed`**, registered into a `linkme` distributed slice via absolute `::leaf_core` paths — `cargo expand` shows one legible const, and what the macro emits equals what a user could write to `RegistryBuilder::register`. Those rows collect at link time (zero life-before-main) into `&'static [Descriptor]` slices. The binary crate (via `#[leaf::main]` + a generated `build.rs`) does the three jobs only it can: **force-link** every participating crate (`use leaf_redis as _;`), emit an **`ExpectedManifest`** for the anti-DCE self-check, and optionally pre-plan auto-config at build time. Runtime assembly then happens **around main**, cold and synchronous, before any `.await`.

That assembly is a **four-stage phase machine** `App<Define> → App<Resolve> → App<Wired> → App<Running>`, whose single load-bearing invariant is: *all definition-shape decisions freeze into one immutable `FrozenContainer` at a single `seal()` boundary, before any async/IO/construction; all value-shape work happens after.* `Define` lifts the slices and runs the anti-DCE self-check; `Resolve` (cold, sync) runs the **condition tier-router** and the **auto-config deferred-batch**; `seal()` freezes the dense `Registry`; `App<Wired>::validate()` runs a **whole-graph startup-validation pass** that resolves every mandatory injection point and aggregates *all* `NoSuchBean`/`NoUniqueBean`/scope-mismatch/cycle failures into one `AssemblyReport` before a single constructor runs; `start().await` is the async fence, after which beans eager-instantiate wave-parallel and hot reads are lock-free against the immutable snapshot.

Conditions are **one typed `CondExpr` algebra** over a stable `ConditionId`, routed by a fixed **per-kind tier-map**: `Cfg`-tier (`OnClass`/`OnCapability`) is pruned at compile and paired with force-link; `ConstFold`-tier folds at build; the **`Runtime` tier is the mandatory floor** (`OnBean`/`OnMissingBean`/`OnProperty`/`OnProfile` + every dynamic/WASM-contributed def with no cfg and no Rust type). Profiles are just the most ergonomic `CondExpr` preset, always runtime. **Auto-configuration** is Boot-style opt-in: an `#[auto_config]` module emits descriptors into a dedicated `AUTO_CONFIGS` slice; the engine runs in `App<Resolve>` in the ladder **`exclude(ContractId) > user-bean back-off (OnMissingBean) > auto-config default`**, registering candidates incrementally so each later one sees the growing definition set. Auto-config beans register at `@Fallback` role so a user bean transparently supersedes. **Discovery** is maximal-magic but *not* a true classpath scan: participation = enabled `dep:`-hidden capability feature ∪ explicit `scan!` list (never an arbitrary transitive dep), guarded by a **triple anti-DCE defense** (force-link + section-retention toolchain guardrails + the expected-vs-found `ExpectedManifest` self-check that turns a silently-dropped crate into a loud `AntiDceError::SourceVanished` naming the crate). **Starters** are `leaf-starter-*` aggregator crates behind umbrella features; the **BOM** is `[workspace.dependencies]` internally (one copy of `leaf-core`, so one `AUTO_CONFIGS` slice) plus the version-pinned `leaf` umbrella as the downstream coordination point.

**Async** is single-mode (no sync/async bimodality): `BoxFuture` at every dyn seam; one capability-split executor (`Spawner`/`BlockingOffload`/`ConcurrencyGate` under one `ExecutionFacility` role bean, runtime-agnostic in core); **ambient context** is one cheap-clone `Cx` bundle re-installed per poll over a leaf-owned task-local (never a `ThreadLocal`), with per-key `Propagation::{Inherit,Isolate}` (the transaction resource is `Isolate`); teardown is an explicit container-driven `TeardownLedger` drain (**no async Drop**), with sync RAII for cancel-safe cleanup; reactive primitives on hot paths, a shared reactive timer-wheel on sanctioned cold paths.

**Interception** is one compile-time-generated transparent delegating newtype per service trait (the rust-native CGLIB analogue) routing through one ordered, replayable, short-circuitable `Interceptor` chain whose innermost step is a `TargetSource` (the per-call target supplier that unifies singleton, scoped re-resolution, and `@Lazy`). AOP, tx, cache, validation, retry, `@Async`, and exception-translation are all just chain entries; advice is **transparent** (`dyn Svc`), while the deferral family (`Lookup`/`LazyRef`/`Inject`) is the **honest visible handle**. **Errors** are one `LeafError` causal chain (a typed `ErrorKind` + a `Cause` narrative + an `Origin`, `std::error::Error`-compatible) rendered through one `Diagnostic` trait, with a `FailureAnalyzer` SPI as the rendering-policy layer (not a parallel error system) and four timing tiers (compile → build/seal → `App<Wired>` aggregated → runtime). The whole thing builds on **stable Rust** — no public API gates on nightly.

## The load-bearing decisions

| ADR id | One-sentence decision |
|---|---|
| `ownership-model` | One uniform erased shared handle (`ErasedBean = Arc<dyn Any+Send+Sync>`, sugared `Ref<T>`) with a two-arm `Published{Shared,Owned}` enum carrying the divergence; scope is three data axes, never a second handle type. |
| `registry-substrate` | A two-phase, freeze-once registry whose value is `ErasedBean` and whose keying is dual TypeId-primary / name-overlay over a dense `BeanId(u32)` slot space, with cross-build identity on a stable `ContractId`. |
| `async-context-model` | A single async-first spine: `BoxFuture` at every dyn seam, one capability-split runtime-agnostic executor, ambient `Cx` re-installed per poll (not ThreadLocal), explicit ledger teardown (no async Drop), uniform `Send+Sync+'static`. |
| `codegen-boundary` | A four-layer "emit data not logic / collect by link / assemble at runtime / codegen only at the binary crate" pipeline with thin macros emitting one const `Descriptor` + `ProviderSeed` via `linkme`. |
| `container-shape` | Two named types — one concrete `Engine` (bare, inert) HAS-A'd by one concrete `Context` (auto-detects infrastructure on `refresh()`) — with everything-is-a-`Provider` resolved as ONE creation primitive plus a single concrete creation driver. |
| `proxy-substrate` | One compile-time-generated transparent delegating newtype per service trait, routing through one ordered/replayable/short-circuitable `Interceptor` chain whose innermost step is a `TargetSource`; service trait mandatory for advised beans. |
| `compile-runtime-split` | A four-stage `App<Define→Resolve→Wired→Running>` phase machine with a single `seal()` freeze boundary and a whole-graph startup-validation pass producing one aggregated `AssemblyReport`. |
| `cross-crate-discovery` | Maximal-magic linkme discovery behind a uniform `&[Descriptor]`, made robust by a triple anti-DCE defense (force-link + toolchain guardrails + expected-vs-found self-check) and ContractId-keyed exclusions. |
| `injection-mechanics` | A single fixed-order traced layer-fold selection spine, one `cmp_order` comparator, single-phase construction (factory params are the only injection surface), and deferral-edge-only cycle breaking. |
| `error-model` | One `LeafError` causal-chain type + one `Diagnostic` renderer + one `FailureAnalyzer` rendering-policy SPI across four timing tiers, with a closed `ErrorKind` plus one open `Integration{ContractId}` arm. |
| `conditional-strategy` | One typed `CondExpr` algebra over a stable `ConditionId` with a fixed per-kind tier-map (cfg/const-fold/runtime, runtime being the mandatory floor); profiles are a runtime `CondExpr` preset. |
| `autoconfig-starters-bom` | Boot-style opt-in auto-config (dedicated `AUTO_CONFIGS` slice, `exclude > back-off > default` ladder, `@Fallback` role), `leaf-starter-*` aggregator crates, and a dual `[workspace.dependencies]`+umbrella BOM. |

## How the pieces compose

The journey from source to a running container is one continuous pipeline; each ADR plugs in at a named seam.

1. **Developer writes `#[component]` / `#[bean]` / `#[auto_config]`.** The thin macro (`codegen-boundary` Layer 1) emits exactly one const `Descriptor` — `self_type` via `const TypeId::of` (1.91), `ContractId` via const hash, one `TypeRow` upcast per declared injectable supertrait (`registry-substrate`), the flat `AnnotationMetadata`, the `ScopeDef` triple (`ownership-model`), a const `InjectionPlan` of `InjectionPoint` rows (`injection-mechanics`), and a `CondExpr` guard tree (`conditional-strategy`) — plus a `ProviderSeed` (a const fn-pointer that *builds* the `Provider`, never a live object). Generic beans hard-error with a `register_component!(Concrete)` hint; the `Send+Sync+'static` bound and malformed `CondExpr`/`${...}` are compile-tier errors (`error-model` Tier 0).

2. **Link time.** Each row collects into its `linkme` distributed slice — `COMPONENTS`, `AUTO_CONFIGS`, `CONDITIONS`, `FAILURE_ANALYZERS`, the bootstrap run-participant streams — with zero life-before-main (`codegen-boundary` Layer 2, `cross-crate-discovery`).

3. **Build time, binary crate only.** `#[leaf::main]` + `build.rs` emit force-link references for the participating set (enabled capability features ∪ `scan!`), the const `ExpectedManifest` of `SourceTag`s, and optionally the `cargo leaf prepare` auto-config plan (`codegen-boundary` Layer 3, `cross-crate-discovery`, `autoconfig-starters-bom`). Toolchain guardrails (MSRV ≥ 1.89 or `-z nostart-stop-gc`, pinned `linkme`, `resolver="3"`) are shipped requirements.

4. **`App<Define>` (around main, cold, sync).** `RegistryBuilder::from_slices` lifts the rows; `self_check(&EXPECTED)` runs the **anti-DCE expected-vs-found** pass — a crate in `EXPECTED` contributing zero rows is a loud `AntiDceError::SourceVanished` naming it (`cross-crate-discovery`). `seal_environment().await` ends the only pre-freeze async fence (5f), snapshotting the `Env` (`async-context-model`, `compile-runtime-split`).

5. **`App<Resolve>` (cold, sync).** The **condition tier-router** evaluates the runtime-tier `CondExpr` leaves (Parse sub-pass over the sealed env, then Register sub-pass) and the **auto-config deferred batch** runs in the `exclude > back-off > default` ladder, registering candidates incrementally so each `OnMissingBean`/`OnSingleCandidate` (a no-instantiation `DefinitionProbe`) sees the growing set (`conditional-strategy`, `autoconfig-starters-bom`). Every outcome is recorded into the `ConditionReport`.

6. **`seal()`.** `RegistryBuilder::freeze()` materializes the dense `BeanId` space, both indices, the slot-indexed `singletons: Box<[OnceCell<ErasedBean>]>`, and runs the `ContractId` collision guard — producing the immutable `FrozenContainer` (`registry-substrate`, `compile-runtime-split`). The typestate consumes mutable access, so in-place edit of a sealed epoch is unrepresentable.

7. **`App<Wired>::validate()`.** The **whole-graph startup-validation pass** runs `Selector::resolve_one` (the fixed-order traced layer-fold, `injection-mechanics`) over every eager single-valued `InjectionPoint`, classifies the construction graph for cycles, and aggregates *all* failures into one `AssemblyReport` (`compile-runtime-split`, `error-model` Tier 2). A `NoSuchBean` is enriched from the frozen `ConditionReport` + anti-DCE manifest ("backed off because…" / "feature X not enabled" / "crate expected but produced zero rows").

8. **`Context::refresh()` → `start().await` → `App<Running>`.** `Context` auto-detects `Role::Infrastructure` providers in its fixed-order refresh template, then eager-instantiates non-lazy singletons wave-parallel per the `WiringPlan` via `Engine::create` (`construct → populate → init → publish` through the `Provider`/`Published` seam). At the `after_init` decoration hook the auto-proxy creator does an O(1) `ProxyPlan` lookup and installs the transparent newtype proxy where advised (`proxy-substrate`). Hot resolution reads the frozen snapshot lock-free; constructor-injected collaborators are resolved once and stored as typed `Ref<T>` fields, so steady-state calls touch no refcount and no downcast.

9. **Steady state and teardown.** Deferral handles (`Lookup`/`LazyRef`/`Inject`) re-resolve on demand through the one `resolve(Strictness, Cardinality)` driver, capturing `Weak<ContainerCore>` (no Arc cycle). The dynamic/WASM lane enters via `reopen_epoch()` → re-`seal()` → `arc_swap` of a fresh `FrozenContainer` epoch (static rows precede dynamic in BeanId order). `shutdown().await` drains the `TeardownLedger` LIFO after `stop_all()` — no async Drop; the Arc strong-count governs the real free, so in-flight scoped work past a boundary stays sound.

## Settled unification calls

**The Provider unification — one creation primitive, one concrete driver.** "Everything is a Provider" resolves to: there is exactly *one* creation/registry primitive, the `ownership-model`'s `Provider` trait, and a *single concrete* (non-pluggable) creation driver, `Engine::create`. `FactoryBean` is one `Provider` impl (no second registry slot; the factory stays a full bean reachable by `BeanKey::ByName`). The bean-registry "swappable-strategy SPI" and the bean-instantiation "pluggable-creation-strategy" are **rejected** — merge/store/guard/resolver are each one concrete implementation already fixed upstream, so a `Box<dyn>` kernel SPI would only re-open settled decisions, widen the ultra-stable ABI, and kill the monomorphized fast path. The consumer-side jakarta-`Provider<T>` family was renamed `Lookup<T>`/`LazyRef<T>`/`Inject<T>` to free the name `Provider` for the construction seam — a name collision, not a primitive collision.

**The `@Order`/`@Priority` owner — one comparator.** `injection-mechanics` mints `leaf_core::order::cmp_order` over `OrderKey{value, source}` (with an `OrderSource{Interface,Annotation,Implicit}` axis so an `Ordered`-impl beats a declared `#[order(n)]` at equal value). Every site calls it: autowiring's `PriorityRank` layer, collection/map ordering, `Lookup::ordered_stream`, runtime-lifecycle phases, and — at *correctness grade*, RoleTier-first — the `proxy-substrate` advisor chain. `cross-crate-discovery`'s `SequenceKey(epoch, ContractId)` is the deterministic tie-break *below* it, not a competing comparator. The triplication the register flagged is gone.

**The construction/cycle constraint — single-phase, deferral-only cycles.** Construction is **single-phase**: one total async factory whose typed parameters *are* the injection points; there is no two-phase construct-then-populate, no `&mut self` hook, no early half-built exposure. This is forced by `ownership-model`'s atomic `Provider::provide → Published` seam. The consequence is joint with circular references: because there is never a half-built instance, constructor cycles are **logically fatal** (reported at the validation pass with the full path and the exact "convert injection point X from `T` to `Lookup<T>`" hint), and setter/field cycle tolerance is *not* offered — the only cycle break is an explicit deferral edge (`Lookup<T>`/`LazyRef<T>`) that is removed from the construction graph and resolves the fully-built target post-build via a `Weak` back-ref. leaf has one cycle behavior everywhere, consistent with Spring Boot's own `allow-circular-references=false` default and with AOT.

## Settled cross-ADR seams (resolved, fold into Phase 3 as co-decided notes)

These four boundary points are decided — not open — and Phase 3 inherits them as fixed:

- **Singleton carry-forward across dynamic epochs.** `reopen_epoch()` seeds epoch N+1's `singletons` array from epoch N's already-initialized `OnceCell<ErasedBean>` cells for retained (static-row, stable) `BeanId`s; only newly-appended dynamic `BeanId`s start empty. A static singleton keeps **one** instance across the atomic swap — the `Once` multiplicity contract holds. (Owner: `registry-substrate` storage; `compile-runtime-split`/`cross-crate-discovery` yield the detail.)
- **Typestate vs `reopen_epoch`.** The invariant is "you cannot mutate a *sealed* epoch in place"; mutability exists only in `App<Define|Resolve>`. `reopen_epoch` consumes `App<Running>` and returns a fresh `App<Define>` whose builder is copy-on-write pre-seeded from the prior `FrozenContainer` — it re-enters the mutable state, never edits the running one.
- **Advised bean injected by concrete type.** If a bean is advised, every injection of it MUST go through a declared service-trait view; a concrete-type (`Ref<Concrete>`) injection of an advised bean is an `App<Wired>` startup-validation error, **never** a silent un-advised raw-target handoff. (Owner: `injection-mechanics` Selector rejects it + `proxy-substrate` eligibility gate.)
- **Advised prototype.** Transparent advice applies to shared-shape beans (singleton + context-scoped, `Published::Shared`); an advised prototype gets a per-resolution proxy whose `TargetSource` constructs-and-owns a fresh instance via a `Published::Owned`-aware lane — so the "owned move, no refcount" prototype invariant is never silently violated. (Owner: `proxy-substrate` adds the `Owned` lane to the `TargetSource` contract.)

## What Phase 3 inherits

**Fixed toolkit (build every feature on these; do not reopen).**

- The handle currency: `ErasedBean`/`Ref<T>`/`Published{Shared,Owned}`, scope as `(Multiplicity, StoreSource, TeardownPolicy)`, `Send+Sync+'static` riding the handle in one place.
- The registry: dense `BeanId(u32)`, TypeId-primary / name-overlay keying, stable `ContractId`, slot-indexed `OnceCell` singleton store, macro-emitted `TypeRow` upcasts.
- The origin-agnostic `Provider::provide → BoxFuture<Published>` creation seam and the single `Engine::create` driver under `Engine`/`Context`.
- The four-layer codegen pipeline (thin macros, `linkme` slices, binary-crate force-link + `ExpectedManifest`) and the `App<Define→Resolve→Wired→Running>` phase machine with `seal()` + the aggregated `AssemblyReport`.
- The condition algebra (`CondExpr`/`ConditionId` + the per-kind tier-map, runtime as the mandatory floor), the auto-config ladder + `@Fallback` role, the triple anti-DCE defense, and starters/BOM topology.
- The async spine (`BoxFuture` everywhere, capability-split `ExecutionFacility`, `Cx` per-poll re-install with `Propagation` per key, ledger teardown / no async Drop, reactive-hot/poll-cold).
- The interception substrate (transparent newtype + `Interceptor` chain + `TargetSource`, advisors as flat const descriptors, the frozen `ProxyPlan`), the one `cmp_order`, single-phase construction with deferral-only cycles, and the one `LeafError`/`Diagnostic`/`FailureAnalyzer` model.

**Left to per-feature Phase 3 (the `openSubQuestions`, all measure-before-build).**

- **Performance escapes:** the opt-in singleton-only arena/`BeanRef<T>` fast lane; the monomorphized single-advisor proxy fast-path; skip-`Cx`-install-when-unchanged; the always-on-vs-opt-out `App<Wired>` validation; whether `cargo leaf prepare`'s build-time plan ships in v1.
- **Macro-emission policies:** which declared injectable supertraits get `TypeRow` upcast rows (all vs only-used); exact `ContractId` derivation inputs and collision policy (now a semver/exclusion surface); the `ErasedArgs`/`ErasedRet` pack/unpack ABI; the pointcut-expression mini-language grammar.
- **Scope/lifecycle precision:** nested context-scope (request-in-session) teardown-on-cancellation ordering; lazy/`SmartInitializingSingleton` interaction with the `WiringPlan`; `@DependsOn` exact semantics.
- **Dynamic/WASM lane (optional, v1-droppable):** second-epoch precedence beyond ordering (can a dynamic row shadow a static `@Primary`?), in-flight reconciliation across the swap, WASM proxy trap/teardown semantics behind the erased proxy and `TeardownLedger`, context marshalling of `Inherit` keys into the WIT call.
- **Pluggable-runtime / bounds:** the single-threaded `?Send` carve-out (deferred behind the pluggable-runtime seam, demand-measured); the `@Async` body-rewrite-plus-chain-marker hybrid; the `AsyncUncaughtFailureHandler` granularity.
- **DX/diagnostics:** the narrowing-`Trace` cost policy and crate-provenance richness; the `RenderStyle::StructuredJson` schema as a stable tooling contract; the type-state builder as a hand-facing convenience; stack-starter curation policy and `@Fallback`-vs-`OnMissingBean` interaction; the exact closed `ErrorKind` variant set and stability policy.
