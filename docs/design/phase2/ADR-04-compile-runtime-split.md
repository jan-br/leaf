# ADR 04 — Compile-vs-Runtime Resolution Split (AOT lineage)  `[compile-runtime-split]`

> **Note (2026-06-14):** the optional *dynamic-WASM source / WASM-"classpath"* mentioned below is **DROPPED** — leaf autodiscovery is native link-time registration only. Ignore WASM/dynamic-module references here; they are struck from the active design.


## Decision
ADOPT a FOUR-STAGE "definition-time freeze, runtime AOT-lite assembly" timeline whose single load-bearing invariant is: **all DEFINITION-SHAPE decisions (which beans/auto-configs exist, in what order, gated how) freeze into one immutable `FrozenContainer` snapshot at a single `seal()` boundary, BEFORE any async/IO/construction; all VALUE-SHAPE work (instantiation, .await, scoped instances) happens after, against that frozen snapshot.** The split is therefore NOT "build-time vs runtime" — it is "definition-phase (cold, synchronous, around-main, single-threaded) vs value-phase (async, post-seal)". The AOT-lineage is honored as a RUNTIME-AOT default (assemble-then-freeze around main) with an OPT-IN build-time pre-plan, NOT as a mandatory build-time freeze.

Concretely, this decision OWNS the phase machine + its typestate, the validation-timing ladder, the condition tier-router, the auto-config engine placement, and the dynamic second-epoch seam. It builds directly on registry-substrate's `RegistryBuilder`→`Registry` freeze and codegen-boundary's four layers — it does not re-key, re-store, or re-emit anything those own.

THE PHASE MACHINE (a runtime typestate `App<Phase>`, the lifecycle realization of register §4's "freeze/seal boundary"):

```
App<Define>   ── cold, sync, around main: lift link/codegen const rows (codegen-boundary L4),
              │  run anti-DCE self_check, register user defs + Registrar emissions, seal Environment
              ▼
App<Resolve>  ── cold, sync: run the CONDITION TIER-ROUTER (cfg already pruned at compile; const-fold
              │  already decided at build; evaluate the RUNTIME tier here) + the AUTO-CONFIG deferred
              │  back-off batch (R1 default) — this is the "all user defs registered" moment OnMissingBean needs
              ▼  seal()  ◀── THE SINGLE FREEZE BOUNDARY: RegistryBuilder::freeze() → immutable FrozenContainer
App<Wired>    ── cold, sync: WHOLE-GRAPH WIRING VALIDATION over the frozen snapshot (the dedicated
              │  startup-validation pass §2.9): resolve every mandatory injection point to a winning BeanId,
              │  detect NoUniqueBean / NoSuchBean / scope-mismatch / @DependsOn cycles — AGGREGATED, all-at-once,
              │  before a single constructor runs. Produces a const-shaped WiringPlan (ordered dependency waves).
              ▼  start().await  ◀── the async fence (5f pre-materialization pipeline ends here)
App<Running>  ── async, post-fence: eager-instantiate per the WiringPlan (wave-parallel, OnceCell-per-slot),
                 hot resolution reads the frozen snapshot lock-free. Steady state.
```

TYPESTATE: `App<Define|Resolve>` expose `register`/`with`; `seal(self) -> App<Wired>` consumes mutable access (a post-seal edit is a COMPILE error on the static path — register §4's "type-state consuming mutable access" option, chosen over a runtime flag because it is the richest-diagnostic timing for post-freeze-edit). A `dyn`-erased `SealedView` is offered at crate edges so `App<Phase>` generics do not go viral (the registry-substrate ADR's frozen `Registry` is the value inside).

VALIDATION-TIMING LADDER (resolves register §4's "eager-instantiate fail-fast vs build/startup validation pass"): we ADOPT THE STARTUP-VALIDATION PASS as the primary wiring-error surface, NOT eager-instantiate-fail-fast. At `App<Wired>` the whole resolvable graph is validated against the frozen snapshot — every mandatory dependency resolved to a winner, every collection counted, every scope-mismatch and @DependsOn cycle detected — and ALL failures aggregated into one `AssemblyReport` before any bean is constructed. Eager construction in `App<Running>` then cannot fail for WIRING reasons (only for genuine runtime/IO faults in a constructor body, which are inherently value-phase). This is the §2.9 "dedicated startup validation pass" — richer than per-bean eager fail-fast (it sees the whole graph and reports every problem at once) and earlier than first-`get`.

COMPILE-TIME is used ONLY where the error stays legible (§2.9): the macro hard-errors on a generic bean, a malformed `CondExpr`, an unknown cfg feature, a `Send+Sync`-violating bean (via `#[diagnostic::on_unimplemented]`). Cross-bean wiring is NOT pushed to compile time — a "type T has no provider" expressed as a trait-bound error is the cryptic error §2.9 explicitly declines; the startup pass explains it fully (chain, candidates considered, missing source crate).

CONDITION TIER-ROUTER (the lifecycle owner of codegen-boundary's per-kind tier-map; settles register §4 condition-eval + §8 conditional strategy): three tiers feed ONE timeline. `Cfg`-tier (OnClass/OnCapability) is pruned at COMPILE (the descriptor physically absent; force-link paired). `ConstFold`-tier (OnRustVersion/OnCloudPlatform-by-target/OnThreading-platform) is decided at BUILD by codegen → arrives as `CondExpr::Const(bool)`. `Runtime`-tier (OnBean/OnMissingBean/OnProperty/OnProfile + EVERY dynamic/WASM-contributed definition) is evaluated in `App<Resolve>`, synchronously, against the growing-then-complete definition set. **The Runtime tier is MANDATORY and is the floor** — because maximal-magic dynamic discovery (owner-decision #1) + the dynamic-WASM lane both contribute definitions with no cfg and no Rust type, gateable ONLY at runtime; cfg/const are accelerator fast-paths over an always-present runtime evaluator. Profiles are ALWAYS runtime (coherence constraint). `CondExpr.phase()` is framework-inferred (max over leaves) — an OnBean nested anywhere auto-defers the whole guard to `App<Resolve>`'s register sub-pass, after the parse sub-pass and after all defs are collected.

AUTO-CONFIG ENGINE PLACEMENT (resolves register §4 "auto-config build-frozen vs runtime"; honors owner-decisions #2/#3): the R1 RUNTIME-DEFERRED-BATCH engine runs in `App<Resolve>` as the DEFAULT, because back-off correctness (OnMissingBean must see the FINAL user-bean set) is intrinsically runtime — it is the deferral-for-correctness moment, placed AFTER user-def registration and BEFORE seal(). The R3 BUILD-TIME-FROZEN PLAN (`cargo leaf prepare` emitting a `--check`-able `autoconfig_plan`) is an OPT-IN acceleration for determinism/wasm/native-image: it pre-runs the build-DECIDABLE half (union+dedup+exclusion-typo-guard+const-prune+topo-sort) at build time, leaving the runtime residual (kill-switch + property-exclusions + OnMissingBean back-off + any dynamic-WASM auto-config) in the SAME `App<Resolve>` slot. The frozen/residual seam is made legible by routing both through one engine that records every decision into the condition report. The definition-order-vs-creation-order boundary (doc 1234) is STRUCTURALLY enforced: the ordered auto-config batch is an `App<Resolve>`/`App<Wired>` artifact discarded before `App<Running>`'s instantiation scheduler exists — the WiringPlan's waves are dependency-driven, never inheriting batch order.

DYNAMIC SECOND-EPOCH SEAM (resolves register §4's WASM/dynamic timing): `App<Running>` exposes `reopen_epoch()` → a fresh `App<Define>'` whose `RegistryBuilder` is pre-seeded from the prior `FrozenContainer` (registry-substrate's `reopen_epoch`), the directory-scanned WASM host-proxy registers identical Descriptor+Provider rows (origin=DynamicWasm, diagnostic-only), then a controlled re-`seal()`+re-validate produces a NEW immutable `FrozenContainer` epoch. We adopt FRESH-SNAPSHOT-PER-EPOCH (not in-place slot append): in-flight resolutions hold an `Arc<FrozenContainer>` of their epoch and complete coherently; new resolutions read the latest epoch via a single `arc_swap`-style atomic pointer (lock-free, no global mutex, safe-publication-for-free). Static rows always precede dynamic rows in BeanId order; dynamic rows never shadow a static winner unless explicitly @Primary. This keeps mid-steady-state mutation UNREPRESENTABLE (you cannot edit a frozen epoch — only publish a new one), exactly as the immutable-snapshot model demands.

## Rationale
- DX#1 + §2.9 richest-diagnostic-timing decide the core split. A SINGLE seal() boundary + a whole-graph STARTUP-VALIDATION pass (not per-bean eager fail-fast, not a forced compile-time wiring check) is the timing with the MOST complete context: it sees the entire frozen graph at once, so it reports every NoSuchBean/NoUniqueBean/scope-mismatch/cycle together with the full assembly chain and the missing-source crate, before any constructor runs. §2.9 explicitly says we do NOT force a check to compile-time if that only yields a cryptic error a deferred pass could explain fully — a 'no provider for T' trait-bound error is exactly that cryptic case, so cross-bean wiring is validated at startup, while genuinely-legible errors (generic bean, malformed CondExpr, Send+Sync violation) stay at compile time.
- The freeze-early immutable-snapshot model is FORCED by the upstream ADRs and by §2.5/§2.4, not re-litigated here. registry-substrate already froze into a dense BeanId Registry whose coherence is built once; this ADR's job is the TIMELINE around it. Putting the freeze at a single seal() between a cold synchronous definition phase and the async value phase (a) gives safe-publication-for-free across all later tasks (immutable data, §2.5 no-global-lock), (b) makes the OnceCell-per-slot lock-free ready read the hot path (§2.4 reactive), and (c) means NOTHING async/IO happens before the graph shape is fixed — the cleanest execution-context story (§2.3).
- Runtime-AOT (assemble-then-freeze around main) as DEFAULT, build-time-plan as OPT-IN, is the only choice that honors maximal-magic discovery (owner #1) AND the AOT lineage AND origin-agnosticism simultaneously. A mandatory BUILD-TIME freeze (pure Architecture C) cannot see what the binary's build doesn't already know (the cross-crate single-crate wall) and structurally cannot host OnMissingBean back-off (needs the final runtime user-set) or a dynamic-WASM source (no build presence). So a runtime assembly pass must exist regardless; making it the default and offering cargo leaf prepare as a determinism/wasm/native-image accelerator captures the AOT benefit (inspectable --check-able plan, fast startup) exactly where linkme/runtime-assembly are weak, without taxing the dominant server target.
- The Runtime condition tier being MANDATORY (not optional) is forced by owner-decision #1 + the dynamic-WASM lane. A maximal-magic-discovered or WASM-contributed definition has no cfg and no Rust type — it can ONLY be gated at runtime. So the runtime evaluator is the floor and cfg/const-fold are accelerators over it, which is also exactly the cross-crate doc's rule ('link-time answers what could contribute; runtime conditions answer what does wire'). Profiles-always-runtime is the stated coherence constraint, preserved.
- Auto-config back-off correctness (owner #2) pins the engine to App<Resolve> at runtime BY NECESSITY: OnMissingBean must observe the final user-bean set, which only exists after user-def registration and only at runtime (a transitive crate's defs are a runtime fact). This is Spring's DeferralForCorrectness intent realized as a lifecycle phase, not a performance trick. The R3 build-plan opt-in is purely an accelerator that pre-decides the build-DECIDABLE half, keeping the runtime residual in the same slot — so there is ONE engine and ONE accountable report, never a split-brain.
- The dynamic second-epoch fresh-snapshot-per-epoch design keeps the no-global-lock + immutable-coherence invariants intact while enabling the WASM 'real dynamic classpath' (the only distinct value the WASM exploration retained). An atomic Arc pointer swap of whole FrozenContainer epochs means a re-freeze is safe-publication-for-free, in-flight work is coherent against its own epoch, and mid-steady-state mutation is literally unrepresentable (you can't mutate a frozen epoch). This is the controlled re-freeze the registry-substrate ADR's reopen_epoch seam anticipated, given precise lifecycle reconciliation semantics.
- Thin macros (§2.10) and unification (§2.11) are preserved by OWNING ONLY the timeline. This ADR adds no new emitted const shapes (it consumes codegen-boundary's Descriptor/CondExpr/slices), no new key/store (it consumes registry-substrate's Registry/BeanId), and no new diagnostics primitive (it routes into the condition report + a unified AssemblyReport). The phase machine is normal testable leaf-boot logic; the macro never sees a phase. The single seal() + single startup-validation pass is the maximal-unification answer — one freeze, one validation moment, one report — rather than per-feature freeze/validate logic scattered across configuration-classes, import-composition, auto-config, and the registry.

## Options considered & rejected
- **Mandatory BUILD-TIME freeze: a full AOT plan (refreshForAotProcessing lineage) computed by cargo leaf prepare / build.rs is THE container; runtime just executes a frozen RunPlan/WiringPlan with no runtime assembly pass** — rejected: Lowest startup cost and earliest diagnostics, but it CANNOT be the default without breaking three fixed constraints. (1) The cross-crate single-crate wall: a build.rs sees only its own crate, so the binary's build cannot enumerate foreign #[component]/#[auto_configuration] rows without an unenforced per-crate manifest convention — a forgetting contributor is silently absent, trading the link-DCE hazard for a missing-manifest hazard. (2) Owner-decision #2 back-off: OnMissingBean must see the FINAL runtime user-bean set; even a build-frozen plan keeps a mandatory runtime residual, so 'build-time IS the container' is false. (3) Origin-agnosticism + the dynamic-WASM lane: a WASM-contributed definition has no build presence and must be assembled/gated at runtime. A runtime assembly pass therefore MUST exist; making build-time the default merely adds a build-frozen/runtime-residual seam ('why didn't my plan change when I set the exclude property?') as a mandatory confusion surface. Kept as the OPT-IN cargo leaf prepare accelerator for determinism/wasm/native-image, where its portability is the real win.
- **Mutable-concurrent-map registry with NO freeze: definitions stay editable through steady state (Spring's literal DefaultListableBeanFactory + markBeanAsCreated runtime flag)** — rejected: Maximally dynamic-edge-faithful but loses on every leaf invariant the immutable model wins. It needs a concurrency-map crate + transactional multi-structure coherence (the name/type/alias/store desync bug class the registry-substrate ADR dissolves by freezing once), reintroduces a global-ish lock surface (§2.5), forfeits safe-publication-for-free across tasks, and makes the post-edit boundary a runtime flag (markBeanAsCreated) rather than the strictly-stronger typestate-consuming-seal compile error (§2.9). The immutable-snapshot + reopen_epoch model already provides the ONLY dynamism leaf actually needs (the WASM dynamic classpath) via controlled re-freeze, without owning mid-life desync. Upstream registry-substrate already rejected this; honoring it.
- **Eager-instantiate fail-fast as the primary wiring-error surface (Spring's preInstantiateSingletons: wiring errors surface AS each bean is built)** — rejected: Spring-faithful but pessimal for §2.9 richest-context. Building beans one-by-one surfaces the FIRST wiring error and aborts, hiding the other N problems and entangling a pure wiring fault (NoUniqueBean) with construction/IO faults in the same async pass — so the diagnostic can't cleanly say 'this is a graph problem, fixable before you run anything'. A dedicated whole-graph startup-validation pass over the frozen snapshot (chosen) validates EVERY mandatory injection point at once, aggregates all failures, and runs BEFORE any constructor — strictly earlier and strictly richer. Eager construction in App<Running> is retained but can then only fail for genuine value-phase reasons.
- **Push cross-bean wiring validation to COMPILE TIME (type-state / trait-bound proofs that every injection point has a unique provider)** — rejected: Tempting for 'compile-time wiring errors' but violates §2.9's explicit clause: we do not force a check to compile-time if it only yields a cryptic error a deferred check could explain fully. A missing/ambiguous provider expressed as a trait-bound failure is exactly cryptic — no assembly chain, no candidates-considered, no missing-source-crate naming, and it is impossible cross-crate anyway (the binary's macro can't see sibling providers, and maximal-magic discovery has no declared scan-scope to prove against). It also cannot express runtime/dynamic/WASM-contributed beans at all. Compile-time is reserved for genuinely-legible local errors (generic bean, malformed CondExpr, Send+Sync bound); cross-bean wiring is the startup pass's job.
- **No single freeze boundary — let each subsystem (config-classes, import-composition, auto-config, registry) freeze/validate on its own schedule** — rejected: Violates unification (§2.11) and creates the exact register coherence-note failure: four features each invent their own expected-vs-found self-check and their own freeze moment, disagreeing on what 'silently dropped' vs 'legitimately backed off' means and on per-element tier. One seal() + one startup-validation pass + one anti-DCE self-check + one condition report is the unified spine; subsystems contribute rows/conditions/auto-configs INTO the single timeline rather than owning private freeze logic.
- **In-place BeanId slot append for the dynamic WASM second epoch (mutate the existing dense slot space, keep one snapshot)** — rejected: Cheapest memory-wise but reintroduces mutation of a 'frozen' structure mid-steady-state — the precise thing the immutable model exists to forbid. In-flight resolutions against the old slot layout could observe a half-extended index (the coherence bug class again), and it needs a lock to serialize the append against concurrent hot reads (§2.5 violation). Fresh-snapshot-per-epoch + atomic Arc swap keeps each epoch immutable, makes re-freeze safe-publication-for-free, and lets in-flight work finish coherently against its own epoch — the dynamism without the desync.

## Consequences
- FORCES leaf-boot to host the App<Phase> typestate machine (Define→Resolve→Wired→Running) and the seal() consuming transition, plus a dyn-erased SealedView for crate edges so the phase generic does not leak into integration-crate signatures. Post-seal definition edits are a COMPILE error on the static path (typestate) and a diagnostic error on the dyn-erased path — the richest post-freeze-edit timing (§2.9). refresh-lifecycle, container-core, and the run-pipeline must expose exactly this progression and place their steps in the correct phase.
- FORCES a whole-graph STARTUP-VALIDATION pass over the frozen snapshot as a first-class subsystem producing an aggregated AssemblyReport: every mandatory injection point resolved to a winning BeanId, every collection counted, every scope-mismatch and @DependsOn cycle detected, ALL failures collected before any constructor runs. Autowiring-resolution/candidate-resolver must expose a no-instantiation, definition-level resolve mode (also what OnSingleCandidate and the area-coherence note require). Eager construction in App<Running> may then fail ONLY for genuine value-phase (constructor/IO) faults, not wiring.
- FORCES the condition tier-router into App<Resolve> as the runtime floor: cfg pruned at compile (force-link paired), const-fold decided at build, runtime tier (OnBean/OnMissingBean/OnProperty/OnProfile + ALL dynamic/WASM defs) evaluated here against the parse-then-register two sub-passes. CondExpr.phase() is framework-inferred (max over leaves). This binds the lifecycle tightly to codegen-boundary's per-kind tier-map (the load-bearing kernel ABI) and to the dynamic-source lane (why the runtime tier can never be dropped).
- FORCES the auto-config R1 deferred back-off engine into the App<Resolve> slot AFTER user-def registration / BEFORE seal() as the default, with the R3 cargo leaf prepare plan as an opt-in accelerator whose residual lands in the SAME slot. The definition-order-vs-creation-order boundary is structurally enforced: the ordered auto-config batch and the WiringPlan are App<Resolve>/App<Wired> artifacts the App<Running> instantiation scheduler cannot read; instantiation waves are dependency-driven only. import-composition's deferred-group phase, auto-config-ordering's sort, and condition-family's gate all run inside this one slot.
- FORCES the anti-DCE expected-vs-found self_check to run ONCE at the App<Define>→App<Resolve> edge (after lifting all link/codegen rows, before conditions), cross-referencing codegen-boundary's build-emitted ExpectedManifest so a DCE-dropped crate surfaces as a loud, named diagnostic rather than a silently-smaller graph. This is the SINGLE substrate-level self-check the area-coherence note demands; the condition report CONSUMES its result (manifest-present-but-no-outcome-and-no-cfg-label = dropped registration), never reinvents it.
- FORCES the async fence (5f pre-materialization) to sit at start().await between App<Wired> and App<Running>: ALL async config-IO (remote property sources, customizers) completes and snapshots into the sealed Environment BEFORE seal(), so the entire definition/resolve/validate phase is synchronous and side-effect-free. The first .await in the whole timeline is eager-instantiate. This pins the environment/binder sealing order and the boxed-future Provider seam (construction is the only async work, post-fence).
- FORCES the dynamic-WASM layer to use reopen_epoch()→re-seal()→atomic-Arc-swap of whole FrozenContainer epochs (fresh snapshot per epoch, not in-place append). Static rows precede dynamic rows in BeanId order; dynamic rows never shadow a static winner unless explicitly @Primary; in-flight resolutions complete against their own epoch Arc. WASM provider trap/teardown semantics live entirely in the Provider impl + TeardownLedger (Phase-3, WASM-local). This is the only sanctioned post-seal mutation; everything else is a frozen-epoch read.
- ACCEPTS that the default path pays a one-shot COLD assembly+resolve+validate cost around main (lift rows, run anti-DCE check, evaluate runtime conditions, run auto-config back-off, freeze, whole-graph wiring validation) — bounded, synchronous, single-threaded, before the async fence, never on a hot path. Determinism/wasm/native-image deployments opt into cargo leaf prepare to move the build-decidable half (union/dedup/sort/const-prune/typo-guard) to build time, leaving only the runtime residual. We do NOT make the build plan the default (it cannot be, per back-off + cross-crate wall + dynamic sources).
- ACCEPTS that 'compile-time wiring errors' are deliberately SCOPED: only locally-legible errors (generic bean, malformed CondExpr, unknown feature, Send+Sync bound via #[diagnostic::on_unimplemented]) are compile-time; all cross-bean wiring (NoSuchBean/NoUniqueBean/scope-mismatch/cycle) is the richest-context startup-validation pass. leaf must document this honestly: the 'AOT lineage' delivers FAST + EARLY + RICH startup validation and an optional inspectable build plan, NOT a fully compile-checked bean graph (which is impossible under maximal-magic discovery + dynamic sources).

## API / type sketch

// ═══ leaf-boot: the phase machine — the lifecycle realization of registry-substrate's freeze ═══
//     (consumes registry-substrate's RegistryBuilder/Registry + codegen-boundary's slices/CondExpr;
//      owns ONLY the timeline, the validation-timing ladder, the tier-router, and the epoch seam)

// ── PHASE TYPESTATE ── seal() consumes mutable access → post-seal edit is a COMPILE error
pub struct App<P: Phase> { reg: RegistryBuilder, env: EnvBuilder, _p: PhantomData<P> }
pub enum Define {}  pub enum Resolve {} pub enum Wired {} pub enum Running {} // P markers
pub trait Phase {}  // sealed

impl App<Define> {
    pub fn new() -> Self;                                         // lifts COMPONENTS/PROVIDERS/AUTO_CONFIGS/CONDITIONS slices (codegen-boundary L4)
    pub fn with(self, plugin: impl Plugin) -> Self;              // explicit register escape hatch (Architecture D)
    pub fn register(&mut self, d: Descriptor, p: Arc<dyn Provider>) -> Result<BeanId, RegisterError>;
    pub fn self_check(&self, expected: &ExpectedManifest) -> Result<(), AntiDceError>; // SINGLE anti-DCE pass
    pub async fn seal_environment(self) -> Result<App<Resolve>, EnvError>; // 5f async-IO fence ENDS here; env snapshotted
}

impl App<Resolve> {                                              // COLD, SYNC, single-threaded
    fn route_conditions(&mut self) -> Result<(), AssemblyError>; // tier-router: cfg pruned@compile, const@build, RUNTIME tier here
    fn run_autoconfig(&mut self, env: &Env) -> Result<(), AutoConfigError>; // R1 deferred back-off batch (default);
                                                                 //   if a build plan exists, replay it + residual (R3 opt-in)
    pub fn seal(self) -> Result<App<Wired>, AssemblyError>;       // ◀ THE FREEZE: RegistryBuilder::freeze() → immutable Registry
}

impl App<Wired> {                                                // COLD, SYNC: whole-graph validation over the FROZEN snapshot
    pub fn validate(&self) -> Result<WiringPlan, AssemblyReport>;// AGGREGATED §2.9 pass: every mandatory dep → winner,
                                                                 //   NoSuchBean/NoUniqueBean/scope-mismatch/@DependsOn-cycle, ALL at once
    pub async fn start(self) -> Result<App<Running>, AssemblyReport>; // validate()+eager-instantiate per WiringPlan waves (async fence)
}

impl App<Running> {
    pub fn container(&self) -> &Arc<FrozenContainer>;            // hot reads: lock-free, immutable snapshot
    pub fn reopen_epoch(&self) -> App<Define>;                   // DYNAMIC second epoch (WASM): pre-seeded from current FrozenContainer
    pub async fn shutdown(self);                                 // reverse-dependency teardown (no async Drop)
}

// ── THE FROZEN CONTAINER ── an immutable epoch; dynamic re-freeze publishes a NEW one via atomic swap
pub struct FrozenContainer {
    registry: Registry,                                          // registry-substrate ADR's frozen dense-BeanId snapshot (unchanged)
    plan:     WiringPlan,                                        // const-shaped ordered dependency waves (definition-phase artifact)
    report:   Arc<ConditionReport>,                              // the accountable account (condition-report ADR)
    epoch:    u32,                                               // monotonically increasing; static rows always epoch 0
}
pub struct EpochCell { current: arc_swap::ArcSwap<FrozenContainer> } // lock-free epoch publication; no global mutex (§2.5)

// ── WIRING PLAN ── output of App<Wired>::validate; NEVER readable by App<Running>'s scheduler (doc-1234 enforcement)
pub struct WiringPlan {
    waves: Box<[Box<[BeanId]>]>,                                 // dependency-ordered eager-init waves (wave-parallel)
    eager: bitset::BitSet,                                       // non-lazy singletons; lazy beans excluded (registers but not built)
}

// ── CONDITION TIER-ROUTER ── the lifecycle owner of codegen-boundary's per-kind tier-map
pub enum EarliestTier { Cfg, ConstFold, Runtime }               // Runtime is the MANDATORY floor (dynamic/WASM defs)
fn evaluate(expr: &CondExpr, ctx: &ResolveCtx) -> ConditionOutcome; // recurses; Const leaves already decided; Profiles always runtime
//   CondExpr::tier() == Runtime  ⇒ whole guard deferred to App<Resolve> register sub-pass (after all defs collected)

// ── ASSEMBLY REPORT ── one aggregated, all-failures-at-once diagnostic (§2.9), distinct from condition-not-met (silent)
pub struct AssemblyReport { failures: Box<[AssemblyError]> }    // NoSuchBean{ty, missing_source}, NoUniqueBean{candidates}, ScopeMismatch, DependsOnCycle

// ── OPT-IN BUILD PLAN ── cargo leaf prepare emits this; App<Resolve> replays it + runtime residual (R3 accelerator)
pub struct AutoConfigPlan { ordered: &'static [BeanId], residual_conditions: &'static [CondExpr] } // include!'d, --check-able

## Forces on other concepts
- **Type-Erasure & Registry Substrate (upstream registry-substrate ADR)**: Honored unchanged and built upon: seal() IS RegistryBuilder::freeze() producing that ADR's immutable dense-BeanId Registry; this ADR wraps it in FrozenContainer with the WiringPlan + report and an EpochCell for dynamic re-freeze. reopen_epoch() is that ADR's reopen_epoch seam given lifecycle semantics (fresh-snapshot-per-epoch + atomic swap). The startup-validation pass consumes candidates(ty)→&[BeanId]; the substrate hands candidates, this ADR's pass picks/validates winners across the WHOLE graph at once.
- **Metadata / Codegen Boundary (upstream codegen-boundary ADR)**: Honored unchanged: App<Define>::new lifts that ADR's COMPONENTS/PROVIDERS/AUTO_CONFIGS/CONDITIONS slices (Layer 4); the anti-DCE self_check consumes its build-emitted ExpectedManifest; the condition tier-router is the runtime EXECUTION of its per-kind tier-map; the optional AutoConfigPlan is its cargo leaf prepare artifact. This ADR adds NO new emitted shape — it only sequences when the lifted rows are condition-routed, auto-config-assembled, frozen, and validated.
- **Autowiring / Candidate Resolution & Collection Injection**: MUST expose a no-instantiation, definition-level resolve mode (resolve a winning BeanId over the frozen snapshot WITHOUT constructing) — used by the App<Wired> whole-graph validation pass, by OnSingleCandidate, and by collection-count checks. determine_winner runs at validate-time over the frozen candidate sets; collection injection counts candidates at validate-time (no primary/fallback collapse). The WiringPlan's waves are built from the resolved dependency edges.
- **Container Refresh Lifecycle & Run Pipeline**: MUST realize exactly the Define→Resolve→Wired→Running progression with seal() as the freeze boundary and start().await as the async fence. The refresh template places: env-IO+seal_environment (async, ends the 5f fence) → anti-DCE check → condition routing + auto-config batch → seal() → whole-graph validate → eager-instantiate waves. RunPlan/RunnerPlan AOT variants (register §3 candidates) ride this same timeline; a build-frozen RunPlan is the opt-in accelerator, never mandatory.
- **@Conditional Condition SPI / Conditional-On-X Family / Condition Report**: The runtime condition tier evaluates in App<Resolve> (parse sub-pass, then register sub-pass after all defs collected); the runtime tier is the MANDATORY floor (dynamic/WASM defs have no cfg). CondExpr.phase() framework-inferred. The condition report is frozen INTO FrozenContainer at seal(); a later NoSuchBean in the startup-validation pass enriches from it ('backed off because…' / 'feature X not enabled'). condition-not-met (silent, recorded) stays distinct from the AssemblyReport's wiring-broken (loud, aggregated).
