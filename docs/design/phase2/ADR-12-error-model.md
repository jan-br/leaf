# ADR 12 — Unified Error / Diagnostics Model  `[error-model]`

> **Note (2026-06-14):** the optional *dynamic-WASM source / WASM-"classpath"* mentioned below is **DROPPED** — leaf autodiscovery is native link-time registration only. Ignore WASM/dynamic-module references here; they are struck from the active design.


## Decision
ADOPT a SINGLE diagnostic spine — one `LeafError` causal-chain type, one `Diagnostic` rendering trait, one `FailureAnalyzer` SPI, and one `tracing`-bridged-but-not-tracing-dependent recorder — layered across FOUR fixed timing-tiers that map 1:1 onto the two upstream ADRs' phase machine. The load-bearing invariant: EVERY framework error, regardless of where it is born, is one node in a uniform `Cause` chain that carries (what was being assembled / what it needed / what was expected / what was actually found incl. candidates-considered / where), and EVERY user-facing surfacing of that chain goes through ONE renderer that produces a Spring-FailureAnalysis-grade `description + action + cause` block. The vocabulary is shared; the *timing* differs by tier, never the shape.

THE FOUR TIERS (each owned by an upstream ADR's phase; this ADR owns the vocabulary that crosses them):

TIER 0 — COMPILE-TIME (macro, leaf-macros). Only locally-legible errors: generic `#[component]` (hard `compile_error!` via `syn::Error` + `register_component!(Concrete)` hint), malformed `CondExpr`/`#[conditional(..)]`, malformed `${...}`/`#{...}` grammar in `#[value(..)]`, `@AliasFor`/stereotype misconfig, unknown cfg feature, and the `Send+Sync`/`'static` bound on a managed bean surfaced via `#[diagnostic::on_unimplemented]` doctrine messages (1.78, stable). Macros emit `proc_macro::Span` source locations (file/line/col via 1.88 APIs) into the const `Origin` so a later-tier error can point back at the exact attribute. Multi-error accumulation per compilation via `syn::Error::combine` → one `to_compile_error()`. NOTHING cross-bean is forced here (§2.9: a "no provider for T" trait-bound error is the cryptic case we decline).

TIER 1 — BUILD / FREEZE-TIME (leaf-codegen build.rs / `cargo leaf prepare`, and `seal()`). Build-decidable errors with a named source but no span: ContractId collision (the codegen-boundary `const{assert!}` guard, lifted to a rendered `Diagnostic`), anti-DCE expected-vs-found mismatch (a DCE-dropped crate → loud, names the missing source crate + "did you add AND force-link leaf-X?"), auto-config ordering cycle / typo'd `exclude=`/`before`/`after`, and `cargo leaf prepare --check` plan staleness. These are `AssemblyError` variants produced at the `App<Define>→App<Resolve>` self-check edge and at `seal()`.

TIER 2 — STARTUP-WIRING (the `App<Wired>` whole-graph validation pass, compile-runtime-split ADR). The richest cross-bean tier: `NoSuchBean`, `NoUniqueBean` (with the full candidate set + per-candidate source-crate provenance + the determine_winner narrowing trace), `ScopeMismatch`, `DependsOnCycle`, strict-mode `UnresolvedValue`, `ConvertError`, `BindError`, constraint-`ValidationError`, `ProfileError` — ALL aggregated into ONE `AssemblyReport { failures: Box<[LeafError]> }` BEFORE any constructor runs. This is the dedicated startup-validation pass §2.9 mandates.

TIER 3 — RUNTIME / VALUE-PHASE (`App<Running>`). Genuine value-phase faults only: a constructor body / config-IO / async task failure, surfaced as a `LeafError` whose `Cause` chain still threads through the bean's assembly context (which bean, which injection edge). Also the dynamic-WASM second-epoch re-`seal()` re-runs Tier 1+2 producing a fresh `AssemblyReport` for that epoch.

THE CAUSAL CHAIN (the one shared type, std-native, no JVM stack-trace mechanism). `LeafError` is a struct, not a flat enum: a typed `kind: ErrorKind` (the closed taxonomy below) + a `chain: Box<[Cause]>` (the explicit causal narrative, Spring's nested-exception intent realized as data not unwinding) + an `origin: Origin` (compile span / config file:line / source crate / dynamic-WASM tag, the same `Origin` the config + registry ADRs already carry) + `mode: Severity`. It implements `std::error::Error` with a real `source()` walk over the chain (intent parity with Spring's `getCause()`), so it composes with `?`, `anyhow`, and `tracing`'s error field — but its OWN richness lives in `chain`/`kind`, never in a backtrace. `ErrorKind` is a CLOSED core enum (NoSuchBean, NoUniqueBean, ScopeMismatch, DependsOnCycle, AntiDce, ContractCollision, AutoConfigOrdering, UnresolvedValue, Convert, Bind, Validation, Profile, Condition-veto-as-info, ConstructionFailed, ConfigIo, Cancelled, …) with ONE open `Integration { kind_id: ContractId, … }` arm so integration crates (incl. DataAccessError translation) extend the taxonomy by-data without widening the core ABI — the by-data-extensible escape the register asks for, NOT a `dyn DataAccessError` trait family.

THE TWO-VALUED DISTINCTION (the register's central tension). "condition-not-met (silent no-bean now)" is NEVER a `LeafError` — it is a `ConditionOutcome` record in the `ConditionReport` (owned by compile-runtime-split's `FrozenContainer`, frozen at `seal()`). "wiring-broken (loud failure later)" IS a `LeafError` in the `AssemblyReport`. They are kept time-separated AND type-separated, BUT JOINED at render: when Tier 2 produces a `NoSuchBean`, the renderer CONSULTS the frozen `ConditionReport` and the anti-DCE manifest to ENRICH the message — "no provider for `T`: bean `RedisClient` backed off (OnMissingBean matched your `myRedis`)" / "feature `redis` not enabled" / "crate `leaf-redis` expected but produced zero registrations — add it AND force-link it". This is the "silent-now informs loud-later" unification: one report explains both why something is absent and why a wiring need went unmet.

THE FailureAnalyzer SPI (the marquee §1.7/§2.9 boot surface). `trait FailureAnalyzer { fn analyze(&self, err: &LeafError, ctx: &AnalysisCtx) -> Option<FailureAnalysis>; }` where `FailureAnalysis { description: String, action: String, cause: Option<Cause> }` — exactly Spring's shape. Analyzers are thin `const` descriptors collected through the SAME codegen-boundary substrate (a `FAILURE_ANALYZERS` linkme slice), `@Order`-sorted, first-non-None wins, with `#[leaf::failure_analyzer]` emitting one descriptor + an `AbstractFailureAnalyzer`-analogue cause-walk shim (matches on `ErrorKind`/downcast over the typed chain — NOT Spring's reflective `AbstractFailureAnalyzer<T>` generic-type matching, which Rust cannot do). Built-in analyzers ship for the predictable boot failures (port-in-use, NoSuchBean, NoUniqueBean, BindError, anti-DCE). Crucially the FailureAnalyzer is NOT a parallel error system: it is the *rendering policy* layer over `LeafError` — it turns the always-present causal chain into teachable prose. This DISSOLVES the register's noted duplication ("FailureAnalysis shares anti-DCE/report-rendering machinery with condition-report"): there is ONE renderer (`Diagnostic`), the condition report and the failure analysis are two *inputs* to it, and exit-code mapping rides the same `ErrorKind` (no separate ExitCodeExceptionMapper SPI — an `exit_code()` fn over `ErrorKind` + an override hook).

THE OBSERVABILITY SPINE. `tracing` is a PLUGGABLE BRIDGE, not the opinionated core. The core ships a minimal, dependency-free `StartupRecorder` (a pull/handle recorder, NOT a push event-stream — chosen because startup steps fire before the bus/beans exist and a handle recorder needs no live subscriber) that records the assembly timeline, condition outcomes, and the anti-DCE self-check result into the `FrozenContainer`. A feature-gated `leaf-tracing` integration bridges the recorder + `LeafError`'s `std::error::Error` into `tracing` spans/events. Rationale: charter §6 says the core never depends on an integration, and §2.6 keeps tokio/observability pluggable; making `tracing` core would force it on every user and on wasm/embedded. The recorder is the single spine the register asks for; `tracing` is its dominant-target bridge.

RENDERING. `trait Diagnostic { fn render(&self, w: &mut dyn fmt::Write, style: RenderStyle) -> fmt::Result; }` implemented once for `LeafError`, `AssemblyReport`, and `ConditionReport`, with `RenderStyle { Human, StructuredJson(serde), TracingFields }` and a no-op default delivery (the register's "no-op default"). The renderer walks the `Cause` chain, prints the candidates-considered narrative, and overlays the FailureAnalysis description+action when an analyzer matched.

## Rationale
- §2.9 is the literal mandate and it dictates the shape: 'errors carry the full causal chain — what was being assembled, what it needed, what was expected, what was actually found (incl. candidates considered), and where.' A struct LeafError { kind, chain: Box<[Cause]>, origin } realizes that field-for-field; the candidates-considered narrative is the determine_winner trace the autowiring design already produces; 'where' is the unified Origin the config/registry ADRs already carry. Making it ONE shared type (not per-feature error enums) is the §2.11 unification mandate — the binder, conversion, validation, autowiring, and boot all fold into one Cause chain and one AssemblyReport accumulator, exactly the register's 'collect-all-the-things-that-went-wrong-across-this-typed-boundary' ask.
- The four-tier timing ladder is NOT re-invented here — it is INHERITED verbatim from the two upstream ADRs and this ADR only supplies the vocabulary that crosses them. compile-runtime-split already fixed: compile (legible-only) → build/seal (named-source) → App<Wired> aggregated startup pass → App<Running> value-phase; codegen-boundary already fixed the #[diagnostic::on_unimplemented] doctrine, the ContractId collision guard, and the anti-DCE expected-vs-found check. Honoring §2.9's 'compile-time where legible, else a dedicated startup validation pass, else runtime; do not force a check to compile-time if it only yields a cryptic error', cross-bean wiring lives at Tier 2 (rich) and only locally-legible errors live at Tier 0 — precisely the upstream decision, given a shared error type.
- The silent-vs-loud / type-and-time separation is the register's stated core tension, and keeping ConditionOutcome OUT of LeafError (it lives in the ConditionReport) while JOINING them at render is the only design that satisfies both halves: a backed-off bean is not an error (it is opinionated-defaults working as designed, owner-decision #2's back-off), yet when a later NoSuchBean occurs the user MUST be told 'it backed off because…'. One renderer consuming both the AssemblyReport (loud) and the frozen ConditionReport + anti-DCE manifest (silent context) yields the single rich-context account §2.9 demands without conflating the two.
- Realizing FailureAnalysis as a RENDERING/POLICY layer over the always-present LeafError — rather than a parallel SPI that re-derives causes — dissolves the duplication BOTH the bootstrap design and the condition-report design independently flagged ('FailureAnalysis shares anti-DCE/report-rendering machinery with condition-report'). There is one causal chain (data), one renderer (Diagnostic), and FailureAnalyzers are just pluggable prose+action providers keyed on ErrorKind. Spring's AbstractFailureAnalyzer<T> reflective-type-match is intent-realized as a downcast/match over the typed ErrorKind chain (intent-not-mechanism parity, §2.2) because Rust has no reflective cause-type matching. Folding exit-code mapping into exit_code(ErrorKind) makes 'port-in-use → analysis + action + conventional exit code' ONE story, exactly as the exit-code-shutdown design asked.
- tracing as a pluggable bridge (not core) is forced by §2.6 (the core never depends on an integration; tokio/observability attach on top) and §7 (std assumed, wasm/embedded targets must build). A dependency-free pull/handle StartupRecorder is the right spine because §1.7's diagnostics fire BEFORE the event bus and beans exist (a push event-stream would have no subscriber yet), and a handle recorder composes with the FrozenContainer's immutable snapshot for free (no live-subscriber, no lock, §2.5). leaf-tracing then bridges to the dominant-target observability stack without taxing every build — the register's 'tracing as core vs bespoke recorder with optional bridge' resolved toward the bridge.
- A CLOSED core ErrorKind enum + ONE open Integration{kind_id: ContractId} arm gives the cross-crate extensibility the Persistence-Exception-Translation design needs (a DataAccessError taxonomy contributed by an integration crate) WITHOUT a dyn DataAccessError trait family that would widen the stable core ABI and fight the registry's by-data, origin-agnostic substrate. It is the same 'extend by data, key by ContractId' discipline codegen-boundary and registry-substrate already chose for descriptors — so a WASM-module or integration-crate error participates in the chain identically (origin-agnosticism), and the silent-untranslated-error self-check the translation design wants becomes just another anti-DCE expected-vs-found check on the CONVERTERS/translators slice.
- Thin macros (§2.10) are preserved: #[leaf::failure_analyzer] and #[value(..)] emit one const descriptor + a hand-writable shim each (degrading to add_failure_analyzer(MyAnalyzer) / a hand-built ValueSpec), the macro carries NO error logic beyond Tier-0 span emission and the locally-legible compile_error!s. All chain-building, aggregation, condition-enrichment, and rendering live in normal testable leaf-core/leaf-boot crates — cargo expand shows one legible const, and the entire diagnostic engine is unit-testable without the container.

## Options considered & rejected
- **Per-feature error enums (BindError, ResolveError, ConditionError, RunFailure, …) each rendered by its own feature, with a top-level enum that just wraps them** — rejected: This is the Phase-1 status quo and it directly violates §2.11 unification and §2.9's 'one rich-context account'. Each feature would invent its own causal-chain representation, its own aggregation policy (fail-on-first vs collect-all), and its own renderer — the register explicitly names this as the tension to resolve ('one accumulator shared by binder, conversion, validation'; 'FailureAnalysis duplicates condition-report machinery'). A wrapper enum does not unify the SHAPE of the chain or the candidates-considered narrative, so a cross-boundary failure (a bind error feeding a wiring error feeding a boot failure) cannot be rendered as one coherent chain. Rejected for a single LeafError whose kind is the taxonomy and whose chain is uniform.
- **A trait-object error spine: Box<dyn LeafErrorTrait> / dyn DataAccessError families, JVM-exception-hierarchy style** — rejected: Mirrors Spring's exception class hierarchy but is the wrong rust-native mechanism (§2.2 intent-not-mechanism). A dyn trait family makes downcasting/matching in FailureAnalyzers and rollback-rules awkward (must downcast to concrete types cross-crate), widens the stable ABI (every method on the trait is frozen), and fights the registry substrate's by-data, ContractId-keyed, origin-agnostic discipline (a WASM error has no Rust type to be a trait object of). A closed ErrorKind enum + one open data arm gives the same extensibility by data, matches Spring's intent (a typed taxonomy you can match on), and keeps a thin ABI. Retained only as the std::error::Error source() facade for interop, never as the internal representation.
- **Make tracing the opinionated diagnostics core (everything emits tracing spans/events; the report is reconstructed from the trace)** — rejected: Tempting for DX but violates §2.6 (core depends on no integration) and §7 (must build on wasm/embedded where a tracing subscriber may be absent), and it is structurally wrong for §1.7: the highest-value diagnostics fire during the cold, pre-bus, pre-bean assembly phase where there is no subscriber and no async context yet, so a push-based span stream has nowhere to go. It also makes the aggregated startup AssemblyReport (collect-ALL-failures-before-aborting) unnatural — tracing is fire-and-forget per event, not an accumulator. Chosen instead: a dependency-free pull/handle recorder as the spine, with leaf-tracing as the dominant-target bridge — keeping the report as the source of truth and tracing as an optional sink.
- **Push every wiring/diagnostic check to compile time via type-state proofs (a 'no provider for T' becomes a trait-bound error)** — rejected: §2.9 explicitly declines this: 'we do not force a check to compile-time if that only yields a cryptic error a deferred check could explain fully.' A missing/ambiguous provider as a trait-bound failure carries no assembly chain, no candidates-considered, no source-crate naming, and is impossible cross-crate under maximal-magic discovery (the binary's macro cannot see sibling providers) and for dynamic/WASM beans. compile-runtime-split already fixed cross-bean wiring at the Tier-2 startup pass for exactly this reason. Tier 0 is reserved for genuinely-legible local errors (generic bean, malformed CondExpr, Send+Sync bound).
- **Fail-on-first-error during the eager-instantiate pass (Spring's preInstantiateSingletons-surfaces-errors model), no aggregated report** — rejected: Pessimal for §2.9 richest-context and already rejected by the upstream compile-runtime-split ADR. Building beans one-by-one surfaces the FIRST wiring fault and aborts, hiding the other N problems and entangling pure wiring faults with construction/IO faults. The chosen Tier-2 whole-graph pass aggregates ALL failures into one AssemblyReport before any constructor runs — strictly earlier and strictly richer. This ADR's job is to give that pass the shared LeafError vocabulary to aggregate, not to re-litigate where it runs.
- **A separate ExitCodeExceptionMapper SPI parallel to FailureAnalyzer (Spring-faithful: two SPIs, one for the message, one for the code)** — rejected: The bootstrap/exit-code-shutdown design itself flagged this as duplication ('rather than two parallel SPIs'). Both the message and the exit code are functions of the SAME typed error; an exit_code() fn over ErrorKind plus a single override hook gives 'port-in-use → analysis + action + conventional exit code' as ONE coherent story keyed on one taxonomy, instead of two registries that can disagree about what a given failure means. Folded into this ADR's single spine.

## Consequences
- FORCES leaf-core to host an ultra-stable error ABI alongside the registration ABI: LeafError, Cause, ErrorKind (closed core + one open Integration{ContractId} arm), Severity, Origin (shared with config/registry ADRs), the Diagnostic trait, RenderStyle, and the FailureAnalysis struct. Like the Descriptor ABI these are emitted-against / matched-against workspace-wide, so they must be frozen early and versioned conservatively; adding an ErrorKind variant is a minor-but-careful change, hence the open Integration arm exists so integrations never need a core bump.
- FORCES the App<Wired> startup-validation pass (compile-runtime-split ADR) to be the single producer of the aggregated AssemblyReport { failures: Box<[LeafError]> } and to aggregate ACROSS subsystems — autowiring (NoSuchBean/NoUniqueBean + candidate trace), binder/conversion/validation (the shared collect-all accumulator), profiles, scope-mismatch, @DependsOn cycles — never fail-on-first. Each subsystem must therefore produce LeafError nodes (not its own enum) and a definition-level, no-instantiation resolve that yields the candidates-considered trace.
- FORCES the renderer to take BOTH the AssemblyReport AND the frozen ConditionReport + anti-DCE ExpectedManifest as inputs, so a Tier-2 NoSuchBean is enriched with 'backed off because…' / 'feature not enabled' / 'crate expected but produced zero registrations — add AND force-link it'. This couples the diagnostics model tightly to the condition-report ABI (CompiledOutByCfg / BuildFoldedFalse / ConditionVeto reason taxonomy must be frozen in the core) and to codegen-boundary's anti-DCE manifest.
- FORCES a FAILURE_ANALYZERS linkme slice (codegen-boundary substrate) collecting const FailureAnalyzer descriptors, @Order-sorted, first-non-None wins, with built-in analyzers for port-in-use/NoSuchBean/NoUniqueBean/BindError/anti-DCE. Generics cannot carry a 'static descriptor → #[leaf::failure_analyzer] hard-errors with a register_failure_analyzer!(Concrete) shim (same rule as #[component]). A DCE-dropped analyzer = a QUIETER (worse-message, never-crash) failure mode, so this slice is a primary driver of the mandatory anti-DCE expected-vs-found self-check.
- FORCES the macro layer (leaf-macros) to thread proc_macro::Span (file/line/col, 1.88 stable) into the const Origin for every #[component]/#[value]/#[bean]/#[conditional], to own the #[diagnostic::on_unimplemented] doctrine message for the Send+Sync+'static managed-bean bound (1.78 stable), and to accumulate Tier-0 errors via syn::Error::combine. No nightly required (§2.8). cargo expand stays legible — the macro carries no chain-building or rendering logic (§2.10).
- FORCES the observability spine to be a dependency-free pull/handle StartupRecorder in leaf-core/leaf-boot frozen into FrozenContainer, with tracing relegated to an opt-in leaf-tracing bridge crate. LeafError implements std::error::Error with a real source() walk for ecosystem interop (?, anyhow, tracing error fields), but its richness lives in kind/chain, NOT a backtrace. The no-op default delivery means zero cost when diagnostics are not consumed.
- FORCES exit-code mapping to be exit_code(&LeafError) -> ExitCode over ErrorKind + a single override hook (no separate ExitCodeExceptionMapper SPI), and the boot RunFailure to carry a LeafError (failing RunPhase/StageId in the chain) rather than a bespoke failure type — unifying the bootstrap design's RunFailure/FailureAnalysis with the container's AssemblyReport under one chain.
- ACCEPTS that the dynamic-WASM second epoch (reopen_epoch → re-seal) re-runs Tier 1+2 and produces a FRESH AssemblyReport + ConditionReport for that epoch (origin=DynamicWasm tagged in each node), and that a WASM-contributed error enters via the open Integration{ContractId} arm with no Rust type — gated/typed only at runtime, consistent with the mandatory runtime tier the upstream ADRs fixed.
- ACCEPTS that 'compile-time errors' are deliberately SCOPED to locally-legible cases; cross-bean wiring is NEVER a compile error (it is the rich Tier-2 pass). leaf must document this honestly: the diagnostics model delivers FAST + EARLY + RICH + AGGREGATED startup validation and teachable FailureAnalysis prose, NOT a fully compile-checked bean graph (impossible under maximal-magic discovery + dynamic sources).

## API / type sketch

// ═══ leaf-core: the ultra-stable diagnostic ABI (shared workspace-wide, like the registration ABI) ═══

// ── THE ONE CAUSAL-CHAIN TYPE (§2.9: what/needed/expected/found+candidates/where) ──
pub struct LeafError {
    pub kind:   ErrorKind,          // the closed taxonomy (+ one open arm)
    pub chain:  Box<[Cause]>,       // explicit causal narrative (Spring nested-exception INTENT, as DATA not unwinding)
    pub origin: Origin,             // shared with config/registry ADRs: Span | File{path,line} | Crate(SourceTag) | DynamicWasm
    pub mode:   Severity,           // Fatal | Warn | Info
}
impl std::error::Error for LeafError {                 // interop facade ONLY (?, anyhow, tracing) — richness is in kind/chain
    fn source(&self) -> Option<&(dyn std::error::Error + 'static)>; // walks self.chain, never a backtrace
}
pub struct Cause { pub what: &'static str, pub detail: CauseDetail, pub origin: Origin } // one node of the narrative
pub enum CauseDetail {                                  // the "what was actually found" payload
    Candidates  { considered: Box<[CandidateInfo]>, trace: Box<[NarrowStep]> }, // NoUniqueBean / determine_winner trace
    Expected    { ty: TypeKey, missing_source: Option<SourceTag> },             // NoSuchBean / anti-DCE
    AssemblyAt  { bean: BeanId, edge: InjectionEdge },                          // construction/value-phase context
    Plain(String),
}
pub struct CandidateInfo { pub bean: BeanId, pub crate_src: SourceTag, pub verdict: CandidateVerdict } // per-candidate provenance

// ── CLOSED CORE TAXONOMY + ONE OPEN DATA ARM (by-data extensible, not a dyn trait family) ──
#[non_exhaustive]
pub enum ErrorKind {
    NoSuchBean, NoUniqueBean, ScopeMismatch, DependsOnCycle,            // Tier 2 wiring
    UnresolvedValue, Convert, Bind, Validation, Profile,               // Tier 2 config/validation
    AntiDce, ContractCollision, AutoConfigOrdering, PlanStale,         // Tier 1 build/freeze
    ConstructionFailed, ConfigIo, Cancelled,                           // Tier 3 value-phase
    Integration { kind_id: ContractId },        // integrations (e.g. DataAccessError translation) extend BY DATA, keyed by ContractId
}
impl ErrorKind { pub fn exit_code(&self) -> ExitCode; } // folds the ExitCodeExceptionMapper into ONE story; override hook on App

// ── THE ONE RENDERER (FailureAnalysis prose overlays the always-present chain) ──
pub trait Diagnostic { fn render(&self, w: &mut dyn std::fmt::Write, style: RenderStyle) -> std::fmt::Result; }
pub enum RenderStyle { Human, StructuredJson, TracingFields }   // no-op default delivery
impl Diagnostic for LeafError {/* walks chain + overlays matched FailureAnalysis + consults ConditionReport */}
impl Diagnostic for AssemblyReport {/* all failures at once */}  impl Diagnostic for ConditionReport {/* silent-now context */}

// ── AGGREGATED STARTUP REPORT (produced by compile-runtime-split's App<Wired>::validate) ──
pub struct AssemblyReport { pub failures: Box<[LeafError]> }    // collect-ALL-before-abort; the shared cross-boundary accumulator
pub struct ErrorSink { /* the one accumulator binder+conversion+validation+autowiring push into */ }
impl ErrorSink { pub fn push(&mut self, e: LeafError); pub fn into_report(self) -> Result<(), AssemblyReport>; }

// ── FailureAnalyzer SPI (Spring shape; collected via codegen-boundary FAILURE_ANALYZERS slice; rendering-policy over LeafError) ──
pub struct FailureAnalysis { pub description: String, pub action: String, pub cause: Option<Cause> }
pub trait FailureAnalyzer: Send + Sync {
    fn analyze(&self, err: &LeafError, ctx: &AnalysisCtx) -> Option<FailureAnalysis>; // match on ErrorKind/downcast over chain
}
#[linkme::distributed_slice] pub static FAILURE_ANALYZERS: [AnalyzerDescriptor] = [..]; // @Order-sorted, first-non-None wins
// thin macro: #[leaf::failure_analyzer] → one const descriptor; degrades to App::add_failure_analyzer(MyAnalyzer) (escape hatch)
pub fn analyze_first(err: &LeafError, ctx: &AnalysisCtx) -> Option<FailureAnalysis>; // the chain walk (built-ins: port-in-use, NoSuchBean, NoUniqueBean, Bind, AntiDce)

// ── OBSERVABILITY SPINE: dependency-free pull/handle recorder (NOT a push stream; frozen into FrozenContainer) ──
pub struct StartupRecorder { /* records assembly timeline + condition outcomes + anti-DCE result */ }
impl StartupRecorder { pub fn handle(&self) -> RecorderHandle; pub fn freeze(self) -> Arc<ConditionReport>; }
// feature-gated leaf-tracing: pub fn bridge(rec: &StartupRecorder); + impl tracing integration for LeafError (core depends on NO integration)

// ── TIER 0 (leaf-macros, compile-time, stable; NO logic beyond span emission + locally-legible errors) ──
// #[diagnostic::on_unimplemented(message="a leaf #[component] must be Send + Sync + 'static")] on the bean bound (1.78)
// generic #[component] → syn::Error::new(span, "generic beans cannot be registered; use register_component!(Concrete)")
// malformed CondExpr / ${..} / @AliasFor → compile_error! with proc_macro::Span (1.88); multi-error via Error::combine

## Forces on other concepts
- **Metadata / Codegen Boundary (upstream codegen-boundary ADR)**: Honored and built upon: the FAILURE_ANALYZERS and CONVERTERS/translators slices ride the SAME linkme substrate as COMPONENTS; the #[diagnostic::on_unimplemented] doctrine and the ContractId collision const{assert!} this ADR surfaces are exactly the diagnostics that ADR already mandated, now rendered through the unified Diagnostic. Origin (with proc_macro::Span) is emitted by the macro per that ADR's thin-macro rule. A DCE-dropped analyzer is caught by that ADR's anti-DCE expected-vs-found self-check — this ADR adds the analyzer slice to the expected set.
- **Compile-vs-Runtime Resolution Split (upstream compile-runtime-split ADR)**: Honored unchanged: the four timing-tiers ARE that ADR's phase ladder (compile / build-or-seal / App<Wired> / App<Running>); the AssemblyReport is produced by App<Wired>::validate; the silent ConditionReport is frozen into FrozenContainer at seal(); the dynamic second-epoch re-seal re-runs Tier 1+2. This ADR supplies the shared LeafError vocabulary those phases aggregate and the renderer that joins the loud report with the silent report. It adds NO new phase.
- **@Conditional Condition SPI / Conditional-On-X Family / Condition Evaluation Report**: The ConditionReport's reason taxonomy (ConditionVeto, CompiledOutByCfg, BuildFoldedFalse, BackedOffByUserBean) MUST be frozen in the core ABI and machine-parseable, because the Tier-2 renderer consults it to enrich NoSuchBean ('backed off because…' / 'feature X not enabled'). condition-not-met is a ConditionOutcome record, NEVER a LeafError — the type-and-time separation is binding. The report is a render INPUT, not an error stream.
- **Autowiring / Candidate Resolution & @Primary/@Fallback**: MUST produce LeafError nodes (NoSuchBean/NoUniqueBean) carrying CauseDetail::Candidates with per-candidate SourceTag provenance and the determine_winner NarrowStep trace ('matched 3; FallbackDemote dropped 1; PrimaryPromote 0 → ambiguous among [appA@crate_x, appB@crate_y]'). The candidates-considered narrative is not optional — §2.9 names it explicitly. Carry the trace only on the >1/error path (cost discipline, §2.5).
- **Binder / Conversion / Validation / @Value Injection**: All four MUST push into the ONE ErrorSink accumulator (collect-all-before-abort at Tier 2) rather than fail-on-first, and emit UnresolvedValue/Convert/Bind/Validation LeafError nodes carrying Origin = File{path,line} (config provenance). Strict-vs-lenient for @Value is a per-site/framework knob whose strict failure is a Tier-2 UnresolvedValue node; a LazyValue.get() failure is necessarily Tier-3. The shared accumulator is the register's 'one collect-all-across-this-typed-boundary'.
- **Bootstrap / FailureAnalyzer / Exit Code (07-bootstrap.md)**: RunFailure carries a LeafError (failing RunPhase/StageId in the chain), the FailureAnalyzer chain is rendering-policy over LeafError (NOT a parallel cause system), and exit-code mapping is exit_code(&LeafError) + one override hook (no separate ExitCodeExceptionMapper). The boot terminal stage runs analyze_first then renders via Diagnostic then drives context-close. Built-in analyzers ship for port-in-use/NoSuchBean/Bind/anti-DCE. AbandonedRunException short-circuits skip analysis+exit-code.
- **Persistence Exception Translation (cross-crate error extensibility)**: DataAccessError is realized as ErrorKind::Integration{kind_id: ContractId} extended BY DATA (not a dyn DataAccessError trait family), so an integration crate contributes its taxonomy + a translator without a core ABI bump. The 'silent-untranslated-error self-check' becomes an anti-DCE expected-vs-found check on the translators slice (a dropped translator = loud, not silently-untranslated). Rollback-rule matching reuses ErrorKind/downcast over the chain.
- **Scope / Lifetime / Ownership Model (upstream ownership-model ADR)**: Honored unchanged: a Tier-3 ConstructionFailed LeafError threads the bean's assembly context (CauseDetail::AssemblyAt{bean, edge}) without holding a live object; the report/recorder is Arc-readable from the immutable FrozenContainer snapshot (safe-publication-for-free, no lock, §2.5). Partial-construction unwind on a failed/cancelled build records a Cancelled/ConstructionFailed node; no async Drop — teardown errors aggregate into the same chain.
- **Observability / tracing integration (charter §2.6, §6)**: tracing is a feature-gated leaf-tracing bridge crate, NOT a core dependency; the core ships only the dependency-free pull/handle StartupRecorder. LeafError implements std::error::Error for interop but never depends on tracing. On wasm/embedded the recorder works with no subscriber. This fixes the register's 'tracing as core vs bridge' toward the bridge and keeps the core buildable on stable with no integration dependency.
- **WASM Optional Layer / Two-Source Duality**: A WASM-contributed error enters via ErrorKind::Integration{ContractId} with origin=DynamicWasm and is rendered identically (origin-agnostic). The dynamic second-epoch re-seal produces a fresh AssemblyReport + ConditionReport for that epoch. A WASM error has no Rust type so it is matched only by ContractId/data at the mandatory runtime tier — consistent with why that tier exists.
