# Referenced-Constructor Injection Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (or executing-plans) to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a stereotype reference a constructor by path (`#[repository(constructor = OrderRepository::new)]`) so state-holding beans can wear stereotypes, while keeping field injection as the no-constructor default — replacing the removed `#[inject]`/`#[advisable]`-constructor/runtime-merge machinery.

**Architecture:** A per-arity `InjectableCtor<Args, T>` "magic constructor" trait (axum-style) plus `construct_with`/`ctor_deps` inference drivers resolve a referenced constructor's parameters through the existing `Injectable` trait, with type inference picking the arity from a bare `Type::new` value (proven by spike). The stereotype macro reads an optional `constructor = <path>` argument from its own attribute: absent → today's field-injection provider; present → a single `construct_with` provider + `ctor_deps` plan and **no** field-default. One provider per bean, so no runtime merge.

**Tech Stack:** Rust, proc-macros (`leaf-macros`/`leaf-codegen`), linkme distributed slices, the committed `Injectable` trait (`leaf-core/src/injectable.rs`).

**Base:** branch `scaffold/workspace` at `21d586b` (workspace green, 1440 tests, `produced_ty` name-check deleted). The design spec is `docs/superpowers/specs/2026-06-15-constructor-injection-design.md` (revised 2026-06-16). The standalone feasibility spike is `/tmp/injectable_ctor_spike.rs` (compiles + runs).

**Gate for every task:** before committing, run a FRESH `cargo test --workspace` (0 failures) and `cargo clippy --workspace --all-targets -- -D warnings` (clean). Cached `Finished in 0.0s` runs re-emit no warnings — force a real recompile (touch the changed crate) before claiming clean. rustc/clippy skip naming-convention lints on macro-generated identifiers but rust-analyzer does not — emit `#[allow(non_upper_case_globals, non_camel_case_types, non_snake_case)]` (as appropriate) on every generated helper item.

- [ ] **Step 3: Implement** the trait, the per-arity impls (0 through 12, generated by a local `macro_rules!` to avoid hand-repeating), and the two free drivers. Build each `InjectionPoint` from `<Pi as Injectable>::RESOLVABLE` with a positional name (`"arg0"`, …) — reuse however `descriptor.rs`/`injectable.rs` turns a `Resolvable` into an `InjectionPoint` today (a `Resolvable::into_point(name)`-style helper; add one if absent, keyed off the existing fields `produced`/`generics`/`qualifiers`/`arity`/`collection`).

```rust
pub trait InjectableCtor<Args, T>: Sized {
    fn construct<'a>(self, ctx: &'a ResolveCtx<'a>) -> BoxFuture<'a, Result<T, LeafError>>;
    fn deps(&self) -> Vec<InjectionPoint>;
}

macro_rules! impl_injectable_ctor {
    ( $( $P:ident ),* ) => {
        impl<F, T, $($P),*> InjectableCtor<($($P,)*), T> for F
        where
            F: Fn($($P),*) -> T + Send + Sync + 'static,
            T: Send + 'static,
            $( $P: $crate::Injectable, )*
        {
            #[allow(non_snake_case, unused_variables)]
            fn construct<'a>(self, ctx: &'a ResolveCtx<'a>) -> BoxFuture<'a, Result<T, LeafError>> {
                Box::pin(async move {
                    $( let $P = <$P as $crate::Injectable>::inject(ctx).await?; )*
                    Ok(self( $($P),* ))
                })
            }
            fn deps(&self) -> Vec<InjectionPoint> {
                let mut i = 0usize; let mut v = Vec::new();
                $( { let _ = stringify!($P); v.push(point_from::<$P>(i)); i += 1; } )*
                v
            }
        }
    };
}
impl_injectable_ctor!();
impl_injectable_ctor!(P1);
impl_injectable_ctor!(P1, P2);
// … through P1..P12

pub fn construct_with<'a, F, Args, T>(ctor: F, ctx: &'a ResolveCtx<'a>)
    -> BoxFuture<'a, Result<T, LeafError>> where F: InjectableCtor<Args, T> { ctor.construct(ctx) }
pub fn ctor_deps<F, Args, T>(ctor: F) -> Vec<InjectionPoint> where F: InjectableCtor<Args, T> { ctor.deps() }
```
(`point_from::<P>(i)` builds an `InjectionPoint` from `<P as Injectable>::RESOLVABLE` and the positional name `arg{i}`. If the deps must be const rather than runtime, that is impossible here because `Args` is unspellable — runtime `deps()` called at assembly is the design choice; document it with a `///` comment.) Add a `#[diagnostic::on_unimplemented]` on `InjectableCtor` steering non-`Injectable` parameters to `Ref<T>`.
